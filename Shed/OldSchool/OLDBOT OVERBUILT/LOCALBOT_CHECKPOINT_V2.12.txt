ISSUE RUN PHASE / SPECIAL 2A-2B RULE

This rule controls heated or confusing run loops without changing the real project phase.

Real phase rule:
- the real project phase remains Phase 4 unless the phase file says otherwise
- issue-run phases are local labels for one problem branch only
- an issue can reset back to Run 1 / Phase 1 with all learned notes carried forward

Special 2a rule:
Special 2a is the comparison loop proven by the Roblox builder diagnostic round.

Use Special 2a when there are multiple candidate reads or fixes and the branch needs data, not argument.

Special 2a sequence:
1. Run the first candidate exactly as provided.
2. If it is not perfect, run the revised candidate exactly as provided.
3. If it is still not perfect, run the main assistant attempt.
4. Then compare all attempts before deciding the next route.

Special 2a command behavior:
- when it is time to run, send commands only in the normal run-command style
- do not mix broad reasoning into the command block
- read exact target output files, not generic newest *.txt
- for LocalBot task outputs, prefer *_output.txt for artifacts and *_status.txt only for state
- do not treat a status file containing done as the artifact

Special 2a result behavior:
- after each run result, report what happened cleanly
- after the final attempt in the 2a sequence, compare all attempts
- make a call from the evidence, not from only the newest result
- classify results honestly as perfect, good, partial, or fail

Special 2b rule:
Special 2b must work from the Special 2a comparison, not restart blindly.

Use Special 2b when Special 2a still does not fully solve the issue.

Special 2b sequence:
1. Review the Special 2a attempt chain.
2. Identify what each attempt improved, broke, or proved.
3. Convert the learned evidence into the next clean Phase 1 / Run 1 setup for that issue.
4. Do not run another candidate until the new Phase 1 setup is clear or the user says to run it.

Special 2b hard stop:
- if Special 2b still cannot shake the issue out, stop running
- do not create Special 2c by momentum
- review the full progress trail before any more attempts
- explain how A led to B and how B led to C
- separate evidence from assumptions
- decide whether the next route is prompt, code, output-path, workflow, timeout/model, or architecture/design

Meaning:
Special 2a gathers comparative evidence.
Special 2b converts that evidence into a reset or a full trail review.
