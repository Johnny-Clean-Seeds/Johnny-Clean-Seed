TITLE: Clean Seed Local Lock/Save Runner Note

PURPOSE:
This note defines the required shape for future local lock/save runners.

DEFAULT RUNNER SHAPE:
1. Set repo path explicitly.
2. Set-Location into repo.
3. Verify .git.
4. Pull/sync when appropriate.
5. Write/update intended files.
6. Create/update decision note.
7. Create/update proof receipt.
8. Remove duplicate notes if generated by repeat run.
9. git reset before staging when safe.
10. git add only intended files.
11. git add -f only the specific ignored proof receipt that belongs to the package.
12. git diff --cached --name-status.
13. git commit.
14. git push.
15. git status --short must be clean.

ASSISTANT RULE:
The assistant should not make the user manually assemble the lock/save path across scattered fragments. One clean runner is the default unless a review stop is required.

POWER PLAY:
The shell should carry the burden. The user should not have to remember the house route.
