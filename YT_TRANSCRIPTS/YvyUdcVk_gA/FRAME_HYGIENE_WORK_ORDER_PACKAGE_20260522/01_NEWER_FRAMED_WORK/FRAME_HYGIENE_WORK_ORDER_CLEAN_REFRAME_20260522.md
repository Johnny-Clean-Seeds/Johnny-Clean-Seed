Dirty meanings are blocked:

Clean meaning:
Use the concepts to keep work in the right lane, with the right authority, the right scope, and the right proof.

## Clean Translation

Identity becomes: What is this object allowed to be?

Examples:
- source ore is not doctrine
- a draft is not a rule
- a transcript is not proof by itself
- a receipt is proof of a process, not authority over the whole system
- a mule output is evidence until dispositioned

Control becomes: What route are we on, what is the next safe move, and who/what has authority?

Examples:
- ACTIVE_ANCHOR controls current active ball
- current user command controls immediate task
- proof gates control PASS
- no queue item becomes active by inertia
- no outside source becomes doctrine without routing

Safety becomes: What containment, reversibility, proof, and boundary protection are needed?

Examples:
- keep originals separated from newer work
- keep receipts with artifacts
- mark candidate status clearly
- prevent false PASS
- keep local-only and public lanes separated
- do not overwrite authority files without explicit scope

## Clean Use Rule

Before using any concept from the transcript, translate it into Clean Seed terms first.

The allowed move is not:
"How do we control the person/frame?"

The allowed move is:
"What frame is this work trying to enter, and is that frame clean, authorized, bounded, and proven?"
