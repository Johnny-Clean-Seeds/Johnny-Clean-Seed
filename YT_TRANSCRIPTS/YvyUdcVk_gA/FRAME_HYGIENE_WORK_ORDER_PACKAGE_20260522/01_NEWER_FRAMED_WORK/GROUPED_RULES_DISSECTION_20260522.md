# Grouped Rules Dissection

## Group 1 - Threat Diagnosis

The source model names three destabilizers:

1. Identity threat
   - self-concept is challenged
   - person/object defends what it thinks it is
   - Clean Seed translation: file/tool/rule/task identity has become unclear

2. Control threat
   - agency, route, or next move feels hijacked
   - Clean Seed translation: work is losing lane control or next-action clarity

3. Safety threat
   - status, place, proof, or containment feels unsafe
   - Clean Seed translation: proof, boundary, reversibility, or containment is weak

## Group 2 - Frame Hygiene

A frame decides what the situation means.

Clean use:
- declare the work frame
- declare the authority boundary
- declare the artifact status
- declare the proof requirement
- declare what is blocked

Dirty use blocked:
- hiding the frame
- smuggling authority
- pretending source ore is doctrine
- using labels to win

## Group 3 - Pre-Frame for Work Order

Before a task starts, define:

1. What are we doing?
2. What are we not doing?
3. What file lane is active?
4. What authority controls the move?
5. What proof will show PASS?
6. What would make this invalid?
7. What must be parked?

## Group 4 - Conflict / Fog Response

When work destabilizes:

1. Stop arguing with the symptom.
2. Diagnose identity/control/safety pressure.
3. Re-state the clean frame.
4. Reduce the pressure without surrendering authority.
5. Return to next safe proof-bearing move.

## Group 5 - Phrase Translation

Original idea: "Do not explain yourself under pressure."
Clean Seed use: Do not defend a false frame. Re-state scope and proof.

Original idea: "Do not argue motive."
Clean Seed use: Do not debate identity labels. Clarify role, lane, and action.

Original idea: "Do not accept loaded language."
Clean Seed use: Define terms before letting them govern the work.

Original idea: "Do not answer binary traps."
Clean Seed use: Reject false either/or routing when the model has more than two clean states.

Original idea: "Use process language."
Clean Seed use: Name the route confusion and restore work-order alignment.

## Group 6 - Clean Warnings

This material becomes dirty if used to:
- dominate people
- manipulate agreement
- corner someone
- hide intent
- win instead of solve
- bypass proof
- force a frame without declaring it

It stays clean if used to:
- protect proof
- preserve lanes
- lower fog
- identify threat pressure
- keep artifacts honest
- stop false authority
- return to work order
