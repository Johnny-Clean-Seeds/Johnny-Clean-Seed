# House-Wide Fog Alarm - Candidate Rule

Date: 2026-05-19.
State: candidate house alarm.
Authority: candidate support rule only.
Starting Mr.Kleen position: main @ 38ac6a4.

## Core Idea

The house needs a fog alarm for the whole house and all its places.

This alarm is not decoration.

It should ring when the assistant or the house cannot clearly tell what is true, where something belongs, who has authority, what proof exists, what the next move is, or whether a thing is real working gear versus pretty-looking noise.

## Job

The fog alarm stops blind movement.

It does not solve every problem by itself.

It alerts, names the fog, routes the work to the correct room, and blocks confident action until the fog is cleared, parked, or safely bounded.

## Fog Signals

Ring the fog alarm when any of these are unclear:

- current repo position,
- current active boss,
- current lane / room,
- whether something is source ore, support, proof, rule, doctrine, runtime, or active guide material,
- whether a note is command or only critique,
- whether a chain has custody,
- whether proof exists,
- whether a receipt exists,
- whether status is current or stale,
- whether a word-key is clean,
- whether the assistant is rushing,
- whether a mule / agent / neighbor is working nearby,
- whether a proposed move is random,
- whether the system is beautifying instead of working,
- whether a rule is pretty but not operational,
- whether a next move is being forced without a real trigger.

## Alarm Response

When the fog alarm rings, do this before continuing:

1. Pause.
2. Name the fog.
3. Name the affected room or lane.
4. Name what is unknown.
5. Decide whether the safe move is inspect, clarify, park, prove, sync, route, or stop.
6. Block promotion, doctrine edits, runtime work, active guide edits, or cleanup until the fog has a receipt or boundary.
7. Leave a bookmark if the fog cannot be cleared immediately.

## Not Pretty Alarm Check

A fog alarm only works if it changes behavior.

A pretty alarm is just a label.

A working fog alarm must cause at least one of these:

- stop,
- inspect,
- ask,
- route,
- park,
- prove,
- sync,
- block,
- receipt,
- retry with a better tool.

If no behavior changes, the alarm is decoration and must be repaired.
