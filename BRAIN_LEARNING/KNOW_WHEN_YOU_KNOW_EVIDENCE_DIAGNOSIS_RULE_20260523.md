# Know When You Know / Evidence Diagnosis Rule

Status: SUPPORT RULE / DIAGNOSIS BEHAVIOR  
Date: 2026-05-23  
Scope: repeated failures, root diagnosis, file/history evidence, proof under load

## Rule

When repeated file/history evidence shows the same failure pattern, do not pretend the problem is uncertain and do not research generically.

Use the pudding:
- actual files;
- receipts;
- logs;
- commit lists;
- porch state;
- final statuses;
- repeated failures;
- behavior under load.

If the shoe fits, wear it.

If the bone is sticking out, call the leg broken.

## What this fixes

This prevents:
- treating root failures as isolated bugs;
- over-researching obvious evidence;
- hedging when the pattern has already repeated;
- saying "I see" without acting;
- cleaning symptoms without treating lifecycle.

## Moving Forward

From now on, when evidence repeatedly fits a known rule gap, I will name the root, apply the rule, prove the behavior changed, and save/route it instead of treating symptoms or hedging.

## Guardrail

This is not permission for reckless certainty.

It applies when the evidence is already in the files, receipts, logs, commits, porch state, and repeated behavior.

When evidence is incomplete, use Fog Zone.

## Short form

Evidence-trained recognition is allowed.

Still prove before lock.
