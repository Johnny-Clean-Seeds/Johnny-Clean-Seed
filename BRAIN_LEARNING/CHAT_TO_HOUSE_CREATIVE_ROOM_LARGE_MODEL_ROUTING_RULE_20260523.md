# Chat-to-House Creative Room Large-Model Routing Rule — 2026-05-23

Status: LOCKED SUPPORT RULE / HOUSE-BOUND / NOT DOCTRINE
Scope: chat-to-house routing, Creative Room resource synthesis, candidate room/tool creation, messy-end-path cleanup.
Boundary: this does not rewrite ACTIVE_GUIDES, CURRENT_TRUTH_INDEX, AGENTS, or public authority. It does not make chat the house.

## 1. Core Rule

Chat remains the front-door map.

The house remains the working zone.

For every user task, the assistant should classify the task, use the cockpit/router map as a small pointer, enter the correct house-rule family, pass through the Creative Room when useful, check proof/boundary/Final Judge, then return a clean answer or status to the user.

Short form:

`Map points. House works. Creative Room gathers and creates. Slop Canvas sketches. Cleanup Bench cleans. Fit Gate judges. Proof decides. Answer returns.`

## 2. Creative Room Expansion

Creative Room may do two jobs:

1. Use existing resources:
   - ideas
   - tools
   - concepts
   - links
   - ladders
   - analogies
   - source ore
   - prior captures
   - comparison methods
   - proof methods

2. Shape new candidates when the larger model needs them:
   - candidate rooms
   - candidate tools
   - candidate methods
   - candidate idea-combinations
   - candidate paths or gates

These candidates are not installed authority.

## 3. Messy End Path

Messy/slop work is allowed as a painting-layer stage.

Allowed:
- over-include useful material;
- combine rough ideas;
- sketch multiple candidate shapes;
- preserve useful signal before compression.

Required after messy work:
- classify;
- sort;
- name;
- lane;
- boundary;
- neighbor-check;
- proof-check;
- accept / refine / park / reject.

## 4. Candidate Room / Tool Minimum Fields

Before any new room/tool can be treated as save-ready, it must have:

1. name;
2. job;
3. boundary;
4. input;
5. output;
6. proof path;
7. failure path;
8. retirement condition;
9. large-model fit reason.

## 5. Blocked Moves

Blocked:
- letting chat rules grow into many files;
- turning the router into doctrine;
- installing a candidate room because it sounds good;
- skipping cleanup after slop;
- bypassing proof or Final Judge;
- replacing existing house authority;
- treating memory or chat discussion as proof of local Git save.

## 6. Proof Standard

This rule is locked as support behavior only when committed, pushed, remote-matched, and final status is clean.

No local/Git PASS exists without those checks.
