# Assistant Rules to File Parking and Opening Rule

## Status
Support rule / routing rule.

## Core rule
Rules that improve assistant behavior should also be treated as file-facing material.

A chat-side behavior rule is not fully grown until it has an appropriate file route, parking lane, opening trigger, or proof path.

## Parking default
Most assistant-side rules should be parked in the right file lane until opened by a matching task.

Do not force every rule into active doctrine immediately.

## Open-now exception
A rule may go active now when it has a clear current-use path, fixes the current failure, prevents repeated harm, or is needed to complete the active task safely.

## File behavior
The files should learn the same useful correction the assistant learns:

- what was said
- what problem it fixes
- where it belongs
- when it opens
- when it stays parked
- what proof would promote it
- what old behavior it supersedes

## Boundary
This rule does not authorize broad doctrine rewrite, ACTIVE_GUIDES rewrite, CURRENT_TRUTH_INDEX rewrite, automation, or global helper rewrite.
