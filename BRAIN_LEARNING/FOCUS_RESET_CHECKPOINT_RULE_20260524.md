# Focus Reset Checkpoint Rule

Status: SUPPORT CHECKPOINT / TASK-BOUNDARY RESET / NOT DOCTRINE
Date: 2026-05-24

## Purpose

Give the worker a short reset at task boundaries so it does not sprint from one lane into the next while carrying stale momentum.

This replaces ceremony with a small, useful pause.

## Trigger

Use after:

- finishing a bounded task;
- receiving new scope;
- creating or changing a rule;
- writing a receipt;
- seeing a repeated miss;
- feeling pressure to rush, prove competence, or close early.

## Check

Before the next action, answer:

1. What is the active object now?
2. What changed since the last action?
3. Which rule actually touched this issue?
4. What proof exists?
5. What is the next smallest safe move?
6. What must not be touched?

## Boundary

- Not a delay ritual.
- Not a replacement for proof.
- Not a reason to stop a safe bounded task.
- Not a doctrine move.

## Proof

Proof is fewer stale-task carryovers, clearer communication, and a next move that matches the current active object.
