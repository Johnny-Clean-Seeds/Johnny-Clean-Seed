# Clean Seed Build vs Other Noise Phase Boundary Rule

Date: 20260522_035244

Status: current-stage intake and phase-boundary rule.

## Rule

During active Clean Seed work, incoming material must be sorted by fit before it is allowed to influence the current build.

Material that strengthens the current Clean Seed Build may enter the working house.

Material that does not fit the current build stage is not deleted by default. It is parked as source ore, idea-horizon material, later-phase material, or external noise until its proper phase opens.

## Clean Seed Build

Clean Seed Build material directly supports the current model, current proof flow, current growth stage, or current house operation.

It may include material that improves:

1. Authority boundaries.
2. Proof handling.
3. Receipts, hashes, clean status, and remote match discipline.
4. Debuggers, alarms, anomaly checks, and failure classification.
5. Artifact quality.
6. Rule clarity.
7. Bloat handling.
8. Whirlpool preparation.
9. Mule handoff and return discipline.
10. Parking, intake, recovery, and lifecycle routing.
11. Source custody.
12. Active build stability.
13. Recursive/circular growth behavior.
14. Neighbor-lane fit.
15. Future compression into denser clean rules.

## Other Noise

Other Noise is any material that may be interesting, useful later, symbolic, speculative, aesthetic, personal, external, phase-early, or disconnected from the current Clean Seed Build lane.

Other Noise may include:

1. Later-stage architecture that is not ready.
2. Symbolic or creative material that does not currently support the build.
3. External platform friction that belongs outside the model.
4. Speculative source ore without a current proof route.
5. Attractive ideas that would distract from the current boss.
6. Unsorted concepts that need parking before use.
7. Research material not yet mapped to house needs.
8. Repeated language that does not improve coverage, proof, routing, or recovery.
9. Interesting patterns whose phase has not opened yet.
10. Anything that would blur doctrine, active guides, proof, parking, or source lanes if inserted too soon.

## Handling rule

Clean Seed Build material may be polished and placed into the correct current lane.

Other Noise must be preserved only if useful, then parked in the correct non-authority lane.

Other Noise must not bleed into active artifacts, active model behavior, doctrine, guides, current truth, proof claims, or current build authority.

## Boundary

Parking is not rejection.

Parking is not promotion.

Current-fit material is not automatically doctrine.

Other Noise is not automatically worthless.

The gate is fit, timing, proof, and phase.

## Clean test

Before material enters the current build, ask:

1. Does this strengthen the current Clean Seed Build?
2. Does this belong to the current phase?
3. Does this improve proof, routing, recovery, coverage, debugging, alarm behavior, artifact quality, or dense growth?
4. Does this preserve authority boundaries?
5. Does this need to be parked instead?
6. Would placing it now create noise, drift, false authority, or phase bleed?

If the answer is unclear, park it.
