# Gear-Change Closeout / Lifted Fog Re-Walk Rule

Date: 2026-05-30
Status: SUPPORT RULE / LIVE BEHAVIOR
Authority: not doctrine, not ACTIVE_GUIDES, not CURRENT_TRUTH_INDEX

## Rule

When a task closes, direction changes, fog lifts, or the worker is about to shift lanes, do not jump straight into the next job.

First close the work like a worker leaving a job site:

1. Put the needed live state into the suit.
2. Burn dead chat weight.
3. Keep the useful suit signal, not the chat bulk.
4. Re-walk the whole just-finished job.
5. Look for lost ideas, proof gaps, weak rules, TODOs, boundaries, stale pointers, receipts, next ropes, and blocked layers.
6. Organize the work so it is ready for later return.
7. Only then proceed.

## Trigger conditions

This rule fires when any of these happen:

- user says next after a completed save;
- direction changes;
- a proof chain closes;
- a tool trial passes or fails cleanly;
- fog lifts and a better shape appears;
- suit load becomes noisy;
- a new major lane is about to start;
- a worker/task appears finished.

## Lifted Fog Re-Walk

When fog lifts, the system can see things that were hidden during close work.

After the narrow job closes, re-walk the whole local field:

- what did we prove;
- what failed;
- what was repaired;
- what is still blocked;
- what belongs in suit;
- what belongs in TODO/parking;
- what needs lock/save;
- what must not start yet.

## Suit refresh contract

The suit must keep:

- current repo position;
- proven layers;
- active blocked layers;
- current next rope;
- dangerous non-starts;
- live behavior rules needed for the next lane.

The suit must not keep:

- repeated terminal logs;
- old command blocks;
- raw chat bloat;
- stale candidate scripts;
- superseded claims;
- proof summaries without receipt/hash.
