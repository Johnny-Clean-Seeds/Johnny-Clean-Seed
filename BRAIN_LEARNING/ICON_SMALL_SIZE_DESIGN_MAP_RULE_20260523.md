# Icon Small-Size Design Map Rule

Status: SUPPORT RULE / CREATIVE WORK REPAIR  
Date: 2026-05-23  
Scope: desktop icons, folder icons, small UI marks, visual design tasks

## Problem

The icon work failed because generation happened before a design map was built and used.

The failure was not only style. It was also task geometry:
- too much empty side space;
- actual object too small inside the canvas;
- fine detail unreadable at desktop-icon size;
- cartoon plant reading;
- ambiguous anatomy reading;
- side ornaments that looked useful close-up but collapsed at icon size.

## Rule

Before generating or editing an icon or small UI mark, build a small live design map in working context and use it as the next-action filter.

Required map fields:
- active object;
- use-case and final size;
- silhouette;
- canvas-fill rule;
- negative-space risks;
- style lane;
- blocked readings;
- required distinguishing features;
- proof test.

## Current icon lesson

For a Clean Seed / Johnny Appleseed / data-kernel icon:
- large seed/kernel must fill the canvas;
- avoid wasted transparent margins;
- metal/data/futuristic read, not cartoon seed;
- blue kernel / data energy;
- top open;
- side cracks;
- only a small emergence reaches out;
- not a normal plant;
- no leafy cute sprout;
- avoid vertical slit/body-part shapes;
- proof at 32px and 48px before calling it good.

## Short form

Small icon first.

Art second.

Silhouette, canvas fill, and negative-space safety are proof requirements.
