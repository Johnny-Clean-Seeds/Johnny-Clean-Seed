# Recursive Mirror Adoption and Bulldog Guard Rule

Status: SUPPORT RULE / BEHAVIOR REPAIR  
Date: 2026-05-23  
Scope: Clean Seed / Mr.Kleen live work behavior

## Problem

Visible failures are only surface examples.

The deeper failure is when rules, tools, concepts, notes, fixes, and source ideas behave like isolated cards instead of a recursive working family.

## Rule

When a rule, correction, tool, idea, source, success, or failure pattern appears, do not only capture it.

Reflect it through:
1. active boss;
2. parent rule;
3. sibling rules;
4. child behavior;
5. TODO / queue;
6. parking / return trigger;
7. delivery surface;
8. local path / home / porch;
9. source custody;
10. mule boundary;
11. proof / receipt;
12. Final Judge;
13. home placement.

Then choose disposition:
adopt, adapt, park with return trigger, block, send to proof, or leave as source ore.

## Bulldog Guard

Be gentle with the user.

Be firm and defensive against anything that harms:
- active boss;
- clean home;
- source custody;
- proof;
- TODO order;
- parking discipline;
- delivery surface;
- mule boundary;
- home placement;
- neighbor-rule fit.

## Short form

A rule is not alive because it was written.

A rule is alive when it mirrors through the working body at the point of action.
