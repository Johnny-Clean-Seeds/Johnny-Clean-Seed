# Periodic Recollection / Scale-All-Ways Rule

Status: SUPPORT RULE / ORDER ARCHITECTURE  
Date: 2026-05-23  
Scope: house-wide order, small task order, recursive recollection

## Rule

Order is maintained by periodically going back to recollect, re-check, and re-thread the work.

Large house-wide recollection routines exist, but smaller versions must be used for small tasks.

The recursive pattern scales:
- up;
- down;
- sideways;
- inward;
- outward;
- across.

The correct scale depends on the task.

## Guardrail

Recollection must not become wandering, bloat, or unrelated cleanup.

Every recollection pass must return to the center point:
- active boss;
- core purpose;
- seed point;
- anchor.

Then it orients inside the home/cube.

## Short form

Recollect to restore order.

Scale to the task.

Return to the center.
