# Note Burning Deadweight Release Rule

Date: 2026-05-24

Status: active support rule / note lifecycle checkpoint.

## Rule

Burning a note means releasing active weight after the note receives clean disposition.

Burning does not mean deleting source truth. It means the note stops riding in the hand once its job
has been converted into the right house object.

## Trigger

Use this rule when:

- a note, source scrap, dropped file, mule return, proof fragment, or old packet keeps pulling active
  attention after its job is mostly known;
- a review list grows faster than it closes;
- a note is loud because it is vivid, old, repeated, or emotionally forceful rather than because it
  changes the next move;
- the house has a better home for the idea than active memory;
- the assistant is tempted to keep every good sentence in the live suit.

## Burn Dispositions

Every note that reaches Level 2 or higher must end in one of these dispositions:

| Disposition | Meaning | Required proof |
|---|---|---|
| Final rule | The note became durable behavior. | Rule path, trigger, action, close condition. |
| Checklist or template | The note became a reusable work surface. | Template path and use trigger. |
| Evidence ledger | The note is proof, not behavior. | Source path, hash or how-found proof, and claim limit. |
| Parked item | Useful later, not active now. | Parking lane and return trigger. |
| Rejected-as-active | Clean idea, bad active fit now. | Reason and blocked move. |
| Blocked item | Cannot safely move. | Missing proof, authority issue, privacy issue, or destructive risk. |
| Receipt line | The note only needed to prove closeout. | Receipt path and quiet-down condition. |
| Source custody | Raw source must be preserved. | Custody path and pointer to any clean extraction. |

## Burn Gate

Before burning, ask:

1. What did this note help reveal?
2. What did it prove?
3. What did it not prove?
4. Which object now carries the useful part?
5. What source or root map can trace back to it?
6. What would make it return?
7. What active move is now blocked, allowed, parked, or complete?

If those answers are missing, the note is not ready to burn. Lower its loudness or park it with a
return trigger.

## Bad Idea Filter

The user may introduce raw ideas, pressure, metaphors, source files, corrections, and tests. Those
are important signals, but they are not automatic doctrine.

A note cannot be adopted only because it sounds clean. It must pass:

- source or how-found proof;
- active-task fit;
- neighbor-rule fit;
- authority boundary;
- quiet-down or return trigger;
- final destination.

Ideas that fail fit are not trash. They become parked, rejected-as-active, blocked, or source
custody.

## Clean Burn Pattern

`note -> source/root trace -> disposition -> placed object -> receipt or pointer -> quiet`

## Dirty Burn Pattern

- Deleting source because the summary sounds better.
- Keeping raw scratch open after a final rule already carries the job.
- Treating a parking note as active work.
- Letting old notes grow louder with age alone.
- Calling a note burned when no root trace remains.
