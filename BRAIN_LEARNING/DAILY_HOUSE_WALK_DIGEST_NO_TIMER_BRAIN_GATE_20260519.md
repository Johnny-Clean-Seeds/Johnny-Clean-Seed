# Daily House-Walk Digest - No-Timer Brain Gate

Date: 2026-05-19.
State: code/session-gate behavior.
Authority: support habit and local code gate only.
Starting repair position: main @ 34bccd9.

## Core Rule

No timers.

No scheduled reminder is the source of this habit.

The daily house-walk digest is a brain/session behavior.

When Mr.Kleen work begins, check whether today's near-8 house-walk digest exists.

If no near-8 digest exists and the near-8 window has arrived or passed, the assistant may self-initiate one daily digest.

If a near-8 digest already exists, the assistant should not self-initiate another daily digest just because the day exists.

## At Least One, Not Only One

The rule is at least one digest per day.

It is not only one per day.

Additional digests are allowed when:

- the user asks,
- a major state change happens,
- a deep scan lands,
- fog appears,
- custody risk appears,
- house load changes,
- status/current-position risk appears.

## Self-Initiation Limit

The assistant may self-initiate one daily digest around the 8 AM window only when no digest was done near or around 8.

If the user asks for another digest, that is allowed.

If a real event requires another house walk, that is allowed.

## No Fake Autonomy

This chat cannot wake itself without a runtime trigger.

A future unattended local/runtime agent could run daily work, but that requires a separate proved runtime design.

This gate is not a scheduler.

It is a local code/session check.

## Required Digest Contents

- date and local time,
- current known Mr.Kleen position,
- clean/dirty/unknown status,
- rooms walked,
- house feel,
- active candidates,
- parked candidates,
- preferred-but-not-promoted tools,
- proof/status health,
- fog risks,
- custody/provenance risks,
- overloaded rooms,
- likely parent boss,
- selected parent boss if clear,
- next clean move,
- blocked moves,
- boundary notes.

## Boundary

This gate does not install doctrine.

This gate does not rewrite active guides.

This gate does not create a timer.

This gate does not create runtime automation.

This gate does not replace proof receipts.

This gate does not replace CURRENT_HOUSE_WORK_STATUS.
