## Rule

When an error exposes a missing companion mechanic, boundary leak, save-route gap, stale assumption, or false certainty, the exposure triggers POWERPLAY / CRIME SCENE.

## Required behavior

1. Stop the old motion.
2. Preserve the failure evidence.
3. Name the exposed gap without pretending the whole concept failed.
4. Separate concept correctness from route/body failure.
5. Identify the violated or missing rule/mechanic.
6. Repair the route in the smallest safe way.
7. Resume only after proof.

## Current example

The User-Said-First Crosscheck House-Rope rule concept was sound. V1 failed because the save script attempted normal Git staging against ignored Work Shed paths. The exposed missing companion was the lane-locked ignored-path save exception: exact-file force-add, manifest/hash, and staged-set verification.

## Boundary

This rule does not authorize broad cleanup, deletion, folder force-add, doctrine promotion, automation, watcher, implementation, full batch, or Whirlpool. It is a repair lens and proof discipline.
