# 25% Rule — Same Process + Smoke Break Cycle

Date: 2026-05-30
Status: BRAIN LEARNING / PROCESS RULE / NOT DOCTRINE
WorkKey: LIVING-SYSTEM-FRONT-DOOR-BRIDGE-TUNNEL-20260530-V1-4

## Correction

The 25% rule is not four different passes.

It is the same ordered job repeated at each wave, using the prior wave output as the new input. Only depth, cleanliness, and completeness increase.

## Required cycle

1. Run the same ordered job.
2. Report the checkpoint line:
   - 25% complete — moving on.
   - 50% complete — moving on.
   - 75% complete — moving on.
   - 100% complete — package ready.
3. Take a smoke-break reset.
4. Drop old run momentum.
5. Start the same ordered job again with the prior output as input.

## Same ordered job

1. Front-door URL/current-ledger check.
2. Selected toolbelt/toolbox/workbench/backpack check.
3. Methods on/off and combined/not-combined check.
4. Three-time / three-zoom ledger walk.
5. Clean findings list.
6. Search/source research on the current issue and felt problems.
7. Creative Room widening.
8. Return to house.
9. Apply/review against what was already built.
10. Checkpoint verdict and carry line.

## Smoke-break reset means

- Release the prior wave's phrasing.
- Keep the proven output.
- Drop stale momentum.
- Re-enter from the front door.
- Use the prior output only as input, not as authority by itself.
- Keep blocked lanes blocked.

## Boundary

This process rule does not authorize broad refactor, delete, move, watcher, automation, Whirlpool, doctrine promotion, ACTIVE_GUIDES rewrite, CURRENT_TRUTH_INDEX rewrite, universal mapper, graph database, or whole-house crawl.
