# End Of Job Report Options Next Direction Note

Status: USER CLOSEOUT PREFERENCE / SUPPORT NOTE / NOT DOCTRINE
Date: 2026-05-24

## User Preference

At the end of jobs, give a full report summary and a good next direction.

Include options when useful.

Always say what I think is best and why.

After every review, include a few clean direction ideas or next steps, then name the one I think is
best and why.

## Clean Closeout Shape

Use this shape when finishing work:

1. Summary of what changed or what was found.
2. Artifacts or file paths created/changed.
3. Verification or limits.
4. Options for the next direction if there is more than one clean path.
5. Best next move and why.

For review work, do not end with only a verdict. End with direction.

## Boundary

This note is not doctrine and does not rewrite active guides.

It is a standing user preference for assistant closeout behavior.

If a task is tiny, keep the closeout compact while still naming the next clean direction.
