# Local Source and Transcript Intake Revision Pass Rule

Status: SUPPORT PASS / LOCAL SOURCE CUSTODY / NOT DOCTRINE
Date: 2026-05-24

## Purpose

Keep bulky source material, transcript text, character/person source flavor, and rough evidence in local custody while still letting useful logic enter the house in clean house language.

The source is not blocked. The source is also not automatically Git content.

## Default Custody

New source drops default to local-only evidence first.

Git-facing files should receive only distilled, source-safe house logic when that logic passes the route:

`source ore -> revise pass -> all-angles pass -> Gate Inventory / Coverage -> Evidence / History -> Proven-Better Adoption -> Parking/TODO or support rule`

## Revise Pass

Before any source idea crosses into the build, ask:

1. What is the usable house concept?
2. What is only source flavor?
3. What object type is this: gate, checkpoint, tool, card, ledger, parking item, receipt, source ore, or doctrine candidate?
4. Does a current rule already carry this job?
5. Is the clean output smaller as a checkpoint, card, or TODO?
6. What proof or repeat signal would justify stronger status later?

## Transcript Handling

Transcript files and raw excerpts stay local unless the user explicitly asks for a public or Git-facing source artifact.

Allowed Git-facing output:

- neutral principle;
- distilled rule;
- local pointer or receipt that does not embed transcript content;
- TODO or parking note that names the local custody lane without quoting source text.

Blocked Git-facing output by default:

- raw transcript text;
- bulky source evidence;
- named source flavor that is not needed for the house rule;
- long quotations;
- source packages added only because they were available.

## Same-Problem Application

If the pass finds a better rule, apply it to the active issue once, then re-run the same issue through the changed route before moving on.

Do not bundle unrelated source fixes together unless proof shows the bundle is cleaner.

## Boundary

- Not doctrine.
- Not public authority.
- No ACTIVE_GUIDES rewrite.
- No CURRENT_TRUTH_INDEX rewrite.
- No Git source dump.
- No broad transcript import.
- No pretending local-only means hidden from the assistant.

## Proof

Proof is a receipt or ledger showing:

- source material stayed local;
- the extracted rule was neutral and smaller than the source;
- no raw source text was added to Git-facing support files;
- unresolved source ideas were parked with a return trigger.
