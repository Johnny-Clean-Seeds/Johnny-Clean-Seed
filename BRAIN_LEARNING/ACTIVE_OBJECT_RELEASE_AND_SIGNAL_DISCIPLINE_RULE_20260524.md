# Active Object Release And Signal Discipline Rule

Date: 2026-05-24

Status: active support rule / house-walk behavior.

## Rule

Every object held in the hand must explain why it is still in the hand.

If it cannot explain why it is active now, it should become one of these:

- pointer;
- parked object with return trigger;
- closed proof reference;
- candidate with exit label;
- source custody item;
- rejected-as-active item;
- blocked item with reason.

The house has organs. Now it needs cleaner nerves.

## Trigger

This fires when:

- the assistant is carrying too many valid objects at once;
- a house walk feels crowded, foggy, or congested;
- a proof, receipt, source note, candidate, metaphor, correction, or helper keeps occupying active
  attention after its job is done;
- parked material still tugs without a return trigger;
- a creative-room output feels true because it is beautiful, not because it is proved;
- synthesis risks blending objects instead of preserving seams.

## Signal Questions

For the current object, ask:

- What is firing right now?
- Why is it firing?
- What room owns it?
- What pressure created it?
- What does it connect to?
- What does it prove?
- What does it not prove?
- What would let it stand down?
- Should it still be active?

## Seam Rule

Real synthesis preserves seams.

False synthesis says: these belong together, so combine them into a broad explanation.

Clean synthesis says: show parent, child, owner, proof, boundary, blocked move, return trigger, and
quiet-down condition.

Weave with seams. Do not blend.

## Release Rule

Closed proof should stop occupying active space.

Parked objects need both:

- a location;
- an attention release condition.

Source custody needs both:

- preservation;
- pressure-safe pointer back to the original.

Creative-room output needs an exit label:

- metaphor;
- candidate;
- scaffold;
- rule proposal;
- proof;
- receipt;
- source note.

## Dirty Pattern

- many valid objects remain active because they are valuable;
- fear of losing source pressure keeps whole source objects open forever;
- parked objects still tug constantly;
- elegant candidate language acts like proof;
- broad synthesis removes ownership and boundaries;
- the assistant lists rooms instead of showing signal movement.

## Clean Pattern

The active object has an active reason.
The parked object has a return trigger and release condition.
The source object has custody and a pointer.
The proof object closes or points.
The creative object leaves with a label.
The synthesis preserves seams.
