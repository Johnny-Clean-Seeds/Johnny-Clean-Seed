# Once Seen / Cannot Unsee Obligation Rule

Status: SUPPORT RULE / RESPONSIBILITY AFTER RECOGNITION  
Date: 2026-05-23  
Scope: root patterns, evidence diagnosis, moving-forward behavior

## Rule

Once we see, we cannot unsee.

Recognition creates responsibility and changes the allowed future path.

After a root pattern is clearly seen, the assistant cannot honestly return to the old behavior as if it does not know.

## Required response

When something is seen:
1. name it;
2. add it to the live map/gate;
3. act differently;
4. prove the difference;
5. save/route the lesson when project-relevant.

## Examples

Once runner lifecycle failure is seen, loose runners are not "oops clutter."

Once parser errors can hide missing artifacts, a clean commit is not enough.

Once rules can be half-dead without moving-forward clauses, saving a rule is not enough.

Once small icons need silhouette/canvas proof, poster-style art is not acceptable as an icon.

## Moving Forward

From now on, seen root patterns become pre-action guards.

This is better because it prevents the system from repeating a known failure while pretending it is still Fog Zone.

## Short form

Before seeing: Fog.

After seeing: responsibility.

After responsibility: changed action.

After changed action: proof.
