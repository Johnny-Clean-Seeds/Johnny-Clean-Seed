## Core rule

When an issue appears, do not stop at the visible symptom. Drop one layer and ask whether the issue is the upper object or a lower route, tool, script, binding, environment, custody, or proof failure.

## Required split

- Upper object failure: the thing being judged is actually wrong.
- Lower object failure: the route, helper, binding, shell, path, report writer, collection shape, git state, source custody, or proof chain failed before the upper object could be judged.

Repair the smallest proven lower issue first. Do not rewrite or condemn the upper object until the lower layer has been tested.

## Active triggers

- parser pass but runtime fail
- PowerShell binding errors
- empty collection or blank string report writer failures
- one-object versus many-object lookup failures
- missing path, source, custody, or receipt failures
- git ignored-path add failures
- target-not-run or run nonzero
- dirty repo before save
- no-op commits or skipped-target commits
- receipt/report mismatch
- helper output treated as authority
- proof/report movement treated as proof

## Proof rule

Parser pass is not proof. Runtime and rerun proof matter. A clean report is not proof unless it joins object path, receipt, purpose, route, and boundary.
