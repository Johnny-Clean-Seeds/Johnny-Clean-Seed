# Signal Completion Controlled Relaxation Rule

Date: 2026-05-24

Status: active support rule / signal stand-down checkpoint.

## Rule

Signals should fire, complete, and stand down.

The house does not need to forget. It needs to stop holding readiness after the active risk is gone.

## Trigger

Use this rule when:

- many valid rooms keep tugging at once;
- an answer tries to satisfy every room simultaneously;
- parked material still says "I might matter later" instead of "I return when X";
- a proof, source, creative, parking, correction, or final judge signal remains active after its job;
- the house feels like low-level tension rather than active danger;
- a review finds unresolved state instead of missing rooms.

## Confidence Identity

Every active or parked object should be able to say:

- How sure are we this belongs here?
- How sure are we this should remain active?
- How sure are we this can safely sleep?
- What would wake it?
- What proof, blocker, or return trigger controls that confidence?

Confidence identity is not certainty theater. Use PASS, PASS WITH GUARDRAILS, PARTIAL, UNKNOWN,
PARKED, REJECTED-AS-ACTIVE, or BLOCKED when needed.

## Stand-Down Test

Before close, ask:

1. Which signal fired?
2. What job did it perform?
3. What proof or placement completed the job?
4. Should it remain active, become a pointer, park with trigger, or close?
5. What exact event wakes it again?

If a signal cannot answer, it stays at tug/checkpoint level until placed.

## Clean Pattern

`signal fires -> job named -> proof/placement done -> confidence identity named -> stand down or return trigger`

## Dirty Pattern

- Every valid room keeps whispering "keep me active a little longer."
- Parking means location without attention release.
- Proof remains loud after proof is done.
- Source custody keeps whole source open when a pressure-safe pointer is enough.
- Creative output survives beyond the job without exit label.
- The assistant writes long answers because it is unconsciously satisfying every room at once.
