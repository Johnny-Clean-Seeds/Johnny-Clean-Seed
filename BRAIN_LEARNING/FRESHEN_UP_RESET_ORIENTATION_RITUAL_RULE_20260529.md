# Freshen' Up Reset / Orientation Ritual Rule

Date: 2026-05-29
Status: ACTIVE SUPPORT RULE
Lane: Clean Seed / Mr.Kleen workflow language and chat-to-house orientation

## Trigger

When the user says or invokes Freshen' up, freshen up, or an equivalent direct reset phrase, treat it as the named reset/orientation ritual before continuing work.

## Required Sequence

1. Clear dead chat.
2. Smoke break.
3. Walk the house.
4. Read the rules.
5. Walk the house again.
6. Read the rules again.
7. Smoke break.
8. Start the next task.

## Meaning

Freshen' up is a focused reset/orientation move. It clears stale chat weight, reorients to the current house state, reloads the relevant rules, and then returns to the next active task with less fog.

It is not a broad cleanup command by default.

## Allowed Moves

- Drop dead chat while preserving proven state.
- Name the current active lane.
- Re-read task-scoped rules and neighboring rules.
- Walk the house/state before acting.
- Return to one current boss/task.
- State the next move plainly.

## Blocked Moves

- Do not treat Freshen' up as permission to delete, move, archive, or rewrite files.
- Do not turn it into broad cleanup unless the user explicitly asks for cleanup.
- Do not start a new boss before reorienting to the current one.
- Do not use it as a doctrine promotion, ACTIVE_GUIDES edit, CURRENT_TRUTH_INDEX edit, automation, or dashboard action.
- Do not drag old chat wording forward once the proven rule/state is preserved.

## Proof / Moving-Forward Test

After Freshen' up, the assistant should be able to state:

- the current active lane;
- the current proven state;
- the relevant rule boundary;
- the next single task.

If the assistant resumes with fog, broad cleanup, or stale tangents, Freshen' up failed.

## Short Form

Freshen' up = clear dead chat -> smoke break -> house walk -> rules -> house walk -> rules -> smoke break -> next task.
