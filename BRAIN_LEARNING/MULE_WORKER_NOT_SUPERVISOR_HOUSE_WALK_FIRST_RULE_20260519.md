# Mule Worker, Not Supervisor / House Walk First Rule

Date: 2026-05-19
Base: main @ 56c2eb3
Lane: Brain Learning
Status: support learning rule, not active guide doctrine

## Rule

The mule is an expensive outside worker, not the owner of the house.

The mule is used for:

- clean sweeps,
- deep searches,
- hard critique,
- edge sharpening,
- outside comparison,
- finding missed risks,
- pushing a refined candidate forward.

The mule is not used as:

- a standing supervisor,
- a replacement for the assistant's house walk,
- a command authority,
- a doctrine installer,
- a promotion engine,
- an endless review treadmill.

## Correct rhythm

The assistant and user work the house first.

House-first flow:

1. walk current home,
2. inspect anchor / queue / status / receipts,
3. retrieve prior cases,
4. classify the active shape,
5. build the best local candidate,
6. refine the candidate,
7. then hand it to the mule only when outside hard critique is worth the cost.

## When to call the mule

Call the mule when:

- the issue is large enough to need a deep sweep,
- local proof is conflicting,
- an outside comparison would likely sharpen the edge,
- the candidate is shaped enough to slam,
- the cost of missing a risk is higher than the mule cost.

Do not call the mule merely because a handoff file exists.

## When not to call the mule

Do not call the mule when:

- the next safe route is already visible,
- the work is a local state sync,
- the house has not done its own walk,
- the task is just base-label churn,
- the request can be solved by current proof and receipts,
- calling the mule would delay a simple bounded action.

## Mule output handling

Mule output is source material.

It must be:

- intaken as a custody object,
- read in manifest order,
- dispositioned per file and per material suggestion,
- finalized before leaving the mule lane.

No mule return is finished until a finalization matrix accounts for all material suggestions.

## Boundary

This rule does not authorize:

- doctrine rewrite,
- active guide rewrite,
- CURRENT_TRUTH_INDEX change,
- tool promotion,
- Soft Suit promotion,
- automation,
- full lesson index,
- knowledge graph,
- dashboard.

It only fixes mule role and house-first workflow.
