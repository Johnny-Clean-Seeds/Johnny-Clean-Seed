# Save Lane Split — Mule Return vs Assistant Progress Rule

Status: SUPPORT RULE / SAVE DISCIPLINE  
Date: 2026-05-23  
Scope: lock/save packets, mule returns, assistant/user progress

## Rule

Do not mix mule return capture with assistant/user progress saves unless the content is strictly the same work product.

Mule return, assistant-created rules, user corrections, research roadmaps, local cleanup, and status updates need distinct save packets or clearly separated sections with explicit lane boundaries.

## Mule return save captures only

- what mule did;
- report path;
- receipt path;
- verdict;
- blockers;
- commit proof if any;
- next handoff.

## Assistant/user progress save captures only

- user corrections;
- assistant behavior rules;
- research roadmaps;
- working methods;
- status updates created by user/assistant work.

If they touch, use a reference.

Do not blend the bodies.

## Short form

Mule helps the house.

Mule is not the house.

Mule return is not assistant progress unless explicitly adopted after review.
