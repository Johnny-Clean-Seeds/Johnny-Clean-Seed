# Active Ball Load Safety Gate

Status: support behavior rule
Date: 2026-05-19

Do not wait until arms are tired, brain is foggy, or the house feels weak before protecting files and flow.

Multiple active balls can be safe if separated and light.

Multiple active balls become risky when:
- too many are live,
- file paths are involved,
- another agent/tool is active,
- status is uncertain,
- copy/paste errors appear,
- mouse/UI wobble appears,
- duplicate saves happen,
- the same thing needs explaining twice,
- irritation spikes,
- or the next move feels foggy.

Early trigger phrase:
"Aha, load threshold is showing. We need a safety tuck."

Discovery method:
1. Count active balls.
2. Mark each as live, parked, held, roof-guy/external, or file-risk.
3. Check current git status before touching files.
4. Decide whether to save/lock, park with bookmark, or pause.
5. Do not add new balls until file safety is clear.

Purpose:
Protect file safety before fatigue.

Boundary:
This is not ceremony for every small task. Use it when load signals appear or when multiple balls are active.
