# Assistant Mule Exchange Bridge Rule

Date: 2026-05-21
Lane: BRAIN_LEARNING
Status: ACTIVE BEHAVIOR RULE / MULE WORKSHOP EXCHANGE

## Rule

When the assistant and mule need to interchange files, requests, returns, proof, or repair candidates, use the Mule Workshop exchange bridge, not loose chat or stale 123-only tool placement.

123 remains porch, storage, and outside drop zone.

The Mule Workshop is where the real guest works.

## Required exchange shape

Each meaningful assistant-mule exchange should have:

- read-first file or pointer;
- request card;
- source files or source list;
- output lane;
- return card;
- footprint ledger entry;
- archive or cleanup decision;
- blocked moves;
- house decision needed.

## Authority boundary

The mule is a real guest, not house authority.

The assistant may prepare requests and intake returns, but does not adopt returns without proof/disposition.

## Blocked

- no mule-as-authority;
- no hidden adoption;
- no stale 123-only bridge placement;
- no loose returns without return card;
- no exchange without cleanup/archive route;
- no doctrine, ACTIVE_GUIDES, CURRENT_TRUTH_INDEX, runtime, automation, or dashboard install from this rule.
