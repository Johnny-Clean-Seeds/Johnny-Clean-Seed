# New-Task Map Instantiation Rule

Status: SUPPORT RULE / RECURSIVE WORK BEHAVIOR  
Date: 2026-05-23  
Scope: all new, unfamiliar, newly shaped, or materially changed tasks

## Rule

Whenever a task is new, unfamiliar, newly shaped, or materially different from the previous task, create a small live working map before acting.

The map is not decorative. It must filter the next action.

## Required map fields

- center point / active boss;
- purpose;
- object;
- use-case;
- lane;
- boundaries;
- source/custody if relevant;
- neighbors;
- blocked readings / failure modes;
- delivery surface;
- proof test;
- Final Judge;
- return/save trigger.

## Recursive method

New task -> make map -> link rules/methods/procedures -> act through map -> test -> update map/house if useful.

## Short form

Map first.

Work second.

The map must do work for the work.
