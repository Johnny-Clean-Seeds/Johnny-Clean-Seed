# Chat Source URL / Ledger / Map / Key Compression Rule

Date: 2026-05-30
Status: BRAIN LEARNING / CHAT LOAD REDUCTION / NOT DOCTRINE
WorkKey: README-FRONT-DOOR-CHAT-SOURCE-20260530-V1-5

## Rule

For Clean Seed / Mr.Kleen chat source loading, the preferred live shape is:

URL -> LEDGER -> MAP -> KEY -> only needed tunnel.

## Meaning

URL gets the worker to the front door.

LEDGER tells what is current, blocked, proved, and next.

MAP tells how rooms/files connect.

KEY explains labels, tunnel types, load shelves, and relation types.

Proof receipts, stale package copies, source ore, and old checkpoint dumps stay out of live chat unless the active question requires them.

## Boundary

This is a load-reduction rule. It does not replace proof, source custody, ACTIVE_GUIDES, CURRENT_TRUTH_INDEX, ACTIVE_ANCHOR, or user command.
