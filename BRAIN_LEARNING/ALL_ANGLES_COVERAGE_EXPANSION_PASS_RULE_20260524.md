# All-Angles Coverage Expansion Pass Rule

Status: SUPPORT PASS / BROADEN-THEN-CONDENSE / NOT DOCTRINE
Date: 2026-05-24

## Purpose

Broaden a candidate across the relevant choice family before the house decides what it is.

This is not a new full gate by default. It is a support pass under Creative / Growth and Problem Solving.

It makes the candidate wide enough to inspect, then condenses it so Gate Inventory can type it cleanly.

## Existing Roots

This pass adapts existing house behavior:

- Broad-Bite Rule Machine: broaden first, then condense.
- Problem Solving 101 / Question Before Broadening: question the object before broadening it.
- Gate Inventory / Coverage: type the candidate before evidence and adoption.

## Order

Use this inside the current true seam:

`Creative/Growth candidate -> All-Angles Expansion Pass -> Gate Inventory/Coverage -> Evidence/History -> Proven-Better Adoption -> Parking/TODO -> Sequencer`

## Expansion Angles

Before condensing, check:

- actor: who or what performs the action;
- action: what is done;
- omission: what happens if the action is not done;
- timing: early, late, too long, too short;
- scope: too narrow, too broad, wrong lane;
- authority: who may decide or promote;
- proof: what evidence is required;
- boundary: what must not be touched;
- object type: gate, checkpoint, tool, card, ledger, parking item, receipt, script, doctrine candidate;
- failure route: how this could become muddy drift;
- return-home: where unresolved material goes;
- retirement path: how to park, demote, or remove later if it fails.

## Condense Rule

After expansion, condense to the smallest clean support object that can carry the job.

Possible outputs:

- KEEP AS IS;
- RENAME;
- SPLIT;
- MERGE;
- ADAPT AS CHECKPOINT;
- ADAPT AS TOOL/CARD/LEDGER;
- PARK / WATCH AGAIN;
- REJECT;
- PROOF FIRST.

## Boundary

- Not doctrine.
- Not public authority.
- Not a replacement for Gate Inventory.
- Not a reason to add everything.
- Not a broad crawl.
- Not a proof substitute.
- No ACTIVE_GUIDES rewrite.
- No CURRENT_TRUTH_INDEX rewrite.

## Proof

Proof is a live candidate where the expansion pass prevents a too-narrow fix, a fake gate, a hidden duplicate, or a missing return trigger.
