# Batch Safe Bin / Trial Suit Rule

Date: 2026-05-19.
State: active batch-intake behavior rule.
Starting base: main @ 3cab2bd.

## Rule

When many files, ideas, critiques, reports, or helper artifacts arrive as a connected batch, do not judge them one file at a time by default.

Put the whole connected batch into a safe bin / trial suit first.

The batch should be felt and tested as a working set against the old brain before promotion.

## Why

Some items only make sense with their companion files, labels, receipts, tests, or order.

One-at-a-time judgment can create fake ghosts:
- a good file looks bad because its support file is missing,
- a checker looks wrong because its wording bridge is not installed,
- a helper looks useless because its companion route is absent,
- a rule appears to collide because dependency order was ignored.

## Method

1. Collect the whole batch.
2. Preserve source and custody.
3. Classify pieces without promoting them.
4. Test the batch together against old brain context.
5. Check ordering dependencies.
6. Detect collisions caused by bad order, bad labels, or missing companions.
7. Install the biggest clean chunk that proves fit.
8. Isolate only risky, private, executable, unclear, or collision-causing items.
9. Park or reject with reason only after set-fit has been tested.

## Boundary

Do not blindly add all.
Do not blindly reject all.
Do not execute all.
Do not default to one-file-at-a-time drag when the batch is connected.
Do not let a weird add order make good material look bad.

## Current Proof Case

The ignored-file/checker batch showed why this rule is needed.

The set contained checker scripts, proof/history items, backup holds, false-positive labels, and checker collisions. The right move was not blind add-all and not one-file-at-a-time judgment. The right move was batch parking, text inspection, collision detection, relabeling, and then clean chunk decisions.

## Next Use

Use this rule before the next mule batch, outside-worker batch, large file drop, or connected idea pile.
