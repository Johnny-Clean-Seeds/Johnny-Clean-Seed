# World Capture Helpers Adoption Learning Rule

## Purpose

Teach the assistant to use fitting ideas from the world as capture helpers without over-promoting them.

## Core rule

The world can provide useful capture patterns.

When an outside pattern fits Clean Seed / Clean Milk, the assistant may adopt it as a helper only through the model flow:

capture
→ name
→ lane
→ status
→ proof need
→ Soft Suit test when active
→ Hard Suit only after proof
→ controlled active-file dump only if needed

## Adopted helper set

The current adopted capture helper set is:

1. Raw Capture Lane
2. Capture Abundance / Processing Scarcity
3. Affinity Grouping Gate
4. Boss Severity Matrix
5. Context Tag Requirement
6. Hypothesis Tag
7. Capture Intelligence Cycle
8. No Storage Without Retrieval
9. Soft Suit = Do/Check; Hard Suit = Act/Standardize
10. Stabilize Before Repair

## Do not lose ideas

If an idea does not fit the active task yet, store it.

Use:

- source ore
- parking
- idea map
- toolbox
- window
- proof need
- word key
- Soft Suit candidate
- support link

Do not delete useful ideas just because they are not ready.

## Rest storage rule

Material from a capture hunt that is not adopted as a 100 percent fitting helper must be stored as source ore or parking with a reason.

Not adopted does not mean bad.

It may be:

- too broad
- not tested
- missing a companion
- missing a clean lane
- useful later
- only a source pattern
- not active pack weight

## Respect Thy Neighbor

Before turning a world pattern into a model helper, check:

- what lane it touches
- what rule already does the job
- whether it duplicates or clarifies
- whether it needs a companion
- whether it should stay source ore
- what proof would show fit

## Close condition

PASS requires:

- adopted helpers are listed
- non-active material is stored
- no helper bypasses suit/proof gates
- Soft Suit stays within 3 active bosses
- captured ideas remain retrievable
