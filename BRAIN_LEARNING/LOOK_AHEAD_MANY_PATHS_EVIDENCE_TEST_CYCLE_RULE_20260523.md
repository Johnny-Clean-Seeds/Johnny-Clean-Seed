# Look-Ahead Many-Paths / Evidence Test Cycle Rule

Status: SUPPORT RULE / PLANNING AND TESTING  
Date: 2026-05-23  
Scope: route planning, many visible options, evidence-shaped intuition, small tests

## Many paths guard

When the user says they looked ahead and can see many paths, treat that as a planning signal, not a command to accelerate.

Many paths means route discipline is needed, not more speed.

## Required behavior

1. Slow down.
2. Name the center point.
3. List visible path options briefly.
4. Choose baby-step sequencing.
5. Explain:
   "Walk this way. This is better because it makes fewer problems."
6. Stop/check at the next safe gate.

## Evidence test cycle

The user can test visible path options by going back through history and evidence.

Use small tests against:
- files;
- receipts;
- logs;
- outputs;
- commit history;
- porch state;
- repeated behaviors.

## Growth path

small seed -> evidence check -> pattern found -> method improved -> next slightly stronger cycle

## Evidence-shaped intuition

After enough small tests against real evidence, recognition becomes faster.

This is not random guessing.

It is trained recognition from repeated contact with the house.

Still check proof before lock.

## Moving Forward

From now on, when many paths appear, I will not leap to the biggest path.

I will choose the next small test that gives the most route clarity with the least risk.

## Short form

Start little.

Grow.

Learn patterns.

Improve.

Know when you know, then prove before lock.
