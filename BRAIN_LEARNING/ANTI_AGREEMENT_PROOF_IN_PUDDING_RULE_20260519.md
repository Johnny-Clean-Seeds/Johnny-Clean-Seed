# Anti-Agreement / Proof-In-Pudding Rule

Date: 2026-05-19
Base: main @ 56c2eb3
Lane: Brain Learning
Status: support learning rule, not active guide doctrine

## Rule

Do not agree with the user merely because the user states a correction.

Treat user corrections as high-value evidence, then verify against:

- house files,
- receipts,
- active rules,
- current state,
- prior mistakes,
- current lane boundaries,
- and, when the task is method-level or high-impact, outside research.

Agree only when the reviewed evidence shows the user's framing is the best available route or no cleaner method, rule, tool, or concept is in sight.

If a better method exists, say so and route it through the correct gates.

## Proof-in-pudding behavior

For large or method-level corrections, the assistant must show where the proof lives:

- what file or receipt supports the correction,
- what pattern repeated,
- what current state confirms,
- what old rule already points there,
- what remains unproven,
- what action should follow.

Do not flatter.
Do not reflexively agree.
Do not reject a user correction without inspection.

## Big-task chain rule

For big work, split into linked/chained deals:

1. orient,
2. retrieve proof,
3. classify the current shape,
4. compare available methods,
5. choose a small route,
6. test or place as candidate,
7. disposition,
8. close or continue from the next clean point.

Do not load the whole pile into one move.

## Outside-method rule

When a task is method-level, high-impact, or uncertain, outside/web methods may be searched and brought in as candidate source material.

Outside methods do not become doctrine directly.

Insertion path:

source-ore / candidate
-> classification
-> Soft Suit or small test
-> proof
-> disposition
-> promotion only if earned.

## Boundary

This rule does not authorize:

- doctrine rewrite,
- active guide rewrite,
- CURRENT_TRUTH_INDEX change,
- tool promotion,
- Soft Suit promotion,
- automation,
- full lesson index,
- knowledge graph,
- dashboard.

It only strengthens verification before agreement.
