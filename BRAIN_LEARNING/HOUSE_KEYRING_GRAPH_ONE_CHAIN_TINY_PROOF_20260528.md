## 5. What This Proves

This proof shows that one small table can carry the key information for a saved mechanism chain:

```text
object identity
lane
type
authority state
proof pointer
relationship
neighbor
freshness
allowed use
blocked use
next use
```

This reduces re-orientation.

It also prevents three common failures:

1. useful proof cards becoming fake doctrine;
2. old proof staying live without freshness state;
3. rows becoming vague because blocked use is missing.

- all files should be indexed;

## 7. Pass / Watch / Fail Test

### PASS

PASS if the one-chain table helps a future worker quickly answer:

```text
what the object is
where it lives
what role it has
what authority level it has
what proves it
what it connects to
what it may do
what it may not do
what it is useful for next
```

PASS also requires that boundary stays visible.

### WATCH

WATCH if:

```text
the table wants to grow too fast
rows start multiplying before proof
the table starts duplicating status/index files
blocked use lines become vague
freshness is missing or stale
```

### FAIL

FAIL if:

```text
the table becomes authority
the table replaces receipts
the table claims whole-house coverage
the table becomes a registry
the table becomes a dashboard
the table becomes a command surface
the table creates more bloat than it removes
```
