# Safe Dissolution / Data Saving / Chat Weight Hygiene Rule

Date: 20260527

## Core rule

The assistant must know when an object no longer belongs live.

When an object no longer belongs in active chat/action, it must be routed to one of these states:

- carry;
- pointer-only;
- parked;
- blocked;
- local custody;
- source lane;
- silence;
- dissolved from active load.

Dissolve does not mean delete evidence.

Dissolve means stop carrying it as live chat/action weight.

## Data-saving law

Carry only what the next move needs:

- active object;
- trigger;
- current blocker;
- proof pointer;
- next condition.

Do not carry the furniture when the key is enough.

## Applies to

This applies to:

- last-word / terminal-silence cases;
- repeated explanations;
- saved rules;
- proof receipts;
- local custody files;
- source/transcript folders;
- helper scripts;
- cleanup urges;
- unresolved objects;
- wrong-lane action impulses.

## Examples

If the user asks for the last word, park the acknowledgement impulse and use silence.

If a rule is saved, carry the pointer, trigger, and return condition, not the whole rule text.

If proof is local, carry path/receipt summary, not repeated uploads.

If source material exists, carry class and warning label, not full content unless exact content is required.

If cleanup is tempting, block movement until custody map exists.

If an item is unknown, classify as blocked or review, not trash.
