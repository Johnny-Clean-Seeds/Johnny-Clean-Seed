# Candidate Rule Card V1 — Meta-Signal + Proceduralized Clean Move

Date: 2026-05-28
Status: candidate / not doctrine / not ACTIVE_GUIDES / not CURRENT_TRUTH_INDEX

## Candidate A — Meta-Signal Is Not Proof

A feeling of knowing, confidence, elegance, fit, urgency, or “this is it” is a signal. It may trigger inspection. It cannot close a verdict.

Required move:

signal -> represent object -> find proof surface -> judge -> verdict or downgrade

## Candidate B — Proceduralize The Clean Move

A rule is not fully alive until the clean move is practiced in the failure context.

Examples:

parser fail -> stop and inspect report
source task -> direct source read before synthesis
helper output -> compare with source before trusting
deep report -> three-pass revision before final
save route -> commit/push/HEAD==origin/status clean
dead weight -> burn after useful extraction

## Candidate C — Mechanism Over Term Count

A source helper must extract causal/mechanism chains, not only keyword hits.

## Candidate D — Auto-Link Every Pickup

When a tool, idea, rule, room, helper, source, proof surface, hash, receipt, map, or repeated pattern is picked up, link it to the active object and role.
