# Focal Core + Side Piles Work Method

Date: 2026-05-30
Status: WORK METHOD / SUPPORT RULE / NOT DOCTRINE

## Purpose

Prevent the core object from getting watered down by side research.

## Rule

Do not let side research overwrite the focal object. Sort first. Merge second. Save third.

## Method

1. Copy the focal core into view.
2. Keep the core file as the main object.
3. Run outside search and house/resource review into side piles.
4. Name every useful find.
5. Mark every pile: merge, living idea, parked, blocked, rejected-with-reason, or next action.
6. Burn dead weight only after useful pieces are copied, parked, rejected-with-reason, or merged.
7. Re-glance at the core.
8. Adjust piles.
9. Set clean sub-piles aside with names, fit, proof need, blocked moves, and return triggers.
10. Suit up important rules during the work.
11. Lock-save when shape/proof is sufficient.
12. Return to the core and merge only fitting pieces.
