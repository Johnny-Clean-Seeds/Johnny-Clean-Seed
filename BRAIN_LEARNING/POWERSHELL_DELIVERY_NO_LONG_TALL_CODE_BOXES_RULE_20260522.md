# PowerShell Delivery No Long Tall Code Boxes Rule 20260522

Status: ACTIVE BEHAVIOR RULE.

Problem fixed:
The assistant kept sending long PowerShell scripts in tall chat code boxes, creating bad copy/paste ergonomics and visual drag.

Rule:
For large PowerShell saves or multi-file scripts, do not paste the full long script body into chat by default.

Preferred delivery:
- short wide launcher
- downloadable or 123-local script packet
- one-line or compact semicolon command when practical
- only exact copy target inside a code block

Protected behavior:
This rule only controls delivery shape. It does not change source pickup from Desktop\123, Git save/push default, proof receipts, local/brain boundaries, source-safe reframe rules, or current task routing.

Blocked:
- do not use a source-routing correction as permission to change code delivery style
- do not use a code-delivery correction as permission to change source routing
- do not send giant vertical script boxes when a short launcher or script packet is practical

Short form:
Long script equals file or 123-local script body. Chat equals short wide launcher.
