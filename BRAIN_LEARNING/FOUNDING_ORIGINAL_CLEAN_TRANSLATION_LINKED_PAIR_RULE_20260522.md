# Founding Original + Clean Translation Linked Pair Rule 20260522

Status: ACTIVE BEHAVIOR RULE.

Problem fixed:
Clean translations can accidentally replace, overwrite, or outrank the raw founding idea. That corrupts source custody and makes the cleaned version look like the origin.

Rule:
When an idea, source note, user phrase, rough wording, weird concept, or source-derived concept matters, save it as a linked pair.

Pair structure:
Founding Original / Source Root
- exact wording or original note
- source path, ID, or hash when available
- custody / origin truth
- weirdness, pressure, roughness, and raw signal preserved
- not cleaned into assistant wording

linked to

Clean Concept Translation / Working Use Form
- clean model-safe translation
- lane
- purpose
- boundary
- proof status
- use condition
- blocked dirty translations
- next action or parking path

Link requirement:
The clean translation must explicitly point back to the founding original. The founding original must explicitly point forward to the clean working form when one exists.

Minimum link fields:
- founding original path / ID
- founding original hash when available
- clean translation path / ID
- lane
- boundary
- proof status
- blocked dirty translations
- return/check link

Authority relationship:
The founding original owns source truth.
The clean translation owns operational use.

Blocked:
- Do not call the founding original the old idea.
- Do not treat the clean concept as the replacement main idea.
- Do not erase raw wording because a clean version exists.
- Do not let the clean version drift away from source custody.
- Do not promote a clean translation as doctrine without proof.
- Do not remove the founding original unless it is unsafe, stale, explicitly rejected, or duplicate with no added value and the reason is recorded.

Short form:
Founding original owns source truth. Clean translation owns operational use.
