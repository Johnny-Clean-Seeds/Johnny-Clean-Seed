# Rope Cut Runner Trace To Root Rule

Date: 2026-05-24

Status: active support rule / release with trace.

## Rule

Cutting the rope means releasing unneeded active weight while preserving a runner trace back to the
root.

Do not cut by forgetting. Cut by routing.

## Trigger

Use this rule when:

- a note, packet, source, rule candidate, proof fragment, or issue no longer needs active attention;
- a list is being cleaned after a house walk;
- a source idea has been extracted into a rule, checklist, ledger, TODO, receipt, or parking note;
- a note should quiet down but still needs a path back to source truth;
- a path, root map, or relationship might be needed later to explain why the note was cut.

## Runner Trace

Before cutting, create or point to a runner trace with:

- Object id or note name.
- Old rope point.
- Source path, source turn, or how-found proof.
- Source hash when available.
- Owner room.
- Relation word: source, checkpoint, proof, neighbor, parking, blocker, promotion candidate, receipt,
  or custody.
- Linked rule or map.
- Final destination.
- What was extracted.
- What was not extracted.
- Proof or receipt path.
- Return trigger.
- Quiet-down condition.

The runner trace may live inside a packet, receipt, parking note, TODO, template, or ledger. It does
not need a new file for every tiny cut if a manifest or receipt carries the trace clearly.

## Cut Gate

Cut only if:

1. The useful part has a final destination.
2. The root/source remains findable.
3. The note's proof limit is named.
4. The return trigger is known or the note is permanently rejected-as-active.
5. No authority, privacy, destructive-action, or local-only risk remains hidden.

If any answer is missing, do not cut. Tug first.

## Clean Cut Pattern

`active note -> runner trace -> final destination -> quiet support`

## Dirty Cut Pattern

- Deleting source truth.
- Cutting because the list feels too long, before destination is known.
- Keeping a vague "remember this later" note with no return trigger.
- Making a runner trace so large that it becomes a new active burden.
- Promoting source pressure into doctrine without proof.

## Root Map Shape

Use this small map when a cut needs traceability:

`source/root -> object -> relation -> owner room -> controlling rule -> destination -> proof -> return trigger`

This is enough for most house work. Do not build a full graph system unless manual traces become too
slow or inconsistent.
