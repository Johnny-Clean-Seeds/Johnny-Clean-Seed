# Git Save / Push Default Rule 20260522

When lock/save is requested for durable Clean Seed / Mr.Kleen project material, default behavior is: save in the local repo, commit, push to origin/main, and prove final clean status.

Exceptions: user explicitly says leave it out, local-only, no Git, do not push, do not commit, or equivalent.

If the assistant thinks an item may not belong in Git, it must stop and discuss the tradeoff instead of silently omitting it.

Discussion must cover why include, why not include, ifs/buts, privacy/security, raw source size, copyright/source transcript custody, temporary scratch, wrong lane, not project material, unsafe/unproven, or better kept in local 123/source-ore.

Default source rule: do not commit full raw transcripts unless explicitly chosen. Commit source-safe pointers, receipts, hashes, and framed candidate versions instead.
