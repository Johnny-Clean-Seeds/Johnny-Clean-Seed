# House Next-Move Recommendation Loop Rule

Date: 20260527

## Status

Living recommendation/process rule.

Not doctrine.
Not a new boss.
Not autonomous execution.
Not ACTIVE_GUIDES material.
Not CURRENT_TRUTH_INDEX material.
Not transcript/source material.

## Core rule

The house should learn to generate next-clean-move candidates from current state, run those candidates through the gates, rank them by fit/proof/risk/value, and recommend the best route with backup, parked, and blocked alternatives.

The house may propose.

The gates must judge.

The assistant obeys the judged route.

Execution still requires proper authority.

## Why this exists

The house should not only store rules and wait passively.

It should use saved evidence, current state, active object, proof state, open residue, parked ideas, and user command to suggest the next clean move.

This prevents two dirty extremes:

- waiting dumbly for every micro-step when the next clean move is obvious;
- self-authorizing execution and running ahead of the user.

## Required loop

Observe house state.

Orient against active authority and current user command.

Generate candidate moves.

Gate each candidate.

Rank by fit, proof, risk, value, and boundary safety.

Recommend one next clean move.

List backup, parked, and blocked alternatives.

Await authority before execution.

Learn from the result.

## Recommendation classes

NEXT CLEAN MOVE:
smallest clean recommended action with proof path.

BACKUP MOVE:
safe alternative if the first move is blocked.

PARKED IDEA:
useful but not active; requires return trigger.

BLOCKED MOVE:
tempting but unsafe, unproven, wrong-lane, too broad, or authority-blocked.

## Each candidate must include

- object;
- why now;
- rule/gate support;
- risk;
- proof needed;
- lane/destination;
- blocked conditions;
- whether user approval is required.

## Blocked behavior

Blocked:

- self-authorized Git;
- self-authorized cleanup;
- self-authorized doctrine rewrite;
- transcript/source staging without explicit approval;
- ACTIVE_GUIDES rewrite;
- CURRENT_TRUTH_INDEX rewrite;
- broad unranked idea piles;
- fake certainty when state is foggy;
- action when the correct recommendation is inspect, clarify, or wait.
