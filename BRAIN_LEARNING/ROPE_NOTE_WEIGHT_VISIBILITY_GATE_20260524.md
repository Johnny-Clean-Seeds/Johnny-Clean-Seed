# Rope Note Weight Visibility Gate

Status: BRAIN LEARNING / SUIT SUPPORT / CHECKPOINT RULE / NOT ACTIVE DOCTRINE
Date: 2026-05-24
Source: user rope-note request, `HOUSE_WALKTHROUGH_ROPE_NOTES_V3`, live local house walk, and decoding research pass.

## Purpose

Make notes on the rope visible in proportion to need.

A note may get louder when it helps the worker keep the path, recover source truth, or prevent a foggy move. A note must quiet down when the path is clear, when its job is done, when stronger proof replaces it, or when it is only interesting.

The note never becomes louder by age, drama, title, repetition, or assistant excitement alone.

## Core Rule

Use the rope as the active path:

`task -> object -> room -> source/proof -> neighbor -> pressure -> checkpoint -> allowed move -> blocked move -> return trigger -> Final Judge`

Attach notes to the rope as visibility signals, not as authority.

A note goes primary only when the path is too foggy to see the real truth and the note is the smallest clean pointer back to source, proof, placement, or blocked movement.

When the fog clears, the note steps back into support, parking, source custody, receipt, or rejection.

## Visibility Levels

Use these levels instead of vague "important" language:

| Level | Name | When It Appears | What It May Do |
|---|---|---|---|
| 0 | Silent custody | Preserved source, no current fit. | Stay findable only. |
| 1 | Whisper | Related but not needed for current path. | Link as support or parking. |
| 2 | Visible note | Helps orient the current room. | Show in checkpoint card. |
| 3 | Checkpoint note | Fog, repeated correction, or proof gap makes it relevant now. | Must be considered before the next move. |
| 4 | Gate note | Boundary, authority, proof, source custody, or blocked move is unclear. | May pause, reroute, or require proof before action. |
| 5 | Alarm note | Continuing would create false PASS, unsafe action, destructive cleanup, or authority drift. | Stop movement until resolved, parked, or yielded. |

## Weight Inputs

Raise visibility only through clean inputs:

- Active-task fit: the note names this task, object, room, or route.
- Source/proof fit: the note points to real source, exact file, receipt, hash, or checked evidence.
- Fog pressure: the active path is unclear and the note reduces that fog.
- Repetition: the same failure appears across separate runs, not just repeated wording in one note.
- Time pressure: an unresolved checkpoint is aging toward risk, stale proof, missed save, or lost custody.
- Neighbor pressure: the note prevents a rule, room, tool, or proof lane from stepping on another.
- User correction: the user corrected this exact behavior or shape.

Lower visibility through dampers:

- No current lane.
- No proof path.
- Duplicate of a stronger note.
- Interesting but not useful now.
- Old but stale.
- Loud wording without source.
- Conflicts with active anchor, active guides, current truth, or user boundary.
- Would add active load when a pointer is enough.

## Bounded Scaling Formula

Do not use math as fake precision. Use this as a small judgment frame:

`visibility = need + proof + fog + recurrence + time-risk + neighbor-fit - dampers`

Then cap by authority:

- Raw source note caps at Level 2 unless it is the only clean source pointer during fog.
- Working note caps at Level 3 unless it exposes a proof, authority, or boundary break.
- Candidate idea caps at Level 2 until tested.
- Repeated user correction may reach Level 4 when it blocks the same live failure.
- Safety, destructive file action, authority drift, or false PASS risk may reach Level 5.

No note may outrank active proof, current user command, active anchor, active guides, or verified file state by volume alone.

## Time Rule

Time has two directions:

- Aging unresolved risk gets louder until it is resolved, parked with trigger, yielded, or blocked.
- Aging inactive source gets quieter unless a current trigger reopens it.

Old does not mean wise.
Old plus unresolved risk, repeated proof, or a current trigger may mean visible.

## Decoding Fit

Use outside decoding ideas only in clean, small form:

- Sensemaking / data-frame loop: a note is useful when it helps test or repair the current frame against observed evidence.
- Abduction: a note may offer the best current explanation, but it stays provisional until evidence supports it.
- Bayesian update: evidence can raise or lower a note's weight; confidence must move with proof, not mood.
- Belief/plausibility interval: when evidence is incomplete, hold a range instead of pretending certainty.
- Information scent: make the note easy to find by clear label, source, room, and trigger; do not make it loud by clutter.

These are support lenses, not new bosses.

## Dirty Milk

Dirty note behavior appears when:

- a note becomes doctrine because it is emotionally forceful;
- a note gets louder by time alone;
- a note hides source truth behind a polished summary;
- a note repeats already-captured material without adding proof, placement, or trigger;
- a note blocks work after its fog is gone;
- a note lets the assistant install a bad idea because it sounds clever;
- the assistant carries every good note instead of turning most of them into pointers.

## Clean Milk

Clean note behavior appears when:

- the note is attached to the rope and has a room, source, trigger, and quiet-down condition;
- the note's visibility matches the actual risk and proof;
- the note points to truth when the path is foggy;
- the note gets smaller after proof or placement;
- bad, stale, unsafe, and bloaty ideas are parked, quarantined, rejected as active, or left as source;
- the assistant works better at the point of use.

## Verdicts

- PASS: the note scaled to the right level, changed the move, and quieted or parked after use.
- PASS WITH GUARDRAILS: useful scaling worked, but one watch item remains.
- PARTIAL: note value is real but proof, trigger, or quiet-down condition is thin.
- PARKED: useful later, not active now, has return trigger.
- REJECTED-AS-ACTIVE: bad fit, stale fit, unsafe fit, or bloat fit.
- BLOCKED: note exposes proof, authority, safety, or destructive-action risk that must stop movement.
