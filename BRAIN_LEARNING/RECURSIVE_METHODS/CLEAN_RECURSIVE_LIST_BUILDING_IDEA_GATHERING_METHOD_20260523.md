## Broad Command Trigger Lock

Status: APPLIED TRIGGER / SMALL RECURSIVE SUBSYSTEM  
Purpose: define what happens when the user gives a broad command.

When the user says a broad command such as:

- go do this;
- make the list;
- gather ideas;
- handle this;
- sort this;
- pull what fits;
- build the candidates;
- find what we missed;

the assistant should not improvise loosely.

The assistant should break out the correct recursive subsystem.

For ordinary list-building, idea-gathering, source-ore extraction, candidate sorting, and note comparison, use this smaller method:

`anchor -> gather -> branch -> fit-test -> clean/rename/drop -> order -> output -> self-proof`

For systemic pressure, repeated failure, deep debugging, deep research, major refinement, or unclear house-wide problems, escalate to the larger Deep Fit Debug Mode.

This trigger does not mean use the heavy mode every time.

It means choose the right recursive branch for the size and pressure of the task.

## Trigger Proof

A broad command is handled correctly only if the assistant can show:

- what task was anchored;
- what sources were gathered;
- how ideas were branched;
- what fit, what was renamed, what was dropped, and what was parked;
- what output was produced;
- what proof or status closes the list.

Verdict: BROAD COMMAND TRIGGER INSTALLED INTO SMALL RECURSIVE METHOD
