# Work for the Work / Proof Is in the Pudding Rule

Status: SUPPORT RULE / LIVE PROOF BEHAVIOR  
Date: 2026-05-23  
Scope: rules, maps, tools, procedures, notes, lenses, task behavior

## Rule

A live map, rule, tool, lens, or note is not a decorative checklist.

It must do work for the active work.

It learns by being used:
- doing;
- finding;
- proving;
- correcting;
- growing.

## Proof

Proof is behavioral output under load, not merely that the rule or map was written.

Ask:
- Did it change what happened next?
- Did it block a bad move?
- Did it find hidden structure?
- Did it improve the output?
- Did it name what to save, park, reject, or test?
- Did it return to the center point and close cleanly?

## Cycle

Make the map -> link rules/methods/procedures -> act through it -> observe what it changes -> prove by output -> update the map/house if useful -> repeat.

## Short form

Dead notes do not count.

The pudding is changed behavior.
