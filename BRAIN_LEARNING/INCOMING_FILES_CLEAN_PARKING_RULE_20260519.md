# Incoming Files Clean Parking Rule

Date: 2026-05-19.
State: active intake / parking rule.
Authority: operating behavior rule, not doctrine install.
Starting base: main @ a690faa.

## Rule

Do not treat Git ignored files as bad milk.

Git ignored is a transport/index state only.

House meaning is:

- incoming material,
- PC-only material,
- not-yet-sorted source ore,
- private/security hold,
- tool candidate,
- proof/history candidate,
- backup/history hold,
- remote-brain candidate,
- archive/delete review candidate.

Nothing is bad milk when it is placed in the correct parking lane, filtered safely, labeled clearly, and retrieved by rule.

## Why This Exists

The folder can contain useful material that is not yet in the remote / URL brain.

That material must not be ignored by awareness.

It also must not be blindly force-added to the remote brain.

## Intake Rule

When a PC-only file appears, do not delete it and do not push it automatically.

First classify it:

1. What is it?
2. What does it touch?
3. Does it contain private/security-sensitive material?
4. Is it useful source ore, proof, tool, backup, or junk?
5. Should it stay PC-only, be added to the remote brain, be archived, or be deleted after approval?

## Whirlpool Rule

Once the clean parking whirlpool starts, incoming material can keep moving without becoming chaos.

The whirlpool needs:

- clean lanes,
- safe filters,
- smart security measures,
- housekeeping rules,
- retrieval paths,
- authority boundaries,
- proof before promotion.

## Boundary

No incoming file is promoted by this rule.
No ignored file is deleted by this rule.
No ignored file is force-added by this rule.
No doctrine install.
No active guide rewrite.
No runtime automation.
