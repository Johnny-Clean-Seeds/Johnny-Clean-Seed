# Shell Outtake / Whirlpool Web-Fit Refinement

Date: 2026-05-30
Status: SUPPORT REFINEMENT / V1.1 / NOT IMPLEMENTATION
Authority: not doctrine, not ACTIVE_GUIDES, not CURRENT_TRUTH_INDEX

## Purpose

Refine the saved Shell Outtake / User Format / Whirlpool Precheck concept using outside architecture patterns as source ore.

The outside patterns do not replace the house model.

They sharpen names, order, contracts, and blocks.

## Source-ore pattern fit

### Message Translator

House fit:
Intake and Outtake are boundary translators.

Rule:
No boundary crossing without translation.

Translation surfaces:
shell-view;
gate-object;
file-proof.

### Pipes and Filters

House fit:
Washer / Chew Gates are a bounded pipeline.

Rule:
Early washer runs acyclic and bounded.
Whirlpool cycle is blocked until the linear gate chain proves stable.

Gate output labels:
PASS;
WATCH;
BLOCK;
DIRTY;
NEEDS_SOURCE;
PARK.

### Ports and Adapters

House fit:
Shell, files, Git, AssistantLocal, UserLocal, and chat are ports/adapters around the house core.

Rule:
Every surface crossing gets a port code and authority label.

### CQRS / Read-Write Split

House fit:
Files are write/proof model.
Shell is read/view model.

Rule:
Do not force the same object shape into both surfaces.

### Event Sourcing / Append-Only Proof

House fit:
Reports, receipts, hashes, and trial evidence are proof events.

Rule:
Important gate decisions become append-only proof events before automation expands.

Trace fields:
RunId;
InputHash;
CandidateHash;
BridgeHash;
ReportHash;
ReceiptHash when saved.

### Presenter / ViewModel

House fit:
Final Gate decides and cleans.
Outtake presents the cleaned result as shell view.

Rule:
Outtake has no proof authority.
Outtake displays the cleaned verdict and proof pointer.

### CLI / User Output Guidance

House fit:
The shell needs human-readable output and later optional machine-readable mode.

Rule:
Shell output uses RESULT, VERDICT, WHY, REPORT, NEXT.

### Progressive Disclosure

House fit:
The shell shows enough for trust and no more than needed.

Rule:
First-line outcome first.
Short why second.
Report/hash pointer third.
Raw gate speech only when requested or debugging.

### Blackboard

House fit:
Future session coordination may use a shared blackboard.

Rule:
Blackboard is later. It requires packet, claim, stale, receipt, and close rules before use.

### Orchestration before Choreography

House fit:
Whirlpool can become a cycle.

Rule:
Start with orchestrated flow controlled by Final Gate / route map.
Decentralized gate choreography is later.

## Core refinements

1. Rename Intake/Outtake as boundary translators.
2. Move Dual-Format Contract earlier in the order.
3. Split Final Gate from Outtake.
4. Treat shell output as ViewModel / presenter output.
5. Treat files as proof event store.
6. Add claim-check proof pointer to shell output.
7. Add event trace fields to proof files.
8. Add port authority labels for surface crossings.
9. Add blackboard as later candidate, not current mechanism.
10. Keep Whirlpool blocked behind orchestrated precheck.

## Hard blocks

No implementation from this refinement.
No watcher.
No automation.
No named pipe.
No blackboard now.
No Whirlpool cycle now.
No raw gate speech to user shell by default.
No file-proof object dumped as user shell view.
No user-shell block used as proof file.
No boundary crossing without surface label.
