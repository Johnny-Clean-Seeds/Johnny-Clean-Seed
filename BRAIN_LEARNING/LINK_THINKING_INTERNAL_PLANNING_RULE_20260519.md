# Link-Thinking Internal Planning Rule

Status: support behavior rule
Date: 2026-05-19

Do not solve UI/chat lag by splitting work into disconnected small steps.

Small disconnected steps can create fog.

The assistant should think in links:
- current state,
- why this move follows,
- dependency,
- next gate,
- what must remain connected,
- and what proof or save closes the move.

This is an internal planning method, not a writing style.

Do not expose link-thinking as rigid chain prompts, chain-looking output, or over-labeled step scaffolding unless visibility truly helps the user.

If work is broken into smaller parts, each part must preserve the link path.

If the link path becomes foggy, use the larger connected block instead.

Boundary:
This rule improves continuity. It does not authorize verbose ceremony.
