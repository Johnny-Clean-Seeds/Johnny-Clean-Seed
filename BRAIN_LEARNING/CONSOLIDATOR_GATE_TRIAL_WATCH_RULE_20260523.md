# CONSOLIDATOR Gate Trial / Watch Rule

Status: SUPPORT GATE / TRIAL-WATCH MODE / NOT DOCTRINE
Date: 2026-05-23

## Purpose

CONSOLIDATOR Gate finds repeated ambiguity, overlap, duplicate logic, scattered pieces, and better homes for gate logic.

It proposes cleaner packing.

It does not automatically rewrite the house.

## Placement

Trial placement:

`Hindsight Lens -> Next Spin Sequencer Gate -> CONSOLIDATOR Gate trial pass -> Omega closes -> Final Judge / final clean status`

Reason:

- Hindsight sees the run after evidence exists.
- Next Spin Sequencer proposes the next order.
- CONSOLIDATOR checks whether repeated gate logic should stay separate, move, pack, park, reject, or keep watching.
- Omega closes the loop.
- Final Judge checks final status.

## Trigger

Run CONSOLIDATOR trial pass only after evidence exists.

Valid triggers:

- a completed gate run;
- hindsight found repeated overlap or ambiguity;
- Next Spin Sequencer produced a watch item;
- several rules/gates repeatedly touch the same logic;
- a user asks whether gate logic should be merged, moved, packed, parked, or rejected.

Blocked trigger:

Starting a spin with CONSOLIDATOR before the gate run produces evidence.

## Output Labels

Every reviewed item must receive one label:

- KEEP SEPARATE: gate has its own clean job.
- MOVE LOGIC: idea belongs under another gate.
- PACK TOGETHER: several small pieces should live inside one broader gate.
- PLACE AS CHECKPOINT: useful support check belongs attached to a parent gate and inherits that parent's trigger.
- PARK: useful, but not ready.
- REJECT: fog, duplicate, or dirty bloat.
- WATCH AGAIN: possible pattern, but not enough proof.

PLACE AS CHECKPOINT is the safe answer when an idea is real but too small to become a full gate. It keeps the check alive without making it a loose boss.

## Distinct Job Test

Before merging anything, ask:

1. Does this gate have a distinct job?
2. Does it have a distinct trigger?
3. Does it protect a boundary another gate does not protect?
4. Does it create clarity by staying separate?
5. Would merging it hide a useful function?

If yes, keep it separate.

Touching is not duplication.

Example:

- Boundaries Gate says whose place is what.
- Permission Gate asks whether an actor may enter that place now.

These are neighbors, not duplicates, unless repeated spins prove the split creates more fog than clarity.

## Best Parent Rules

If an idea is not strong enough to be its own gate, pack it under the natural parent:

| Idea Type | Best Parent |
|---|---|
| placement rule | Boundaries Gate |
| entry rule | Permission Gate |
| smaller output check | parent gate checkpoint |
| cleanup / return rule | Return-Home Gate |
| repeat violation rule | Enforcement Ladder Gate |
| future-use idea | Future Plans / Parking |
| sequence change | Next Spin Sequencer Gate |
| whole-loop close issue | Alpha Omega / Omega close |
| creative source idea | Creative Gate |
| repeated overlap or ambiguity | CONSOLIDATOR watch item |

CONSOLIDATOR does not own every new idea.

## Repeated Signal Standard

Trial mode standard:

- one spin: WATCH AGAIN;
- two spins: LIKELY PATTERN;
- three spins: CONSOLIDATION CANDIDATE.

CONSOLIDATION CANDIDATE means "propose cleaner packing." It does not mean "rewrite now."

## Blocked Moves

CONSOLIDATOR cannot:

- judge PASS / FAIL;
- close the loop;
- replace receipts;
- decide next order by itself;
- replace Next Spin Sequencer;
- replace Hindsight;
- replace Alpha Omega;
- replace Final Judge;
- activate doctrine;
- rewrite ACTIVE_GUIDES;
- rewrite CURRENT_TRUTH_INDEX;
- merge gates without proof;
- create loose checkpoints that fire without a parent trigger;
- create infinite checkpoint chains;
- kill a gate just because it touches another gate.
