# Small Build Strength / Clean Phase Wrap Rule

Status: SUPPORT RULE / BUILD-PHASE JUDGMENT  
Date: 2026-05-23  
Scope: small builds, compact rules, short packets, icons, narrow phases

## Rule

Limited space or capacity does not make a build weak.

A build is not weak because it is:
- small;
- compressed;
- scoped;
- constrained;
- narrow;
- minimal.

A small build can be strong if it has:
- a clear center point;
- bounded cube/home;
- correct role;
- proof surface;
- clean phase wrap.

## Weakness condition

A build becomes weak when it is:
- unfinished;
- unwrapped;
- unclearly phased;
- not closed into a clean working phase;
- missing proof;
- missing its next state.

## Short form

Small is not weak.

Unwrapped is weak.
