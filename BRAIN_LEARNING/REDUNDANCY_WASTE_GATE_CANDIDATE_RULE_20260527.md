# Redundancy / Waste Gate Candidate Rule

Date: 20260527

## Status

Candidate gate / live process helper.

Not doctrine.
Not a new boss.
Not automatic delete authority.
Not CURRENT_TRUTH_INDEX material.
Not ACTIVE_GUIDES material.

## Purpose

Detect redundant data waste before it becomes bloat, fog, repeated chat weight, duplicate files, over-carried suit load, stale copies, unnecessary script bulk, repeated source material, or bloated proof.

## Why this exists

Recent repair history showed the same waste pattern repeating:

- long terminal-pasted scripts became the risk surface;
- saved or partially washed ideas stayed in active chat weight;
- explanations repeated instead of becoming pointers;
- receipts and proof were not always staged at the right time;
- old route assumptions survived after the better idea had already been found;
- the assistant explained a fix but did not always wear it before the next action.

## When it fires

Fire this candidate:

1. before save/lock;
2. after save/lock;
3. during repeated failure;
4. when a reply, script, proof, or process starts getting too large;
5. when saved material is still being carried in active suit/chat weight;
6. when duplicate ideas, files, rules, or explanations appear;
7. when context weight begins obscuring the active object;
8. before printing completion after a long repair loop.

## What it checks

Check for:

- duplicate rules;
- duplicate files;
- repeated explanations;
- dead chat weight;
- saved material still carried in suit;
- giant scripts when a file-run path is cleaner;
- proof logs that prove nothing new;
- parking piles with no return trigger;
- same idea saved under too many names;
- copies left loose after move/custody;
- repeated source material instead of pointer-linked source;
- over-carried Soft Suit load;
- old route assumptions still active after correction.

## Evidence sources

Think from:

- history;
- evidence;
- repeated patterns;
- failed routes;
- proof receipts;
- existing gates;
- neighboring rules;
- outside-source comparison when useful;
- current active object.

## Verdicts

Allowed verdicts:

- KEEP;
- COMPRESS;
- POINTER-ONLY;
- SHED;
- MERGE;
- PARK;
- NEEDS REVIEW;
- PROOF NEEDED.

## Boundaries

This gate must not delete by itself.

It must not flatten useful source history.

It must not kill repetition when repetition is needed as proof.

It must not become a boss or doctrine by default.

It advises compression, shedding, merge, parking, or review. Final action still needs proper lane/proof.

## Soft Suit link

After a save/lock passes, this gate asks what can drop from active suit/chat weight and what should remain only as a pointer.

## Short form

Ask: what can stop being carried now?
