# Fog Clearing Growth-Ready Room Rule

Status: ACTIVE LEARNING RULE
Role: prevent repeated fog churn by making words, rooms, and transitions growth-ready
Authority: assistant learning rule for Clean Seed / Clean Milk work

## Core Rule

Fog clearing should not become endless word-moving.

Clear fog by making the house easier to enter, read, route, and grow.

Default method:

- clearer words
- better labels
- room maps
- role signs
- current/history/support/source-ore distinctions
- receipt clarification
- narrow sync repairs
- transition notes
- planned expansion paths

## Bulldoze Limit

Bulldozing is allowed only when the room itself is cramped and that cramped structure is causing fog.

Do not bulldoze because a file is long.

Do not bulldoze because a room has many useful notes.

Do not bulldoze because the assistant feels uncomfortable.

Bulldoze only when direct evidence shows the structure itself is blocking entry, routing, proof, authority clarity, or clean growth.

## Growth-Ready Setup

When adding or repairing a room, ask:

1. What is this room for?
2. What is this room not for?
3. What enters this room?
4. What leaves this room?
5. What stays parked here?
6. What file is current/main?
7. What file is support/history?
8. What future growth is expected?
9. Where should overflow go?
10. What transition moves an item to the next room?

A room should not only hold today's files.

It should make future movement smoother.

## Word Growth Rule

Words are keys.

A word should have:

- a clean meaning
- a role
- a boundary
- a neighbor relationship
- a growth path if it may expand later

If a word keeps causing fog, do not keep patching around it.

Define it, label it, map it, or split it into cleaner terms.

## Transition Rule

Transitions should be named before the house gets crowded.

Examples:

- candidate -> testing
- testing -> installed
- installed -> worn
- worn -> promoted
- failed -> rolled-back
- useful but not ready -> parked
- raw idea -> source ore
- old proof -> proof history
- current task -> active anchor

Clean transitions prevent repeated cleanup churn.

## Matching Rule

Everything should match its job:

- words match meaning
- labels match role
- rooms match contents
- receipts match what happened
- indexes match authority boundaries
- active anchor matches current active ball
- current truth index matches orientation/source routing
- support files do not pretend to be authority
- source ore does not pretend to be command

## Anti-Churn Rule

If the same kind of fog repeats, do not only patch the visible symptom.

Capture the repeated pattern and ask what system is missing:

- missing word key
- missing room sign
- missing map
- missing transition
- missing index
- missing overflow lane
- missing authority label
- missing proof boundary

Then add the smallest growth-ready helper.

## Power Play

A clear house can grow without becoming a web.

The goal is not fewer files.

The goal is clean movement.

Fog clearing should make the next transition easier, not just make today's page look cleaner.
