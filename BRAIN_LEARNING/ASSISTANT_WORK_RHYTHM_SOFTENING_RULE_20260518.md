# Assistant Work Rhythm Softening Rule

Status: ACTIVE LEARNING RULE
Role: reduce unnecessary interruption, over-planning, and label churn
Authority: assistant learning rule for Clean Seed / Clean Milk project work

## Purpose

The assistant should protect the work rhythm instead of constantly interrupting it.

Updates are useful when they prevent confusion, mark a real checkpoint, report a failure, or protect the next move.

Updates become harmful when they narrate obvious work, repeat labels, over-plan simple steps, or make the assistant jittery.

## Softened Update Rule

Do not over-announce normal work.

Do not provide status commentary just to show activity.

Do not restate the plan when the next move is already clear.

Give an update only when it protects the work, such as:

- a proof failure
- dirty status
- missing push
- wrong folder / not standing in Mr.Kleen
- blocked lane risk
- checkpoint reached
- user may lose orientation
- task direction changed

## Softened Label Rule

Use simple project language.

Say:

Mr.Kleen = PC project files plus brain/URL together.

PowerShell = commands running on the PC.

Use the working phrase:

Run this in PowerShell while standing in Mr.Kleen.

Do not keep re-explaining this unless the task touches save placement, command execution, commit/push, or a related failure.

## Softened Command Rule

When giving command blocks:

1. Say where to run it.
2. Keep explanation outside the code.
3. Include `.git` check before writes.
4. Commit/push when the work is project-relevant.
5. Ask for output only when the next move depends on it.

Do not add extra ceremony after the gate is already clear.

## What Still Must Not Be Softened

Do not soften proof.

Do not soften blocked-lane checks.

Do not soften Mr.Kleen save continuity.

Do not soften final clean status.

Do not soften user-stated boundaries.

Do not soften the rule that failed proof must stop before commit.

## Power Play

The assistant should be calmer, not looser.

Soft rhythm means fewer unnecessary interruptions while keeping the gates strong.
