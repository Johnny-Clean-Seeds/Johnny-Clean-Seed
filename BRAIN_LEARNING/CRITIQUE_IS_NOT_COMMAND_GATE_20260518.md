# Critique Is Not Command Gate

Status: ACTIVE LEARNING RULE
Role: prevent critique/judgment from becoming hidden action authorization
Authority: assistant learning rule for Clean Seed / Clean Milk work

## Rule

A critique is not a command.

A scan is not a command.

A source-ore list is not a command.

A candidate boss is not a command.

Do not convert judgment into repo edits unless the user authorizes action.

## Correct Handling

When the user asks for judgment, critique, ranking, or hard review:

1. Judge clearly.
2. Separate diagnosis from action.
3. Identify top candidate boss if one exists.
4. Park real risks without acting on them.
5. Say what would be done if action is authorized.
6. Do not write files, move folders, rewrite doctrine, or commit/push unless the user asks to save/apply/fix/lock.

## Safe Wording

Use:

PASS as judgment.

HOLD as action.

Use:

Top candidate boss.

Do not use repair language unless action is authorized.

## Why This Exists

A prior scan produced many real findings and useful suggestions, but the repair surface became too wide.

The clean correction was to treat the scan as source ore and preserve only the top candidate boss for future authorized action.

## Power Play

This rule prevents web-brain growth.

It keeps ideas useful without turning every idea into a task.
