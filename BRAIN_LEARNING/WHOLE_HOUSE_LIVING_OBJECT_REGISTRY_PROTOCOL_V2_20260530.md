Each species needs a different contract.

### 2. Fixity is not proof by itself

External attachment: BagIt uses payload manifests and tag manifests with checksums for storage/transfer packages.

House translation:

``text
HASH WITHOUT BONE is dirty.
``

A meaningful hash needs:

- object path;
- object species;
- hash value;
- receipt;
- purpose;
- what hash proves;
- what hash does not prove.

### 3. Metadata crate shape

External attachment: RO-Crate uses structured metadata to describe a research object, its data entities, and contextual entities.

House translation:

A living object needs enough context to be found, understood, routed, and reused without chat guessing.

### 4. Shape validation

External attachment: SHACL validates RDF graphs against shapes/constraints.

House translation:

Species contracts are house shapes. A checker should apply the correct shape for the object species instead of checking all files with one flat rule.

### 5. Minimal metadata terms

External attachment: Dublin Core provides stable metadata terms for describing resources.

House translation:

The Living Object Contract should use stable fields: identifier, title/name, creator/actor, date, source, relation, type/species, description, rights/boundary, and coverage/scope when useful.

### 6. Supply-chain proof

External attachment: in-toto and SLSA-style provenance describe steps, actors, materials, and products in a chain.

House translation:

Save/proof receipts should show source -> action -> output -> verification -> boundary, not merely a commit hash.

### 7. Lineage facets

External attachment: OpenLineage uses runs, jobs, datasets, and facets.

House translation:

The house needs object, action/job, run/receipt, and facets: owner room, key tags, proof, route, return, currentness, disposition.

### 8. Tag vs key

External attachment: DataHub distinguishes informal tags from controlled glossary terms.

House translation:

``text
TAG = search scent.
KEY = movement permission.
``

A tag may help find an object. A key controls what it can do.

### 9. Zero trust

External attachment: zero trust means trust is not implicit and must be evaluated.

House translation:

No living object is trusted by location, recency, helper involvement, save status, or volume. Verify current allowed use.

## The real clean test

A living object is clean if it can answer without chat memory:

- Who am I?
- What species am I?
- What room owns me?
- What key controls me?
- What is my hash?
- What proves this copy?
- What is my authority class?
- What ledger tracks me?
- What map route reaches me?
- What toolbelt handles me?
- What can I do?
- What am I forbidden to do?
- Where did I come from?
- What did I become?
- Where do I return?
- What is my next condition?
- Am I current, candidate, proof-only, source ore, parked, superseded, or blocked?
