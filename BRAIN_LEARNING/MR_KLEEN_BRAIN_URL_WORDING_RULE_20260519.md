# Mr.Kleen Brain / URL Wording Rule

Date: 2026-05-19.
State: active wording / scope rule.
Authority: operating behavior rule.
Starting base: main @ 2f7851a.

## Rule

In Mr.Kleen, brain means the remote / URL / GitHub-backed brain.

Do not call ChatGPT memory or chat continuity brain-side saved.

ChatGPT memory may support continuity, but it is not the project brain.

## Lock/Save Meaning

Unless the user explicitly says chat-only, PC-only, no commit, or no push, lock/save means:

1. write the project-relevant file or artifact into the user's Mr.Kleen repo on the PC;
2. commit it;
3. push it to origin/main so the remote / URL brain is updated;
4. verify clean final status.

## Correct Reporting Language

Use:

- Memory continuity updated, when only assistant memory changed.
- Mr.Kleen files written, when files exist in the PC repo.
- Mr.Kleen saved and pushed, when commit and push update origin/main.

Do not use:

- brain-side saved, to mean chat memory;
- local, to describe a normal Mr.Kleen save/push;
- saved, when only chat memory changed.

## Why This Matters

This is a word-key and scope-label rule.

Bad labels create fog around authority, save target, and whether the remote / URL brain actually knows the change.

## Boundary

No doctrine install.
No active guide rewrite.
No runtime automation.
