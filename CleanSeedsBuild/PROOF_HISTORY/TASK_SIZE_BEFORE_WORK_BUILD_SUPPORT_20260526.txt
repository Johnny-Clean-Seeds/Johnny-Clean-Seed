Purpose:
Task Size Before Work is the front-door governor for clean motion. It forces the assistant to match object, size, surface, proof, state, and user burden before acting.

Operating rule:
Size first. Surface second. Work third. Proof closes.

Before work begins, classify:
- object type
- task size
- delivery surface
- user burden
- state cleanliness
- proof need
- failure history
- authority / lane
- close condition

Task sizes:
- Tiny: direct fact, definition, yes/no, one-line judgment.
- Small: short explanation, simple instruction, one command, quick visible image observation.
- Medium: code fix, workflow repair, practical plan, file handoff, local tool step, terminal recovery.
- Large: deep synthesis, research paper, multi-source comparison, system review, think-tank expansion.
- Boss: recurring failure, doctrine/rule/gate change, repo state, proof/save route, compatibility issue, whole-house scan, repeated assistant behavior failure.

Branches:
- simple direct answer
- ambiguous answer plus one fork
- file/source first
- code shape first
- correction patch first
- deep synthesis
- big boss/system repair
- repo/save proof route

Hard stops:
- pause
- PowerShell continuation prompt such as >>
- unread error
- unclear object
- missing path
- wrapper confusion
- unverified README/file claim
- path being called code
- PASS without proof
- dirty save route

Clean build boundary:
This file carries the rule inside the build support/proof lane only. Active guide promotion would require separate guide-file review, backup, receipt, and proof.
