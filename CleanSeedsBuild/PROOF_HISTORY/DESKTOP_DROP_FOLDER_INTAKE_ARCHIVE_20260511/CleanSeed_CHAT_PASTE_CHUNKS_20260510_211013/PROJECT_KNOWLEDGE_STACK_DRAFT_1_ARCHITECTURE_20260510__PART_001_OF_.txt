4. Folder definitions

00_CURRENT_CONDUCT
HOME:
Current operating state, active route, active boundaries, load order, and next clean route.
JOB:
Tell ChatGPT what to load, obey, and avoid right now.
KEY:
CURRENT_TRUTH_INDEX, active guide references, AGENTS.md bridge limits, and explicit user scope.
PARKING:
Temporary current-state notes that are not yet ready for the master index.
REST STOP:
Clean accepted current-state items waiting for the next index update.
NEXT GATE:
Current-state review and user approval before any active guide or truth-index update.
Belongs there:
Current conduct summaries, current cleanup state, active-guide load instructions, bridge boundaries.
Must not belong there:
Source-ore, old guides by identity, unreviewed guesses, executable or blocked material.
Evidence proves placement:
Exact active path, current status, receipt, or user-approved conduct update.
Dirty milk if:
It treats proof-history or OldSchool material as current authority by identity.

01_CLEANED_RULE_STACK
HOME:
Cleaned rules and concepts that have been reviewed, grouped, and statused.
JOB:
Preserve the organized rule stack without bloating active guides.
KEY:
Source link, rule family, status, evidence, and install state.
PARKING:
Cleaned but not installed rules.
REST STOP:
Accepted rules waiting for batch review.
NEXT GATE:
Active-guide review or proof-history-only closure.
Belongs there:
Mirror Law, Respect Neighbor Rules, Sorting Rules, old/new judgment method, cleanup-control rules after review.
Must not belong there:
Unlabeled guesses, raw source-ore, historical wording without context.
Evidence proves placement:
Proof-history source path, receipt, and status as cleaned/parked/installed/rejected.
Dirty milk if:
The stack becomes a fourth active guide or silently installs rules.

02_ORIGIN_CONCEPTS
HOME:
Original concept sources, old wording, recovered ideas, and source lineage.
JOB:
Preserve where ideas came from without promoting them.
KEY:
Source path, source type, timestamp if available, and proof-history link.
PARKING:
Origin material that needs later review.
REST STOP:
Origin items whose value is accepted but placement is not ready.
NEXT GATE:
Content review, old/new comparison, or source-ore review.
Belongs there:
OldSchool concept references, recovered markers, older guide-looking files, concept checkpoints.
Must not belong there:
Active guide truth by identity, final cleaned doctrine, executable items.
Evidence proves placement:
Original path, intake map, receipt, and source-ore status.
Dirty milk if:
Old source material overrides the active guides because it looks familiar or newer.

03_EVIDENCE_AND_RECEIPTS
HOME:
Receipts, manifests, ledgers, maps, hashes, proof files, and no-change confirmations.
JOB:
Make proof durable and findable.
KEY:
Receipt name, linked action, source/destination path, hash if available, no-change lines.
PARKING:
Evidence needing exact role map.
REST STOP:
Closed receipts waiting for indexing.
NEXT GATE:
Proof-history exact role map or receipt index update.
Belongs there:
Receipts, correction notes, cleanup drafts, intake maps, quarantine ledgers, manifests.
Must not belong there:
Active doctrine by identity, unproved claims, memory-only proof.
Evidence proves placement:
Receipt text, exact path, hash if relevant, linked report.
Dirty milk if:
Proof is compressed into memory or deleted because it looks repetitive.

04_NOTES_AND_CANDIDATES
HOME:
Guesses, parked ideas, design candidates, metaphors, lane concepts, future rule candidates.
JOB:
Hold useful uninstalled material without giving it authority.
KEY:
Label, source, reason for parking, blocked action, reopen condition.
PARKING:
Default state for all guesses and not-ready ideas.
REST STOP:
Clean candidate accepted for later batch, but not installed.
NEXT GATE:
Candidate review, old/new comparison, bloat check, or active-guide placement review.
Belongs there:
Near-Right Is Not Proof, Guesses Have Parking Spots, All Lanes Have Parking, city/town metaphor candidate.
Must not belong there:
Current truth, delete authority, active guide replacement text, executable material.
Evidence proves placement:
Checkpoint file, proof-history note, user statement, or review packet.
Dirty milk if:
A candidate becomes active truth because it sounds right.

05_BLOCKED_AND_QUARANTINED
HOME:
Shortcuts, scripts, command-looking items, zips, no-key material, quarantine evidence, and blocked gates.
JOB:
Keep risky or inert evidence visible and non-executing.
KEY:
Exact path, blocked reason, item type, quarantine/custody record, and user approval before any action.
PARKING:
Blocked evidence lane.
REST STOP:
None unless a blocked item is later safely reclassified.
NEXT GATE:
Blocked item gate, quarantine review, or exact user decision.
Belongs there:
.lnk, script-like, zip/archive, command-looking, no-key items, recovered quarantine indexes.
Must not belong there:
Active guide files, normal proof receipts misread as commands, source-ore content without blocked reason.
Evidence proves placement:
Extension/type, name, custody note, or blocked-item review.
Dirty milk if:
A shortcut or command-looking item is opened, run, extracted, renamed into use, or deleted without gate.
