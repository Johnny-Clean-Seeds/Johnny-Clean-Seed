PRE-RUN INSTALLER / FILE EXISTENCE GATE

Status:
Local-file-saved support rule.

Rule:
Before asking the user to run a downloaded installer, script, file packet, or local-save command, verify the file path first.

Correct order:
1. Name the target local path.
2. Make sure the file is actually saved there.
3. Use Test-Path or equivalent before execution.
4. If missing, stop cleanly with installer-location FAIL.
5. Do not call it package failure.
6. Use the smallest clean fix, such as an inline PowerShell save packet when a download path is unreliable.

Reason:
A script cannot be run from Downloads unless the file actually exists in Downloads.
