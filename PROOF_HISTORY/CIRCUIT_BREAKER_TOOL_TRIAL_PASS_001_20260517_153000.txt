TOOL NAME:
Circuit Breaker

TOOL CLASS:
Control-pattern candidate.

PROBLEM IT TARGETS:
Repeated same-class failure loops can burn time, tokens, context, and trust.
The system may keep retrying the same bad route while calling it persistence.

Examples:
- same script keeps failing from same missing state
- same assistant drift keeps recurring
- same cleanup/scan route keeps producing no new proof
- same tool path is blocked but still retried
- same correction is repeated without changing method

CIRCUIT BREAKER JOB:
Stop repeated same-class failures after a clear threshold and force a safer state.

FIT TEST:
Circuit Breaker is useful in Clean Seed work if it can do all of these:
1. Detect repeated same-class failure.
2. Count or identify recurrence enough to justify stopping.
3. Stop retries before waste becomes damage.
4. Return a clear state: BLOCKED / FAIL / NEEDS NEW ROUTE / USER DECISION.
5. Preserve the failed evidence.
6. Avoid deleting or hiding failed attempts.
7. Route to repair, rollback, park, or user decision.

TRIAL AGAINST RECENT FAILURE TYPES:
Recent relevant failures:
- fake-running posture repeated before correction
- stale/missing state risk in staged scripts
- possible over-scanning / over-naming before useful state change
- wrong or old repo path risk after rename

Correct Circuit Breaker behavior:
- Do not retry the same failing route blindly.
- Stop after repeated same-class miss.
- Name the failure class.
- Preserve evidence.
- Change method before retrying.

PASS CONDITION:
Circuit Breaker passes if it stops repeated same-class failure and forces a changed method or clean block.

FAIL CONDITION:
Circuit Breaker fails if it becomes a panic stop, blocks useful persistence too early, or hides failure evidence.

OBSERVED RESULT:
Circuit Breaker is a strong rack tool.
It complements Readiness Probe:
- Readiness Probe checks before action.
- Circuit Breaker stops repeated failure during/after action.

COMPRESSED OPERATING SHAPE:
When a route fails:
1. Classify the failure.
2. If first failure: repair normally.
3. If same-class failure repeats: trip breaker.
4. Stop blind retry.
5. Preserve evidence.
6. Choose changed method, rollback, park, or user decision.
