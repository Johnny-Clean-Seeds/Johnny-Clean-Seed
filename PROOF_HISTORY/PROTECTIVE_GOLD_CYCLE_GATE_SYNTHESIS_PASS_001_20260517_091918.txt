PRIMARY SYNTHESIS
Protective Gold Cycle Gate

JOB
When gold appears during a task, protect the active task while preserving the gold.

CORE ROUTE
SIGNAL -> TRAY -> TAG -> RETURN -> DIGEST -> CLASSIFY -> TEST/SAVE/INSTALL

WHY IT EXISTS
Gold appears during active work.
If chased inline, the assistant drops the active task and creates more loose ore.
If ignored, useful gold is lost.
The correct move is capture without derailment.

DIRTY ROUTE
ORE -> EXCITEMENT -> INLINE CHASE -> ACTIVE TASK DROP -> MORE ORE LOST

CLEAN ROUTE
ORE -> TRAY -> TAG -> RETURN TO ACTIVE TASK -> DIGEST CHECKPOINT -> CLASSIFY -> TEST/INSTALL ONLY IF APPROVED

COMPONENTS

1. SIGNAL
Meaning:
A useful idea, repeated pattern, protective rule, algorithm fit, bit/string clue, or child-gold item appears.

Action:
Recognize it without chasing it.

2. TRAY
Meaning:
Temporary visible holding area.

Action:
Place the item in a bounded tray, not active doctrine.

3. TAG
Meaning:
Each item gets minimum routing metadata.

Required tag fields:
- source
- why it matters
- active task it interrupted
- return trigger
- risk if chased now
- likely home

4. RETURN
Meaning:
Go back to the active task.

Action:
Do not let gold capture become the new task unless user explicitly redirects.

5. DIGEST
Meaning:
Review tray items at a checkpoint.

Action:
Use question ladder only after the active task is safe.

6. CLASSIFY
Possible states:

- reference
- quarantine
- install candidate

- duplicate support
- needs user approval

7. TEST / SAVE / INSTALL
Meaning:
