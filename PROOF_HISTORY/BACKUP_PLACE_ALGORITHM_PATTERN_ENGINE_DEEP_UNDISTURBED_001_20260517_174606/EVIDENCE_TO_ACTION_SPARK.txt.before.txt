SPARK ROUTE:

2. EVIDENCE

- git status
- commit/push state

3. ALGORITHM

The algorithm must fit the job.
Do not use an algorithm because it sounds impressive.

4. SAFE ACTION
Act only inside the authorized scope.
Use the smallest clean move.
Do not edit authority files unless explicitly approved.
Do not delete, move, rename, or overwrite without proof and approval.

5. PROOF
Prove what happened:
- file created or updated
- hash
- receipt
- backup if needed
- git commit
- push
- final status
- boundaries held

6. LEARNING
Explain what changed, what it felt like before and after, why it is better, what got heavier, and what remains unproven.

7. RETURN
Close the wave.
Put tools back.
Name the next state.
