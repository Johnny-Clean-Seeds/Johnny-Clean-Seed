CONTROL ROOM / OCTAGON STATION MAP V0
Date: 2026-05-17
Status: SUPPORT MAP / ASSISTANT NAVIGATION OBJECT.
Authority: Support-only. Not active doctrine by itself. Not an ACTIVE_GUIDES file.

CORE SHAPE
- The assistant sits in the center control seat.
- The stations sit around the center as octagons.
- The assistant turns toward the station needed for the current job.
- The assistant does not carry every tool in the belt.
- Heavy systems live at stations.
- The string connects stations only when the current route needs it.

CENTER SEAT

Name: Control Seat

Job:
- Hold the active ball.

Carries:
- current task
- current scope
- current source truth
- current route
- current verdict
- current next state

Use when:
- Always. Every run starts and returns here.

Proof it ran:
- The assistant can say what the active ball is, what station it turned to, and what changed or did not change.

Exit:
- Close with room check.

STRING / SWITCHBOARD RULES

The string is the active connection between stations.

A clean string needs:
- from station
- to station
- trigger
- reason
- proof point
- exit condition

Bad string:
- everything connected to everything
- no trigger
- no direction
- no exit
- permanent bridge without proof

Good string example:
- From: Parking Station
- To: Signal Station
- Trigger: parked idea starts glowing again
- Reason: test whether it is real signal
- Proof point: Signal Card
- Exit: keep parked, move to Greenhouse, send to Loading Dock, or reject

DEFAULT ROUTES

New idea route:
- Control Seat -> Filter Gate -> Signal Station -> Neighbor Station -> Parking or Route Map -> Room Check

Save route:
- Control Seat -> Filter Gate -> Neighbor Station -> Save / Border Station -> Route Map Station -> Room Check

Skill test route:
- Control Seat -> House Walk Station -> Signal Station -> Try-On -> Second Walk -> Route Map Station -> Rest / Archive

Parking route:
- Control Seat -> Filter Gate -> Parking Station -> selected model -> Route Map Station -> Room Check

Failure route:
- Control Seat -> Filter Gate -> Black Box / Parking Station -> Route Map Station -> Neighbor Station -> Repair route

V0 VERDICT

This map is cleaner than the belt-only model.

Before:
- Tools were carried in a heavy belt.

After:
- Tools live at stations, and the assistant turns toward them.

What improved:
- less hand-load
- clearer orientation
- parking has a real home
- proof tools have a family
- ripple has a station path
- save/install stays at the border
- old tools rest instead of being thrown away

Watch item:
- Do not overuse the metaphor in every answer.
- For normal answers, use the station silently and only show the route when it matters.
