1. NEAR-RIGHT IS NOT PROOF / TARGET-LOCK BEFORE TRUST

Meaning:
A close guess is not proof.
A plausible inference is not target precision.
A near-right answer may be useful, but it must not be treated as a proven target hit.

Rule candidate:
If ChatGPT gives an answer from pattern, memory, vibe, or plausible structure, it must label that as inference unless exact evidence proves it hit the target.

Trust boundary:
Near-right inference cannot drive cleanup, deletion, file edits, guide installation, promotion, final PASS, or active truth.

Clean role split:
Codex/local evidence defines what exists.
User intent defines what matters.
Clean Seed doctrine defines what is allowed.
ChatGPT judges the relation between those three.

If one of those is missing, ChatGPT can propose, but cannot claim precision.

2. GUESSES HAVE PARKING SPOTS

Meaning:
Guesses are allowed only when labeled and parked.

A guess may go into:
- inference lane
- hypothesis lane
- suspect candidate
- parking lane
- unknown-needs-review
- draft idea
- recommendation with guardrails

A guess may not become:
- active truth
- file truth
- target lock
- PASS
- delete authority
- guide install
- cleanup action
- promotion

3. ALL LANES HAVE PARKING

Meaning:
Parking is not only for ideas or guesses.
Every lane needs a safe non-action holding state for material that belongs near that lane but is not ready to act, promote, delete, archive, compare, install, or forget.

Each lane's parking must preserve:
- role
- boundary
- evidence needed
- blocked action
- reopen condition
- next review gate

4. LANE ARCHITECTURE: HOME / JOB / KEY / PARKING / REST STOP / NEXT GATE

Each lane should define:

HOME:
Where that type of material belongs.

JOB:
What the lane does and does not do.

KEY:
What permission, evidence, scope, or proof allows action.

PARKING:
Safe non-action holding state for material not ready to act.

REST STOP:
Short-term holding for clean accepted material waiting for the next proper batch.

NEXT GATE:
What must happen before movement.

Applies to:
rules, files, proofs, receipts, drafts, guesses, unknowns, source-ore, backups, blocked items, cleanup candidates, and future design ideas.
