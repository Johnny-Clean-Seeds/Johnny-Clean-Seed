# Next Issue Clean Wash Routine Card

Status: SOFT SUIT ROUTINE / TRIAL TOOL / NOT DOCTRINE
Date: 2026-05-24
Stamp: 20260524_061135

## Purpose

Use the next issue as live test material instead of inventing a new doctrine pile.

The routine turns a next issue into a clean route, saved proof, and a next state.

## Active Soft Suit Slots

1. Hold Signal Before Shape.
2. Gate Relationship Grammar Card.
3. Clean Wash Signal.

## Routine

1. Smoke break if changing task or repeating a wrong branch.
2. Name the next issue in one sentence.
3. Hold the signal before assigning shape.
4. Ring-label the material from R5 to R0.
5. Pick object type using the grammar card.
6. Name relation verb, authority line, proof need, and exit path.
7. Run the smallest washer cycle that can judge it.
8. Save the summary to parking/support, not chat, unless user asked for full paste.
9. Rerun the same issue once through the changed route.
10. Stop at Clean Wash Signal when two passes produce no new route, proof need, conflict, promotion pressure, or file custody action.
11. Feed the next issue.
12. After two trials, compare proof and decide: failed-fit, park, keep Soft Suit, or adopt as support model.

## Washer Cycle

| Step | Question |
|---|---|
| Source | What is the issue and where did it come from? |
| Hold | What shape am I tempted to assign too early? |
| Route | Which room/gate/lane should judge it? |
| Neighbor | What existing rule or gate already carries part of it? |
| Grammar | What object type and relation verb fit? |
| Proof | What exact proof can this claim use? |
| Exit | Reject, park, helper, checkpoint, candidate, promotion review, or close? |
| Rerun | Does the same issue route cleaner now? |
| Signal | Did two passes produce the same clean output? |

## Clean Wash Signal

Use only when the issue has:

- ring label;
- object type;
- relation verb;
- authority line;
- proof need;
- exit path;
- no new route on rerun;
- no new conflict on rerun;
- no promotion claim.

## Report Shape

Save the report with:

- issue source;
- cycles run;
- flatline / clean wash signal cycle;
- grammar decision;
- proof used;
- files touched;
- blocked actions;
- next issue selected;
- final judge.

## Boundary

- Not doctrine.
- Not active-guide rewrite.
- Not CURRENT_TRUTH_INDEX rewrite.
- Not automation.
- Not broad cleanup.
- Not promotion by itself.
