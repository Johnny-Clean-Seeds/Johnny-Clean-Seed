# Gate Relationship Grammar Card

Status: ONE-PAGE SUPPORT CARD / SOFT SUIT TEST TOOL / NOT DOCTRINE
Date: 2026-05-24
Stamp: 20260524_061135

## Purpose

Prevent early identity and neighbor confusion.

Use this before a signal becomes a rule, gate, test, candidate, save, proof, promotion, or next action.

Core question:

`Am I trying to make this clean before I have let it be strange?`

## Ring Label

| Ring | Label | What belongs here | Allowed move |
|---|---|---|---|
| R5 | Source ore | transcript, web source, rough note, symbolic idea, weak signal | hold, tag, source passport |
| R4 | Signal hold | strange pressure, absence, overload, repeated hint, proto-object | compare, wait, park, or shape lightly |
| R3 | Route/context | lane, room, boundary, value, adapter, bounded context | pick smallest judge |
| R2 | Proof/provenance | receipt, hash, how-found, rerun, source, observed behavior | support exact claim only |
| R1 | Candidate/promotion | Soft Suit item, checkpoint, helper, tool, card, ledger | trial fit and neighbor check |
| R0 | Active core | active anchor, approved guide behavior, final judge | slow layer; change only by scoped proof and approval |

## Object Type

| Type | Job | Authority |
|---|---|---|
| Signal | Shows pressure or strangeness. | none |
| Proto-object | Holds signal before final shape. | none |
| Candidate | Possible fix/rule/check/tool under test. | Soft Suit only |
| Parent gate | Major route owner that decides, blocks, permits, proves, sequences, or closes. | only what current guides grant |
| Checkpoint | Small partner that checks parent output. | no independent authority |
| Helper/adapter | Translates or assists. | no authority by itself |
| Neighbor | Adjacent room/gate/tool with typed relation. | informs, does not override |
| Card/ledger | Records shape, steps, or proof. | evidence container, not judge |
| Parking item | Useful unresolved material with return trigger. | not active |
| Proof object | Receipt, hash, rerun, source, or observed behavior. | supports exact claim only |
| Promotion candidate | Repeatedly proven candidate seeking authority. | not active until separate proof and approval |

## Relation Verb

Use one verb per relationship:

`checks`, `supports`, `blocks`, `routes`, `adapts`, `proves`, `parks`, `promotes`, `supersedes`, `informs`, `facilitates`, `collaborates`, `serves`.

If the relation needs more than one verb, split the object or park the residue.

## Legal Moves

| From | To | Allowed when |
|---|---|---|
| Signal | Proto-object | shape is unclear or too early |
| Signal | Parking | useful but not current |
| Signal | Candidate | pressure repeats and has lane, job, boundary, neighbor, proof need |
| Proto-object | Candidate | comparison shows stable shape |
| Candidate | Checkpoint/helper/card/ledger | useful but not a full gate |
| Candidate | Parent gate | repeated proof shows distinct decision job and trigger |
| Proof object | Claim support | proof fits exact claim and burden |
| Promotion candidate | Authority | scoped proof and user-approved scope exist |

## Illegal Jumps

- Signal to doctrine.
- Signal to proof.
- Signal to final PASS.
- Candidate to active guide.
- Helper to authority.
- Card, ledger, or receipt to proof beyond what it records.
- Symbolic map to operating doctrine.
- One good run to promotion.

## Clean Wash Signal

Stop the wash when the last two passes produce:

- same ring label;
- same object type;
- same relation verb;
- same authority line;
- same proof need;
- same exit path;
- no new route;
- no new promotion pressure;
- no new file custody action;
- no new conflict.

Close as:

`CLEAN WASH SIGNAL / SUPPORT ONLY / NEXT ISSUE READY`
