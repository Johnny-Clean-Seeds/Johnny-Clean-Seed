## Gate And Neighbor Grammar

Object types:

| Type | Job | Authority |
|---|---|---|
| Signal | Indicates pressure, absence, overload, or strangeness. | None. |
| Proto-object | Holds a signal before final shape. | None. |
| Candidate | Named possible helper/rule/check/fix under test. | Soft Suit only. |
| Parent gate | Major route owner that decides, blocks, permits, proves, sequences, or closes. | Only what active guides already grant. |
| Checkpoint | Small partner that checks parent output. | No independent authority. |
| Helper/adapter | Translates or assists a route. | No authority by itself. |
| Neighbor | Adjacent gate, room, or tool with a typed relation. | Can inform, not override. |
| Card/ledger | Records structure, proof, or steps. | Evidence container, not judge. |
| Parking item | Useful unresolved material with return trigger. | Not active. |
| Proof object | Receipt, hash, source, rerun, observed behavior, or ledger. | Supports only exact claim and burden. |
| Promotion candidate | Repeatedly proven candidate seeking later authority. | Not active until separate promotion proof and approval. |

Legal moves:

| From | To | Allowed when |
|---|---|---|
| Signal | Proto-object | Shape is unclear or early identity pressure is present. |
| Signal | Parking | Useful but not current. |
| Signal | Candidate | Pressure repeats and has lane, job, boundary, neighbor, and proof need. |
| Proto-object | Candidate | Comparison shows stable shape. |
| Candidate | Checkpoint/helper/card/ledger | Gate Inventory says it is not a full gate. |
| Candidate | Parent gate | Repeated proof shows a distinct decision job and trigger. |
| Candidate | Parking | Useful but not current or not proven. |
| Proof object | Claim support | Evidence fits the exact claim and proof burden. |
| Promotion candidate | Authority | Scoped promotion proof and user-approved scope exist. |

Illegal jumps:

- Signal -> doctrine.
- Signal -> proof.
- Signal -> final PASS.
- Candidate -> active guide.
- Helper -> authority.
- Card/ledger/receipt -> semantic proof beyond what it records.
- Symbolic map -> operating doctrine.
- One good run -> promotion.
