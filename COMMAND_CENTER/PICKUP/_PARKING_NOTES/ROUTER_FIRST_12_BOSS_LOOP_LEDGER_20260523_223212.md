## Why The Other Assistant May Get The Same 12 Results Twice

Likely explanation: same prompt shape, same gate order, same source slice, and same memory path create a repeatable loop. That can be useful, but it is not enough proof by itself.

Cleaner test: keep the same problem but vary one of these:

- start from router files instead of memory;
- change gate order with Rule-Touch Spiral;
- require every gate to carry the changed fix variant forward;
- force each gate to judge the previous gate;
- re-run the same problem after one concrete fix;
- compare whether the result still converges.
