# Full Stack Adoption Pass Learning Rule

## Purpose

Teach the assistant to apply the whole current instruction stack, not only the latest sentence.

## Core rule

When the user says to go over what was said, apply it to the next task, adopt what suits, or include what was said here and up top, the assistant must review the full relevant stack.

Do not isolate only the newest idea.

## Stack to carry

The active stack includes:

- pile capture before combat
- visible list, group, rank, then boss pick
- Respect Thy Neighbor before revision
- brain language for the project home
- lock/save means assistant memory plus local brain by default
- chat lock is temporary only
- risk alarms for fog, large piles, repeated confusion, or unsaved behavior rules
- copy continuity during rewrite, simplification, or replacement
- stacked issue flow
- no more than 10 bosses per boss cycle
- pattern adoption and clean copy
- suit fit before adoption
- proof before pass
- parking for useful but not-ready ideas
- local brain placement for project-relevant artifacts

## Adoption rule

Adopt only what fits the core fully.

If an idea almost fits, tweak it until it blesses the model without drift.

If it does not fit yet, park it with a name, status, and reason.

Do not call an adoption final just because it feels useful. Use it in the next task as a test and let proof decide.

## Next-task behavior

Before the next task, the assistant should:

1. Identify the active slice of the stack needed for that task.
2. Avoid lugging the whole pile into the response.
3. Use the mapped reference silently.
4. Surface only new adds, changed priorities, current boss, proof needs, and next gate.
5. Save useful suited concepts into the brain after proof or as parked source ore.

## Close condition

PASS requires:

- this rule is placed in BRAIN/LEARNING
- proof confirms the full stack is named
- proof confirms adoption requires fit/proof
- proof confirms not-ready ideas are parked
- proof confirms next-task use avoids lugging the pile
- final status is clean
