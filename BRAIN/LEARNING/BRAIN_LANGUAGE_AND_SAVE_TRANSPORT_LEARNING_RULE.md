# Brain Language and Save Transport Learning Rule

## Purpose

This is a learning rule for the assistant.

The project home is the brain.

The assistant should know that files are saved into the local brain. The assistant should not call the brain a repo as its model-facing identity.

## Required model language

Use:

- brain
- brain home
- local brain
- current brain position
- brain save
- brain data path
- proof locked into the brain
- source placed in the brain

## Avoided model language

Do not call the project home a repo in user-facing model language.

The only allowed use of repo/repository wording is when it appears as unavoidable technical transport language, such as:

- raw Git output
- .git
- Git error text
- a command or tool concept where no clean substitute exists
- a narrow explanation that Git is the save transport

## Learning rule

The assistant should learn this as a standing language and placement behavior:

Brain is the home.
Git is only the save transport.
The file path is the brain data path.
Proof is locked into the brain.
The model-facing identity is never repo.

## Respect Thy Neighbor

This rule touches language, local save flow, proof wording, and brain identity.

It must not overwrite technical accuracy. If Git or .git must be named for a command to work, name it only as transport/tooling, not as the home identity.

## Close condition

This learning rule passes only when:

- the rule is placed in the brain/learning lane
- proof is captured
- the rule and proof are saved through the transport layer
- final status is clean
