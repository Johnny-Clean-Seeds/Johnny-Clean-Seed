# Next Run Pressure Release Learning Rule

## Purpose

When the brain is slammed by a large pile of captured ideas, issues, bosses, fixes, or source-ore drops, the answer is not to overload the active suit.

The answer is controlled next runs.

## Rule

When the pile is too large:

1. Capture the whole pile.
2. Split it into atomic candidates.
3. Group it by lane, burden, neighbor, proof need, and pattern.
4. Rank it by leverage.
5. Keep only the allowed active Soft Suit load.
6. Park the rest with status and next action.
7. Complete the current run.
8. Prove it.
9. Save it.
10. Start the next run from the next clean boss-stack position.

## Pressure response

A huge pile is not permission to skip the gate.

A huge pile is a stronger reason to use the gate.

## Required phrase

When the brain is slammed, say:

This is a next-run pile. Current run holds the top active Soft Suit candidates. The rest are parked for next runs.

## Boundaries

Do not skip the Soft Suit limit under pressure.

Do not dump all captured fixes into active files.

Do not treat obvious fit as proof.

Do not treat parked as rejected.

Do not let next-run piles remain loose in chat.

## Close condition

PASS means the pile is captured, the active load is bounded, parked items remain retrievable, and the next run starts from a clean saved position.
