## Active operating stack

1. Brain language: call the project home the brain.
2. Dual save: lock/save means assistant memory plus local brain by default.
3. Chat lock: chat-only holding is temporary and not durable save.
4. Risk alarms: large/foggy/repeated/behavior-shaping chat material must be placed or parked.
5. Copy continuity: preserve useful meaning when simplifying or replacing.
6. Stacked issue flow: stack, group, rank, fight one boss, then move down.
7. Ten-boss limit: never run more than 10 bosses in one cycle.
8. Pattern adoption: copy useful patterns cleanly, reshape for the lane, prove fit.
9. Full-stack adoption: apply the whole relevant stack, not only the latest sentence.
10. Long idea intake: split, name, lane, status, rank, execute only current boss.
