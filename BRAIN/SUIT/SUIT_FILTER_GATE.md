# Suit Filter Gate

## Purpose

This gate controls movement from idea to Soft Suit, from Soft Suit to Hard Suit, and from Hard Suit to active files.

## Flow

Idea / source ore / chat lock
→ Soft Suit
→ linked Hard Suit test
→ fit test
→ neighbor check
→ proof
→ combined-effect test
→ Hard Suit active pack or support link
→ controlled active-file dump only if needed

## Gate 1 — Enter Soft Suit

An item may enter Soft Suit if it has:

- name
- job
- lane
- reason
- neighbor awareness
- proof need
- status

Soft Suit max: 3 active bosses.

Soft Suit may test 1, 2, or 3 active bosses. It does not need to fill all 3.

## Gate 2 — Promote to Hard Suit

An item may enter Hard Suit active pack only if:

- it fits the core 100 percent
- it does not drift the model
- it has proof
- it respects neighbors
- it has a clear operating job
- it is not better left parked
- it needs to be active pack weight

If useful but not active pack weight, link it as a support reference instead.

## Gate 2A — Combined-effect test

Before promotion from Soft Suit to Hard Suit, test the candidate against the current Hard Suit pack.

Check:

- does it strengthen the pack
- does it duplicate a proven item
- does it conflict with a proven item
- does it overload the suit
- does it need a companion
- does it clarify a weak point
- does proof show clean combined behavior

A candidate may be tested alone or beside another Soft Suit candidate when interaction matters.

Soft Suit may hold no more than 3 active bosses during this combined-effect test.

Hard Suit may hold up to 10 active pack items, but it does not need to fill 10.

## Gate 3 — Controlled dump to active files

An item may dump from Hard Suit into active files only if:

- active files need it
- a pointer is not enough
- the change improves routing, proof, safety, or execution
- it will not bloat doctrine
- final proof can verify the dump

## Respect Thy Neighbor

Before promotion or controlled dump, check:

- what lanes are touched
- what rules already do this job
- whether the item needs a companion instead of promotion
- whether it should be a map, window, toolbox item, source ore, support link, or parking item

## Failure path

If an item fails fit:

- do not delete it automatically
- mark it failed-fit
- park it with reason
- keep useful pattern material if any

## Close condition

PASS requires Soft Suit, Hard Suit active pack, support links, combined-effect testing, and active-file dump boundaries to stay separate.
