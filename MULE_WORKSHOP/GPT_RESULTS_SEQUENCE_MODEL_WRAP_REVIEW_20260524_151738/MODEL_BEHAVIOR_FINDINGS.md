# Model Behavior Findings

Status: MODEL BEHAVIOR REVIEW / SUPPORT ONLY
Date: 2026-05-24

## Main Finding

The instant GPT did not fail because it lacked all insight. It failed because insight arrived before
activation.

It could name rope, washer, original work, and active object. But early answers did not yet prove
those ideas had reshaped the output.

## What Got Better

The user's tightening improved the model by forcing:

- source/object continuity;
- paragraph depth;
- washer before delivery;
- replay after failure;
- proof behavior through hash/copy language;
- creative room plus freeze;
- active-object diagnosis;
- release language for parked/source/proof signals.

Best content appeared when the model described:

- the house as inhabited rather than empty;
- congestion from living material rather than dead junk;
- over-activation as the main disease;
- source custody as heavy from fear of losing pressure;
- parking as not complete until attention releases;
- creative outputs needing exit labels;
- the house as a nervous system whose signals must stop firing after their job.

## What Stayed Weak

Weaknesses still visible:

- artifact links and hashes are reported, but the source packet here does not prove the sandbox files
  exist locally;
- some answers use ritual terms without showing all rooms walked;
- final repetitions compress too much;
- the model can become conclusion-shy or artifact-dependent after repeated tightening;
- the model's "Thought for" and "Analysis paused" moments show lag/fragility under pressure.

## Rigidity Finding

By the last run, the model is becoming more rigid:

- it keeps the prior diagnosis as the rope object;
- it uses the expected room terms;
- it summarizes rather than deeply unfolding;
- it sends an artifact link rather than proving content in the visible answer;
- it finds a useful new phrase, unresolved state, but does not fully develop it in the excerpt.

This is not failure by itself. It is a phase signal.

The model has moved from "too loose" to "frame-heavy." That means the next stage should reduce
frame slightly during diagnosis/creative work while keeping proof tight at closeout.

## Final Artifact Finding

`WHOLE_HOUSE_INSPECTION_V3.txt` adds a clean missing piece:

The house's burden is not memory itself. It is unresolved state.

The artifact reframes over-activation as low-level muscle tension: proof, source, parking, creative
room, soft suit, and final judge are all acting responsibly, but each keeps asking to remain active a
little longer.

Clean extraction:

- confidence identity: how sure are we this object belongs here, should remain active, or can sleep;
- controlled relaxation: signals fire, complete, and stand down;
- parked until X is clean; might matter later is tension.

## Weak/Fast Agent Lesson

For a faster or less intelligent agent, the first normal Clean Seed wraps need stronger scaffolding:

- active object card;
- source custody card;
- room route;
- one correction at a time;
- issue replay washer;
- visible changed-output test;
- final receipt.

But the scaffold must expire by phase. If the same strong scaffold stays active during creative room,
the agent becomes rigid and stops discovering.

## Strong/Slow Agent Lesson

For a stronger or slower agent, do not over-box the creative/diagnostic phase.

Give it:

- source and boundary;
- active object;
- allowed/blocked moves;
- output proof target;
- freedom to notice missing pressure.

Then tighten again at washer, staging, commit, and closeout.

## Clean Conclusion

Training should not ask, "Did the model understand?"

Training should ask:

`Did the model's next output visibly change because the rule fired?`

Closeout should ask:

`Can the signal now stand down without losing source truth?`
