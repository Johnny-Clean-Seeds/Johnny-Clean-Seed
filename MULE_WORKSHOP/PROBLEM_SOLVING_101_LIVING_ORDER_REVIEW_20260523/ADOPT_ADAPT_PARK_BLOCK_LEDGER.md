## Adopted

### ADOPT - Question Before Broadening

Source fit:

The source names the exact missing behavior: before broadening a list or method, interrogate the object.

House fit:

Matches Broad-Bite, No Parking Without Fit Decision, Patch-Through, and Final Judge.

Installed as:

`BRAIN_LEARNING/PROBLEM_SOLVING_101_QUESTION_BEFORE_BROADENING_RULE_20260523.md`

### ADOPT - Ask the Object Ladder

Core questions:

- what is it;
- what is its job;
- what changed;
- what is missing;
- what is buried;
- what is implied but unnamed;
- what is misnamed;
- what is misplaced;
- what needs proof;
- what would make the fix real.

### ADOPT - Scale Selector

Small tasks should stay small. Boss tasks get boss loops. Repeated failure gets smoke break first.

This protects against both shallow summaries and ceremony creep.

### ADOPT - Fit Decision Tags

Use ADOPT / ADAPT / PARK / BLOCK / PROOF / LEAVE-BE for source ideas and broadened-list changes.

### ADOPT - Gate Firing Order

Order:

1. Smoke Break / correction trigger if active.
2. Task classification.
3. Object questions.
4. Surface selection.
5. Exact answer or bounded execution.
6. Patch-through only when boss-level.
7. Proof / Final Judge.
8. One teaching nudge if helpful.

### ADOPT - Broadening Final Judge

A broaden pass is not done when more items are added. It is done when old material was accounted for, missing ideas were hunted, dropped ideas were recovered, new items were tagged, surface/home was chosen, proof needs were named, and the next action is clear.
