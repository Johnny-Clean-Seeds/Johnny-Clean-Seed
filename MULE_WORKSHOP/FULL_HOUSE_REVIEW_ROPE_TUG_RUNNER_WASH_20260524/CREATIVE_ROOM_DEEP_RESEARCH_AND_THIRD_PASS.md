# Creative Room Deep Research And Third Pass

## Purpose

Take the cleaned house list into outside methods, argue against the house, then build a third shape
that fits without letting an outside framework become a new boss.

## Outside Side Against The House

Outside methods challenge the house in useful ways:

- If WIP is not limited, the rope becomes a net.
- If notes do not have addresses and links, "nothing is trash" becomes hoarding.
- If provenance is missing, clean summaries can hide source loss.
- If review cadence is missing, parked notes either never return or tug forever.
- If uncertainty domain is not named, the assistant may over-research clear work or over-certify
  complex work.
- If priority has no cost-of-delay lens, old/interesting notes can outrank live risk.

## House Side Against Outside Methods

The house pushes back:

- GTD "trash" is too risky as a default word here. Use rejected-as-active, blocked, source custody, or
  no-return release with proof.
- Kanban can become a board fetish if the house forgets source and final judge.
- C4 can become diagram theater if exact file/proof is available and enough.
- RICE/WSJF can create fake precision.
- Zettelkasten can collect beautiful notes with no active work.
- PARA can hide source proof if it routes only by project.
- Scrum/PDCA can become ceremony if the washer keeps running after the route is stable.

## Third Shape

The clean third shape is the Rope Control Stack:

`source -> note -> tug -> classify -> map -> decide -> place -> burn/cut -> runner trace -> wash -> return`

Where:

- source preserves raw truth;
- note attaches to the rope without becoming authority;
- tug fires when fog or movement risk appears;
- classify names object, intent, room, proof, authority, and disposition;
- map chooses context, room, object, or exact proof level;
- decide uses safety, user command, source custody, delay, impact, confidence, effort, and WIP;
- place sends it to rule, checklist, ledger, parking, receipt, source custody, TODO, or block;
- burn releases active note weight after placement;
- cut preserves a runner trace to root;
- wash repeats until two passes produce the same route;
- return leaves one active boss or a clean closeout.

## Second Creative Dive

Question:

What else can be dragged up after the third shape?

Answer:

The missing word is "runner," but not as a script. A runner is the trace object that runs backward
from quiet note to root truth.

That makes cutting safe:

`quiet note -> runner trace -> root/source -> owner rule -> final destination`

This solves the user's "all should link to all" request without building a full graph. The house
gets enough route memory to recover truth, while the active worker does not carry every note.

## Final Research Verdict

Adopt:

- WIP limits and feedback cadences;
- capture/clarify/organize/reflect/engage as note lifecycle;
- actionability as placement principle;
- atomic note plus structure pointer;
- provenance passport;
- C4-style zoom;
- Cynefin fog domain check;
- PDCA/Scrum-style inspect/adapt rhythm;
- RICE/WSJF as optional rough ranking only.

Park:

- full graph database;
- numeric note ranker;
- automated alerting;
- full board system;
- full ADR for every small support change.

Reject as active:

- deletion as "burning";
- time alone as loudness;
- outside methods as doctrine;
- scoring as proof;
- endless washer loops.
