# Similarity Matrix And Routine

Date: 2026-05-24

## Obvious Similarities

| Pattern | LocalBot | CleanMilkChecker | Current House |
| --- | --- | --- | --- |
| Small start | Tiny local runner loop | One supplied input to one verdict | One bounded work object |
| Gate before trust | Status/output checks | INVALID/YIELD and proof requirements | Gates, washer, proof ledgers |
| Failure is material | Timeout/repeat/recovery tasks | Missing evidence and unclear authority tests | Notes, TODOs, runners, parked lanes |
| Recovery is a stage | Recovery phase closes before practical use | Recovery/route proof before Stage 4 | Soft-suit replay and washer return |
| Close prevents drift | Final wrap parks leftovers | Final seed closeout parks automation | Anchor blocks broad doctrine rewrite |
| Human verification stays close | Local, inspectable, manual | Manual checker, not automation | Local files, hashes, receipts, Git |

## Model Tightening Lesson

The user's repeated tightening of the instant GPT assistant produced better structure over time, but also showed the danger: by the last run, the assistant became more rigid. That means the seed wrap cannot use one global clamp.

Clean ratio:

- early discovery: more freedom
- failure handling: more frame
- recovery: frame plus replay
- final wrap: high frame, low novelty
- creative room: deliberate contrast, then washer

The build should train each bot for the stage it is in, not force every bot to act like a final closeout clerk.

## Clean Bloat Pattern

Clean bloat is allowed inside a stage when it has:

- a source pointer
- a boundary
- a reason for volume
- a proof target
- a parking lane for extra material
- a condense step before active use

Dirty bloat is volume with no stage owner, no close condition, and no route change.

## New Routine: The History Echo Stage Card

Use this when old work needs to become current power.

1. Pick one history object.
2. Write its source paths and hashes.
3. Name its stage shape.
4. State what it proves now.
5. State what it does not prove now.
6. Translate it into one stage card.
7. Replay the house wearing only that condensed card.
8. Promote only the part that changes route.
9. Park the rest with a runner trace.
10. Burn active weight when the note stops changing the route.

## Washer Condense

First read:

History shows the same shape over and over.

Replay read:

The shape is not just "be organized." It is a build unit: source object, stage card, proof, recovery, closeout.

Outside challenge:

Imported methods agree with the house when translated cleanly: WIP limit, event log, traceability, gate review, cognitive load control.

Final route:

Make stage cards the working object for normal clean seed wraps. Keep raw history as source ore, not live load.

## Habit Loop

When a pass feels foggy:

rope tug -> source object -> stage card -> washer -> next move.

When a pass feels heavy:

note burn -> runner trace -> parked room -> active list shrink.

When a pass feels too rigid:

creative-room contrast -> disproof question -> replay -> washer.

When a pass feels too loose:

one object -> one proof target -> one close condition.
