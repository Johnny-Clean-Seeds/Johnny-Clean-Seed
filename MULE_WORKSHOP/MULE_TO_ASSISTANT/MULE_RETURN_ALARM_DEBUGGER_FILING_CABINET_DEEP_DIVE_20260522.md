## 4. Mission understanding

The cabinet must help Mr.Kleen catch real damage during messy growth without choking useful growth.

Real damage means:

1. Unproved material becomes authority.
2. The assistant says PASS, saved, clean, or done without proof.
3. Live instructions, source ore, mule output, parked material, or Other Noise enter the wrong artifact.
4. Source, proof, recovery, or future growth material is deleted, overwritten, hidden, or collapsed too early.
5. The wrong route or root is chosen and work proceeds anyway.
6. Rules become too narrow, too vague, falsely linear, or decorative.
7. Useful bloat is deleted before it can become dense growth, or dirty bloat is promoted.
8. Local scripts, copy blocks, PowerShell, Git, or bridge routes create false success or partial state.
9. Mule, bridge, helper, memory, chat, source ore, or checker output becomes authority by accident.
10. Alarms become ceremony that adds friction without changing behavior.

In the current phase, the system should block only real house damage. It should warn on route/wording/tool risk. It should watch useful bloat, source pressure, candidates, and future compression material.

## 5. Better alarm definition

A better alarm/debugger:

1. Prevents real damage.
2. Fires before damage when possible.
3. Names the actual failure family.
4. Creates the correct amount of friction for the current phase.
5. Gives a clear next action.
6. Preserves proof, source, and recovery.
7. Does not stall useful growth work.
8. Leaves evidence that it worked.

Additional test from this review:

An alarm is real only when it changes the action at the point of work. If it is only named after the user catches the issue, it is late. If it was available but did not fire before damage, it failed-to-fire. If it has no event evidence, it is decorative or unproved.

## 6. Severity model

BLOCKER:

Stops the move until handled. Use for authority drift, promotion, doctrine/ACTIVE_GUIDES/CURRENT_TRUTH_INDEX rewrite risk, false PASS, dirty save, wrong base, remote mismatch, proof loss, destructive action, source/lane corruption during durable writing, and helper output becoming authority.

WARNING:

Names risk and changes route without freezing all work. Use for route/root fog, wording quality, partial proof, script delivery risk, mule/bridge lifecycle mess, and artifacts that need revision before delivery.

WATCH:

Preserves useful mess without blocking growth. Use for useful bloat, source ore, candidate tools, Soft Suit material, unproven checker candidates, Whirlpool pressure, outside pattern candidates, and later-phase ideas.

META:

Controls the alarm system itself. Use for phase-weighting, friction calibration, firing proof, no-rule-king behavior, and alarm fatigue.

Status labels:

- WORKING: fired early and changed behavior.
- LATE: appeared after user or another check caught the issue.
- PARTIAL: helped, but not early or complete enough.
- DECORATIVE / NO PROOF: named but no behavior evidence.
- FAILED-TO-FIRE: should have stopped a real mistake and did not.
