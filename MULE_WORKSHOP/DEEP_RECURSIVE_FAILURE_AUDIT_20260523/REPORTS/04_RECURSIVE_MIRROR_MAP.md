# Recursive Mirror Map

Status: audit output / proposed live-use card candidate.

Purpose: show where a new rule, correction, tool, source, TODO, proof, or handoff must reflect before the assistant can claim clean closure.

## Mirror Chain

1. Active Boss
2. Parent Rule
3. Sibling Rules
4. Child Behaviors
5. TODO / Queue
6. Parking / Return Trigger
7. Delivery Surface
8. Local Path / Home / Porch
9. Source Custody
10. Mule Boundary
11. Proof / Receipt
12. Final Judge
13. Home Placement / Git Split

## Failure Maps

### Failure: Rule Saved But Not Fired

Existing rule/map that should have caught it:

- Rule-Firing Confirmation Before Action Gate.
- Rule Obedience Proof.
- Rule Checker Failure Prevention.
- Final Judge Gate.

Broken mirror edge:

Saved rule did not reflect into a required point-of-work firing card before action.

Needed fix:

Every durable action report starts with the fired gate and ends with Final Judge checking whether the gate actually fired.

### Failure: Clean Home Path Changed But Front Door Stayed Old

Existing rule/map that should have caught it:

- CURRENT_TRUTH_INDEX active-anchor split.
- Command Map Housekeeping.
- Home/path proof.
- Lifecycle sync gate.

Broken mirror edge:

Home placement did not reflect into CURRENT_TRUTH_INDEX, AGENTS, Command Center context cart, teleporter pointer, and save scripts.

Needed fix:

Home Path Sync candidate with scoped path replacements and proof, not broad rewrite.

### Failure: Work Shed References Without Active Work Shed

Existing rule/map that should have caught it:

- Work Shed support-only rule.
- No guessing placement.
- File load certification.
- No parking without fit decision.

Broken mirror edge:

Shed exclusion removed the actual lane from clean home, but status/TODO/proof references still point at it.

Needed fix:

Work Shed Reference Reconciliation: mark references historical/archive, point selected evidence to Documents archive, or create clean non-shed stubs if approved.

### Failure: TODO Board Exists But Does Not Always Control Re-entry

Existing rule/map that should have caught it:

- TODO First Work Control.
- TODO Proof-Ranked Weakest-Link Selection.
- Current Work Queue Loss Prevention.

Broken mirror edge:

TODO evidence did not consistently reflect into active boss selection at re-entry.

Needed fix:

Re-entry readback must name ACTIVE_ANCHOR, open TODOs, proof-ranked weakest link, and live-task override if user gives one.

### Failure: Parking Becomes Hidden Storage

Existing rule/map that should have caught it:

- No Parking Without Fit Decision.
- Current Work Queue states.
- Stop and Tool / Bookmark rule.

Broken mirror edge:

Parked state did not guarantee return-trigger scan.

Needed fix:

Parking ledger check: each parked object needs owner/lane/reason/return trigger; re-entry checks only relevant triggers, not every parked item.

### Failure: Delivery Fix Known But Not Adopted

Existing rule/map that should have caught it:

- Adopt / Adapt Found Fix Live Application.
- PowerShell copy delivery lint.
- Desktop is not a parking lane.

Broken mirror edge:

The discovered fix stayed as diagnosis instead of changing the live delivery route.

Needed fix:

When a delivery failure exposes a safer route, switch the current lane before sending another artifact.

### Failure: Local-Only Material Tracked In Git

Existing rule/map that should have caught it:

- Save Location Default Local + URL.
- Git Save / Push Default.
- No Parking Without Fit Decision.

Broken mirror edge:

The status label did not mirror into Git tracking state.

Needed fix:

Local-only tracking audit: decide keep tracked, untrack, relabel, or split receipt pointer from content.

### Failure: Proof Receipt Exists But PASS Is Too Broad

Existing rule/map that should have caught it:

- Done is not PASS.
- Final Judge Gate.
- Proof/PASS/Save Integrity alarms.

Broken mirror edge:

Receipt proved a bounded save, but the final closure sometimes overreads it as behavior change or whole-system proof.

Needed fix:

Final Judge must state proof scope, unproved behavior, parked follow-ups, and whether future live use is still required.
