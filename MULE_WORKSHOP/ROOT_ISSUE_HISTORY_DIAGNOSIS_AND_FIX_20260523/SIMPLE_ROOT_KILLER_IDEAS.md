# Simple Root Killer Ideas

Date: 2026-05-23

Status: candidate fixes / local-only until reviewed.

## Killer Idea 1: Lifecycle Before Action

Root killed:
Task success masking lifecycle failure.

Simple form:
No object acts until its lifecycle is named.

Exact fix:
Before action, write:
`object / current state / next state / proof / closeout / parked-installed-saved-returned`.

Where it belongs:
Candidate BRAIN_LEARNING support card or work-report header.

Moving forward:
From now on, I will name the object lifecycle before action and not treat one successful step as whole-object PASS.

Proof:
Use on the next runner, source intake, rule save, icon, or mule packet and show that it blocks or changes a move.

Safe local application:
Used in `GATED_DIAGNOSIS_NOTES.md` for the raw issue source, diagnosis packet, fix candidates, and Git state.
