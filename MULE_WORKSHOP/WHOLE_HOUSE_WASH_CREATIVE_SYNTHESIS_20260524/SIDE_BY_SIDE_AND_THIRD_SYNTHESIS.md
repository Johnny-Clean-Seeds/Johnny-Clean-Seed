## Third Synthesis: Evidence-Backed Placement Mesh

Clean third model:

`object -> provenance passport -> relation word -> unsafe-move check -> decision record -> placed lane -> proof -> quiet-down or return trigger`

Where:

- object means the file, note, rule, fix, receipt, TODO, backup, or source item being handled;
- provenance passport names source, activity, actor/tool, evidence, and derivation;
- relation word says whether the object is source, proof, checkpoint, neighbor, parking, blocker, or
  promotion candidate;
- unsafe-move check prevents missing, wrong, mistimed, or over-held moves;
- decision record captures adopt, adapt, park, local-only hold, reject-as-active, or block;
- placed lane is Git-safe build, local-only hold, parking, TODO, receipt, backup, or active guide;
- proof shows what was checked;
- quiet-down or return trigger prevents permanent loud notes.

## Second Creative-Room Dive

Question asked:

What else can be dragged up after the side-by-side?

Answer:

The missing piece is not another framework. It is the junction between dirty-tree save hygiene and
rope-note visibility.

If a dirty file exists, it is a note on the rope whether or not it was written as a note. It should
get louder only until it receives disposition:

- `ADD TO BUILD`
- `UPDATE TRACKED`
- `LOCAL-ONLY HOLD`
- `PARK WITH RETURN`
- `WRONG LANE MOVE`
- `BLOCKED / ASK`

After disposition, it quiets into its lane.

## Clean Third Rule

Dirty working-tree material is a rope note at Level 3 until classified.

It may become:

- Level 1 if it is safe parked context;
- Level 2 if it belongs as a support pointer;
- Level 3 if it must be considered before staging;
- Level 4 if it affects authority, proof, source custody, privacy, or path truth;
- Level 5 if staging it would leak sensitive/local-only material, create false PASS, or damage files.

Once classified and placed, it quiets.

This is the bridge that keeps the same dirty-tree mystery from repeating.
