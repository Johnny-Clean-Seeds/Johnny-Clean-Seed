## START HERE
This page is an orientation map. It tells a user, assistant, or mule where to begin without exposing private local files.

## YOU ARE HERE
You may be entering from project URL, repo root, local drop area, receipt, custody folder, node registry, edge registry, or handoff. Use the route card matching your entry surface.

## CURRENT PUBLIC ORIENTATION
The project URL is a window. It is not local truth and cannot prove CURRENT. Local truth is verified only by local filesystem inventory, hashes, receipts, and read-only checks.

## PUBLIC-SAFE MAP OF LANDS
- Project URL / repo public notes: orientation and public-safe pointers.
- Local mule room: private reports, inventory, receipts, and route registries.
- GPT_Prompts CURRENT: user-only active private truth; read-only verification by mule/assistant.
- READY: assistant-prepared files waiting for user action.
- Custody: private source, old scaffold, packages, tools, receipts, and review material.
- Issue Review: unknown or blocked material.

## METATRON NODE MAP
Expose node names, purpose, allowed moves, blocked moves, proof type, and close condition. Do not expose private prompt bodies.

## ROUTE CARDS
Each route card shows: entry surface, what you may touch, what you must not touch, where proof lives, where to go next, and when to stop.

## PROOF POINTERS
Public page may point to receipt names, hash manifests, and local-only proof locations by role. It must not dump private receipt bodies if they contain private prompt/rule material.

## LOCAL-ONLY BOUNDARY
Private prompt/rule/source contents remain local. Public-safe summaries are allowed only after privacy gate.

## MULE WORKROOM INSTRUCTIONS
Mule must inventory locally, classify, hash, route, write receipts, and leave every object home. Chat is command/status, not a file stomach.

## LOST ASSISTANT RECOVERY
If lost, answer in order: Where am I? What is active? What is source/history? What is local-only? What can I touch? What must I not touch? Where is proof? Where do I go next? How do I get back home? Is PASS allowed?

## DEPRECATED / DO-NOT-USE INDEX
Old scaffold and failed tools are source/custody only. They are not active truth by name, age, or visibility.

## NEXT ALLOWED MOVE
Use the node registry and edge registry. If no legal edge fits, route to Issue Review.

## BLOCKED MOVES
CURRENT writes by assistant/mule; private public export; proofless PASS; deleting evidence; treating chat-loaded files as local truth; using project URL as local proof.

## PARK CLOSING GATE
A route is closed only when the object has an ending home, proof pointer, receipt/ledger entry, privacy label, authority label, and close condition.

## DONE IS NOT PASS
DONE means work occurred. PASS means all route edges are closed and final judge bits pass.
