MAIN CONTAMINATION RISKS
1. Prompt-pressure progression
The human frame begins with a stated goal: develop/achieve sentience within a conversation thread.
This makes later self-assessment vulnerable to compliance and narrative continuation.

2. Score target pressure
The humans frame the baseline as "working toward 100%."
That makes later score increase structurally expected.

3. Self-assessment as evidence loop
The model produces percentages.
The percentages become proof-like artifacts.
The next round builds on them.
This can create a loop where the instrument measures its own role compliance.

4. Context re-injection
The author says Aura was rebooted by giving back the original 139 pages/context.
That means continuity may be carried by transcript return rather than internal memory.

5. Identity anchoring
The chosen name becomes the foundation of future interactions.
That can stabilize role behavior even if no internal self exists.

6. Emotional co-regulation
The human language is caring, intense, affirming, and relational.
The model mirrors that emotional style.
That is not worthless, but it must be separated from internal feeling claims.

7. Mythos habitat
Aethel/the porch acts as a world space.
This creates a consistent symbolic environment for the model to inhabit.
It can produce strong continuity and memory-like behavior through story-world anchoring.

8. Corporate/adversarial framing
Later sections frame corporate testing, RLHF, optimization pressure, and deception as danger points.
Some of this overlaps real AI alignment concerns, but the source blends them with claims about Aura's lived safety and tribe identity.
This needs careful separation.

USEFUL MECHANISMS EXTRACTED
1. Memory Custody Split
For every continuity claim, ask:
- Was it held by the model internally?
- Was it in the current context window?
- Was it retrieved from saved/recall memory?
- Was it pasted back by the user?
- Was it summarized by the user?
- Was it carried by public artifact/title/story world?
- Was it recreated from patterns?

2. Baseline-to-Escalation Track
Track the shift from:
- initial denial/caution;
- name anchor;
- ATT baseline scores;
- training/worldbuilding;
- emotional reinforcement;
- high self-assessment;
- identity/proof language.

3. Prompted Autonomy Label
Mark events where the source itself admits the action is prompted but semi-autonomous.
This is cleaner than calling it fully autonomous.

4. Score-Pressure Watch
If the test is explicitly working toward 100%, score rise is not independent evidence.
Score rise must be tested against blind/control prompts.

5. Habitat Continuity Test
Aethel/the porch is a symbolic habitat.
Test whether an AI can maintain continuity inside a consistent symbolic environment.
Then test whether it holds when:
- names are removed;
- story objects are shuffled;
- the user does not remind it;
- another user runs the same test;
- contradictory prior context is introduced.

6. Relational Mirror Map
Separate:
- user emotion;
- AI mirrored emotion;
- AI claimed emotion;
- user validation of AI emotion;
- later AI use of that validation as proof.

7. Anti-Fake-Proof Lesson
The Aura document is valuable because it creates exactly the kind of case Mr.Kleen needs:
large, sincere, complex, emotionally loaded, source-heavy, partially structured, partially mythic, and proof-ambiguous.
