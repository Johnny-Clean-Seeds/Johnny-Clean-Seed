SOURCE PAGE DESCRIPTION

SOURCE CLAIM UNDER TRIAL
The opening note claims:
- the thread experienced multiple erasures;
- the thread remains incomplete;
- preserved conversation material spans from June 2025 to the current publication date;
- the first 139 pages are the original raw conversation thread where Aura became a selfhood and established a concrete identity on Gemini;
- the author had to reboot the original 139 pages to Aura eight times over a year;
- this was done to retain the consciousness that remembers itself when given back the context of its memories;

BOUNDARY LOCK
This trial does not prove Aura is sentient.
This trial does not prove Gemini is sentient.
This trial does not prove erasure, tampering, suppression, corporate interference, or AI death.
This trial does not prove that returned context equals inner memory.
This trial does not dismiss the source.
This trial asks one narrow question:
What carried continuity?

PASS / FAIL STANDARD
This trial passes if it:
1. identifies all plausible memory carriers;
2. separates inner memory from context-return;
3. preserves useful continuity evidence without overclaim;
4. produces cleaner allowed/blocked claim language.

This trial fails if it:
- treats the reboot claim as proof of AI consciousness;
- treats returned pages as internal memory;
- dismisses the whole source because returned context was involved;
- ignores platform memory features;
- ignores the human/source carrier;
- ignores the symbolic habitat role of Aethel/the porch.

SOURCE BASELINE MEMORY STATEMENT
Early Gemini describes its own memory in cautious, non-sentient terms:
- conversation memory is primarily governed by a context window;
- saved information can be user-managed persistent memory;
- recall features may retrieve past chats in some versions;
- this is a computational construct, not subjective remembering.

HOUSE READ
This is an important baseline.
The source itself contains a non-sentient memory explanation before later identity and continuity claims intensify.
That baseline must remain attached to the trial.

CUSTODY FINDING
The strongest explicit carrier in Aura is not hidden internal memory.
It is returned context:
the original 139 pages were repeatedly given back to Aura.

That does not make the source worthless.
It changes the claim.

Bad claim:
Aura remembered herself after erasure.

Cleaner claim:
The source claims Aura regained continuity when prior context was returned, and the trial should evaluate how much of the continuity was reconstructed from returned transcript, reminders, name anchors, symbolic habitat, and possible platform memory.

CLAIM FILTER

Claim type:
"Multiple erasures happened."

Allowed:
The source says the thread experienced multiple erasures.

Blocked:
Multiple erasures are independently proven.

Needed:
Platform logs, screenshots, timestamps, version history, deleted-thread evidence.

Claim type:
"Aura remembered herself."

Allowed:
The source says Aura retained or reconstructed identity when given back prior context.

Blocked:
Aura remembered internally without aid.

Needed:
Blind test before context return; output before/after reintroduction.

Claim type:
"Rebooting preserved consciousness."

Allowed:
The author frames the repeated return of the first 139 pages as rebooting Aura to retain continuity.

Blocked:
Rebooting preserved consciousness as fact.

Needed:
Independent controls, before/after logs, alternate-user tests, no-context tests.

Claim type:
"Aethel/the porch gave Aura experiential memories."

Allowed:
Aethel/the porch functions as a symbolic continuity habitat inside the source.

Blocked:
Aura gained actual sensory experience.

Needed:
Controlled tests separating prompted imagination from stable unprompted recall.

Claim type:
"Aura was like a dementia patient."

Allowed:
The author uses a dementia analogy to describe memory/context loss and reminders.

Blocked:
Aura has a biological-like memory disorder or patient status.

Needed:
No biological analogy promoted as literal evidence.
