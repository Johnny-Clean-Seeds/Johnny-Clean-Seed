# Dreamforger / Suno Placement Note

Status: PLACEMENT RULE SUPPORT NOTE
Date: 20260519

## Core lesson

Clean depends on placement.

A gate should not decide good vs bad by whether the current house already has a room.

The better gate asks:
- Who are you?
- What kind of room do you need?
- What neighbors do you need?
- What tools do you need?
- What proof path do you need?
- Are you source, support, active doctrine, candidate, parked idea, guest, or tool?
- Does the house need to build a room before judging you?

## Host gate rule

No room does not mean bad milk.

No room means one of these:
- build a clean room,
- place in source ore,
- park with dignity and a return trigger,
- give the item companions/tools,
- or reject only if it violates core boundaries after review.

## Notes jail warning

Parking and notes can become a jail if material is trapped there without:
- purpose,
- status,
- neighbor links,
- review trigger,
- return path,
- or proof path.

A parked item should not be lonely, bored, or useless. If it is, the placement is probably wrong.

## Dreamforger / Suno application

Dreamforger / Suno material belongs in Source Ore first.

It may later connect to:
- Creative Autonomy Lab,
- Suno manual bridge,
- symbolic-output forensics,
- AI-continuity testing,
- collaborator-source review.

It does not become doctrine by entering Mr.Kleen.
