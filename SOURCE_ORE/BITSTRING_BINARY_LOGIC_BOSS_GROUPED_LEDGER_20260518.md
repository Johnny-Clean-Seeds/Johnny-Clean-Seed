## Core Boundary

Markdown owns meaning.

Authority still comes from the active command chain.

## Boss Groups

### Boss Group 1 — Meaning Compression

Use:
Candidate methods that turn long room/rule states into compact checkable signatures.

Risk:
Compression can hide meaning if the readable source is not preserved.

Fit Rule:
A bitstring may summarize or check a readable rule, but it must not replace the readable rule.

Status:
Parked.

### Boss Group 2 — Binary State Checks

Use:
Candidate methods for yes/no gate checks, pass/fail state, present/missing flags, and small proof signals.

Risk:
False confidence if a binary state is treated as full proof.

Fit Rule:
Binary checks can support proof, but cannot become proof alone unless the proof scope is exactly binary.

Status:
Parked.

### Boss Group 3 — Mask / Filter Logic

Use:
Candidate methods for filtering large issue piles, matching tags, selecting groups, or detecting missing fields.

Risk:
Mask rules can drop useful weak signals.

Fit Rule:
Mask logic must preserve source trail and allow review of rejected/parked items.

Status:
Parked.

### Boss Group 4 — Pattern / Drift Detection

Use:
Candidate methods for comparing old/new states, detecting repeated failures, repeated wording, duplicate commits, or structural drift.

Risk:
Pattern detection can over-call a problem.

Fit Rule:
Pattern signal triggers review; it does not authorize repair.

Status:
Parked.

### Boss Group 5 — Proof Encoding

Use:
Candidate methods for representing proof checks as compact fields, bit flags, or check vectors.

Risk:
Encoded proof can become unreadable ceremony.

Fit Rule:
Every encoded proof item must map back to readable proof text.

Status:
Parked.
