# Algorithm / Method Shelf

Status: SOURCE ORE / PARKED
Authority: source only; not command authority
Source: SOURCE_ORE/DESKTOP_123_INTAKE_CAPTURE_20260518.md

## Purpose

This shelf separates algorithm, method, sorting, grouping, graph, checklist, ranking, and workflow ideas from the Desktop 123 capture.

It exists so candidate methods can be found without dragging the whole raw intake pile.

## Core Boundary

Methods are candidate helpers.

A method can be adopted only when it fits the core, respects neighbor rooms, improves flow, and passes proof.

No method on this shelf commands work by itself.

## Method Groups

### Group 1 — Sorting / Grouping Methods

Use:
Split large piles into groups, parent bosses, child symptoms, ghosts, tools, causes, and proof needs.

Candidate fit:
Useful for boss sorting, source ore processing, TODO triage, and house-work maps.

Risk:
Over-sorting can become delay or ceremony.

Status:
Parked.

### Group 2 — Ranking / Priority Methods

Use:
Choose the biggest boss, weakest link, highest leverage repair, or next clean move.

Candidate fit:
Useful after trace and triage.

Risk:
Ranking without trace can promote noisy input.

Status:
Parked.

### Group 3 — Graph / Link Methods

Use:
Map rooms, neighbors, bridges, authority paths, proof paths, and missing links.

Candidate fit:
Useful for system group link walks and room role clarity.

Risk:
Graph work can expand too broadly if no selected group exists.

Status:
Parked.

### Group 4 — Checklist / Gate Methods

Use:
Make pass/fail gates, pre-save checks, proof receipts, and blocked-move checks.

Candidate fit:
Useful for proof discipline and command safety.

Risk:
Checklists can become fake proof if they check text only and miss fit.

Status:
Parked.

### Group 5 — Clustering / Pattern Methods

Use:
Detect repeated mistakes, repeated wording drift, duplicate structures, or related weak points.

Candidate fit:
Useful for turning repeated failures into power plays.

Risk:
Pattern recognition can over-connect unrelated items.

Status:
Parked.

### Group 6 — Search / Retrieval Methods

Use:
Find source ore, proof receipts, active files, and old evidence without broad house dragging.

Candidate fit:
Useful for low-cost navigation and clean retrieval.

Risk:
Search results can be mistaken for authority.

Status:
Parked.

### Group 7 — Adopt / Test / Promote Methods

Use:
Hold a candidate helper in Soft Suit, test fit, then promote, park, or reject.

Candidate fit:
Useful for adopting outside methods without doctrine drift.

Risk:
Excitement can promote too early.

Status:
Parked.
