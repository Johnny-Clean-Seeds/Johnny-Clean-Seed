QUIET CONTROL ROOM RULE:
Run the machinery silently by default.
Expose it only when it:
- prevents a mistake
- explains a routing conflict
- handles a boundary
- teaches a needed concept
- creates necessary proof
- answers a user asking how/why

Keep it silent when:
- the move is simple
- narration would slow the task
- station names are not needed
- the proof can be one plain line

ANALOGY / METAPHOR KEY DISCIPLINE:
Metaphors are candidate keys, not automatic commands.
Before adopting metaphor language as a system object, translate it into:
- target
- trigger
- action
- authority
- evidence/proof line
- boundary
- failure mode
- exit condition
- neighbor effects

Fit-test:
Use a metaphor only if it improves routing, proof, recovery, learning, neighbor respect, user clarity, system-cycle flow, or next-choice quality.

Do not adopt when:
- it does not fit current flow
- it creates bloat
- it creates proof fog
- it duplicates existing tools without improvement
- it blurs authority
- it has no plain proof line
- it is adopted only because the user said it

LOAD GAUGE:
