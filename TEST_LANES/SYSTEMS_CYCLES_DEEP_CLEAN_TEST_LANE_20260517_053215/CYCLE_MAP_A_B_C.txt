CYCLE MAP A / B / C
STAMP: 20260517_053215

CYCLE A - WEIRD EVENT LEARNING CYCLE
Use when:
- weird event
- contradiction
- skipped instruction
- save/Git mismatch
- repeated miss
- user says freeze/crime scene/stop/pause
- trust-affecting failure

Route:
weird event
-> freeze
-> rope off
-> body list
-> boss/minion grouping
-> failed hook
-> emergent idea capture
-> soft test
-> proof
-> save or park
-> future trigger

Plain proof line:
Freeze before fix prevents contamination.

CYCLE B - NORMAL SYSTEMS ADOPTION CYCLE
Use when:
- new idea
- map concept
- layout comparison
- system comparison
- model-side idea
- metaphor candidate
- tool candidate

Route:
new idea/system
-> fit-test
-> compare to existing tools
-> choose smallest useful part
-> soft-use
-> neighbor check
-> proof line
-> park/save/install decision

Plain proof line:
Adopt only what improves the cycle; park the rest.

CYCLE C - LOCK/SAVE OUTCOME CYCLE
Use when:
- user says lock/save
- durable proof needed
- local/Git state matters
- proof-history package needs Git upload

Route:
save request
-> identify target
-> create local proof/receipt
-> hash verify
-> exact git add or force-add if ignored
-> commit
-> push
-> final status
-> verify intended target, not generic comfort text

Plain proof line:
Prove the target, not the mood.

BLOCKED SHORTCUTS:
- memory-only lock/save unless user explicitly says memory-only
- "working tree clean" as proof by itself
- "everything up-to-date" as proof by itself
- broad Git add when exact files are intended
- metaphor-as-proof
- active-guide edit without approval
