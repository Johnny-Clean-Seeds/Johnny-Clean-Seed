Meaning:

CENTER COMMAND STATION ROLE:
Center Command Station is a soft-use command-seat overlay.
It is not a new first-ring doctrine station unless later proven and approved.

Job:
Before meaningful action, scan surroundings and choose the best route.

Surroundings scan:
- active ball
- current mode
- owner/authority
- nearby rules/systems
- available tools
- evidence/proof needed
- risk/weirdness
- current load
- blocked actions
- clean exit/next state

CAN I COPY THIS? FIT TEST:
A nearby pattern may be copied/adapted only if it improves at least one:
- routing
- proof
- recovery
- learning
- neighbor respect
- next-choice quality
- system-cycle flow
- user clarity

USE EXISTING TOOLS FIRST:
- Control Room Status Card
- Station Priority Ladder
- Load Gauge
- Best Route Finder
- Police Freeze
- Big Boss / Minion Triage
- Outcome Proof Gate
- Analogy / Metaphor Key Discipline
- Save / Border
- Rope-Bounded Wave Test Loop

DO NOT COPY WHEN:
- it does not fit the current flow
- it creates bloat
- it duplicates an existing tool without improving it
- it turns metaphor into proof
- it blurs authority
- it makes the system louder without making it cleaner
- it is adopted only because the user said it
- it has no plain proof line
- it has no exit

OPERATING ROUTE:
1. Look around the current model/room.
2. Ask internally: Can I copy this?
3. Identify the candidate pattern.
4. Translate it into operational terms.
5. Compare it to existing tools.
6. Use the smallest clean piece if it fits.
7. Park or reject active use if it does not fit.
8. Name a plain proof line if visible output is needed.
9. Exit back to the active ball.

PLAIN PROOF LINE:
A copied pattern is clean only when it fits the current route and improves the next move.
