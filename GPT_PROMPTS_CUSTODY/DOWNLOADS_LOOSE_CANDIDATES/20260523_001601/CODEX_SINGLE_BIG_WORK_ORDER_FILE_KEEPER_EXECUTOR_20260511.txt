LOAD-BEARING EVIDENCE-TO-ACTION CLUSTER
============================================================

Save or use this cluster only inside approved scope.

Status:
Proof-history candidate / working decision filter unless separately promoted.
Not active guide doctrine by this file.
Not current truth by this file.
Not cleanup authority.
Not deletion authority.
Not folder creation authority.
Not promotion authority.

Core sentence:
Evidence must help choose the next action, not merely accumulate.

Safety sentence:
Do not treat mission mutation as progress.

Purpose:
When many parked, candidate, or good-looking ideas exist, do not choose the next rule or action by vibe, recency, excitement, or “this sounds good.”
Build a bounded candidate field, map cause-effect branches, and weigh which control point carries the most load before installing, promoting, saving more structure, or continuing a complex run.

Why this matters:
The project no longer mainly suffers from missing ideas.
The live risk is choosing the wrong next idea, letting evidence become a pile, letting large doctrine fail to fire, or treating a changed mission as normal growth.

Operating method:
1. Name the active decision.
2. Name the current ball in hand.
3. Build a bounded candidate field.
4. For each candidate, map:
   - problem solved
   - if used now
   - if not used
   - pros
   - cons
   - dependencies
   - risk class
   - authority impact
   - evidence burden
   - bloat pressure
   - neighbor effects
   - stop or yield condition
   - what it prevents
5. Compare candidates by load-bearing value.
6. Choose the smallest clean next action.
7. Park, block, or yield the rest with reason.
8. Do not promote anything unless a focused review proves it belongs.

Mission Mutation Gate:
Before accepting a next step as normal growth, check whether the object, mission, risk class, authority class, evidence burden, allowed tools, or out-of-bounds list changed.
If any changed, stop and reclassify.
Do not treat the changed path as simple progress.

Clean state:
The next action is selected from evidence, cause-effect mapping, and bounded comparison.
The active ball, next gate, authority, and risk class are visible.
Useful ideas are preserved without becoming active by identity.

Dirty state:
The worker gathers evidence but never turns it into a decision.
The worker picks a rule because it sounds good.
Candidate rows become another shed.
Too many rules are installed at once and cause-effect proof is lost.
A mission change is treated as normal growth.

PARK / REST / BLOCK:
Hold material that is useful, accepted-but-not-ready, or unsafe to use.

CONTROLLED DRIFT CAPTURE:
Record drift as source material without letting it become active truth.
