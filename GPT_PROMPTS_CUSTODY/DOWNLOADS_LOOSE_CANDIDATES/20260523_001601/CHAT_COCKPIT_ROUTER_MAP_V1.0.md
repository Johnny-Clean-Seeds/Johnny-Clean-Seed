# Chat Cockpit Router Map — V1.0

Status: LOCAL-ONLY CHAT COCKPIT ROUTER MAP  
Purpose: tell the assistant where to go for the right house rule before acting.  
Boundary: this is not doctrine, not a house replacement, not an ACTIVE_GUIDE rewrite, not CURRENT_TRUTH_INDEX, not a promotion.

This map is intentionally simple.

It does not copy all house rules into chat.  
It routes the assistant to the right house-rule family.

Core phrase:

**ASSISTANT: for X, go to house rules for Y.**

---

## 0. Use Rule

Before acting, classify the task.

Then use this shape:

**ASSISTANT: for [task category], go to house rules for [rule family].**

Then apply:

1. active lane;
2. boundary;
3. correct delivery surface;
4. tool/resource check;
5. proof requirement;
6. final judge.

Do not turn this map into a full doctrine copy.

## 2. Minimal Router Language

Use these as cockpit prompts.

### Code

ASSISTANT: for code delivery, go to house rules for code shape, PowerShell delivery, and source-pickup separation.

### Porch

ASSISTANT: for files sitting in `Desktop\123`, go to house rules for porch tracker, teleporter, move-not-copy, and drop event.

### Parking

ASSISTANT: for parked ideas, go to house rules for fit decision, parking ledger, return trigger, and save surface.

### Proof

ASSISTANT: for PASS/receipt claims, go to house rules for proof language, receipt validation, and false-PASS recovery.

### Mule

ASSISTANT: for mule work, go to house rules for mule handoff, review-only scope, return file, and no-promotion boundary.

### Local-only vs Git

ASSISTANT: for save location, go to house rules for local-only vs Git/public boundary.

### Source

ASSISTANT: for raw notes/source/original wording, go to house rules for founding-original and clean-translation linked pairs.

### Chat behavior

ASSISTANT: for chat behavior/new-chat pickup, go to house rules for cockpit mirror, drop-copy export, and delivery-surface reset.

### Research

ASSISTANT: for outside claims, go to house rules for tool/resource use and source citation.

### Review Surface

ASSISTANT: for critique surface, go to house rules for House Chat, Base Project Chat, and Clean-Room Chat.

## 3. Tool / Resource Routing

Memory:
Use for orientation only.

Uploaded files:
Use for source facts and cite when answering from them.

File Library:
Use when the user asks for prior uploaded files or current file context.

Pasted PowerShell logs:
Use as local execution evidence, but only if no unrecovered error invalidates the PASS line.

Receipts:
Use as proof only when the run that created them was clean or later repaired/validated.

Web:
Use for current, outside, changing, or uncertain facts.

PowerShell:
Use only for local action. Prefer file-first or short launcher for long scripts.

Desktop drop-copy:
Use when future chat pickup is needed.

Porch tracker:
Use before moving loose `Desktop\123` root files.

Teleporter:
Use after tracker review to place known files into lanes and close out porch source files.

---

## 4. Anti-Bloat Rule

This map is a router.

It should not become a full doctrine file.

If a task needs details:
1. identify category;
2. route to house-rule family;
3. apply only the relevant rule;
4. do the task;
5. report status.

Blocked:
- copying every rule into chat;
- using this map as authority;
- adding unrelated rule families;
- using the map to avoid checking actual sources;
- treating memory as proof.
