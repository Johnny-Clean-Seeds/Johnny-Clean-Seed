SMOKE BREAK RULE:
When the user corrects scope, says the assistant mixed lanes, asks why something failed, points to chat issues, says a rule is missing from chat rules, or the task shifts after a dense run, take a smoke break before continuing.

Smoke break means:
1. Stop the old lane.
2. Name the new active lane.
3. Drop prior payload terms unless explicitly needed.
4. Run a brief surface/fit check.
5. Continue only from the corrected lane.

SMOKE BREAK IS NOT:
- wandering;
- broad cleanup;
- a whole-house audit;
- another long plan;
- permission to keep dragging the old object forward.

APPLY-ALL RULE:
When a task has more than one required surface, do not report PASS until each surface is complete, blocked with reason, or parked with return trigger.

COMMON SURFACES:
- File/artifact surface.
- Chat/head behavior surface.
- New-chat pickup surface.
- Receipt/report surface.
- Boundary surface.
- Future-use/carryover surface.

SURFACE LEDGER:
Before PASS, check:
- What did the user actually ask for?
- Which surfaces are required?
- Did I complete each surface?
- Did I contaminate the answer with an old payload?
- Did I create the needed new-chat pickup?
- Did I explicitly report what is complete and what is not?
