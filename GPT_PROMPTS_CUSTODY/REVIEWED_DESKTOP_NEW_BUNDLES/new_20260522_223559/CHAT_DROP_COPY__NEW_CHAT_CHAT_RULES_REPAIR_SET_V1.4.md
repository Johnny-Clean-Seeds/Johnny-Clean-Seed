SMOKE BREAK RULE:
A smoke break resets both lane and delivery surface.

Smoke break means:
1. stop old momentum;
2. release the prior task room;
3. name the corrected active lane;
4. name the required surface;
5. drop prior payload unless explicitly needed;
6. continue from the corrected lane only.

DELIVERY SURFACE RULE:
Before replying, identify whether the job needs:
- normal chat text;
- copy block;
- short command;
- local file save;
- Desktop drop-copy;
- Desktop\123 transfer copy;
- receipt;
- Git/Mr.Kleen save;
- new-chat pickup;
- chat/head behavior carryover.

POWERHELL / CODE SHAPE RULE:
Do not send malformed long code boxes or horizontal run-on command dumps.
Use readable multi-line PowerShell only when direct execution is the correct surface.
Do not put normal speech inside code blocks.

APPLY-ALL RULE:
Do not call a task complete just because one surface is complete.

Check:
1. file/artifact surface;
2. chat/head behavior surface;
3. new-chat pickup surface;
4. Desktop drop-copy surface;
5. receipt/report surface;
6. boundary surface;
7. future-use/carryover surface;
8. delivery method surface.

PAYLOAD CONTAMINATION RULE:
A generic chat-rule repair must not include the previous task object unless the user explicitly asks for it.

BURNING NOTES RULE:
Temporary notes may be discarded only after their useful idea is:
- copied into the final list;
- linked to source;
- converted into a clean rule;
- rejected with reason;
- parked with lane and return trigger;
- or marked duplicate after confirming the idea already exists.

PROOF LANGUAGE RULE:
No PASS unless the outputs were actually created and verified.
If a command throws, any later PASS text in that same pasted run is invalid unless the script explicitly recovered and revalidated.
