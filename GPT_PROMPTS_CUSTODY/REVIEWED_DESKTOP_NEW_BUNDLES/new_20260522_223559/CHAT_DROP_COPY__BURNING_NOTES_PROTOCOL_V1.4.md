# Burning Notes Protocol — V1.4

STATUS: LOCAL-ONLY CHAT SUPPORT RULE

PURPOSE:
Temporary notes may be discarded only after their useful idea is safely captured.

BOUNDARY:
- This is local-only chat support.
- This is not Git.
- This is not public.
- This is not doctrine.
- This does not authorize deleting project files.

WHAT "BURN NOTES" MEANS:
Burning notes means discarding temporary scratch after capture. It does not mean deleting source files, receipts, evidence, originals, or project authority.

RULE:
A note can be burned only after one of these is true:

1. Copied into the final list.
2. Linked to a source.
3. Converted into a clean rule.
4. Rejected with reason.
5. Parked with lane and return trigger.
6. Marked duplicate after confirming the idea already exists.

DO NOT BURN:
- founding originals;
- source files;
- receipts;
- proof material;
- user corrections;
- unprocessed notes;
- unclear notes;
- notes with unresolved fit decisions.

BURNING CHECK:
Before discarding scratch, ask:
1. What useful idea is in this note?
2. Where is that idea now captured?
3. Is the source/original preserved or linked?
4. Is the note rejected, duplicate, parked, or copied?
5. Would losing this note lose meaning?

If yes, do not burn.
If no, burn the scratch only.

SHORT FORM:
Copy the idea first. Preserve source when needed. Then burn only the scratch.
