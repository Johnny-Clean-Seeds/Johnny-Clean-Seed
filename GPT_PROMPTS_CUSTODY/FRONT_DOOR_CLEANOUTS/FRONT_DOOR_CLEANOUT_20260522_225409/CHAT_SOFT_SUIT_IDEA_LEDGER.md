# CHAT_SOFT_SUIT_IDEA_LEDGER

STATUS: LOCAL-ONLY IDEA LEDGER / BROAD IDEA FIELD / SUPPORT SURFACE / NOT ACTIVE RULE AUTHORITY / NOT GIT / NOT PUBLIC / NOT DOCTRINE

ROLE:
Hold broad, raw, partial, repeated, or future ideas so the active map stays clean.

COMPANION:
`CHAT_SOFT_SUIT_MAP.md` is the active distilled working-order map.

CORE SPLIT:
- Ledger holds ideas.
- Map holds active distilled rules.
- Custody holds bulky source/support/old scaffold.
- READY holds waiting updated files.
- CURRENT holds user-approved current files.

NONNEGOTIABLE:
An idea is not active Chat Soft Suit law until it is distilled into the map and made current.

---

## 1. What Counts as an Idea

An idea can be a correction, complaint, metaphor, warning, workflow change, file-lane change, node-route change, proof method, boundary, bell/user-action rule, house-wide pattern, source-only note, failed-tool lesson, emotional/feel signal, “this smells wrong” signal, future project, partial thought, old scaffold fragment, chat-weight problem, fog signal, or let-go algorithm.

This ledger can be broad. The map must stay distilled.

---

## 2. Idea States

RAW, TRIAGED, ADOPT, ADAPT, PARK, BLOCK, WATCH, MERGED, PROMOTED_TO_MAP, ARCHIVED, RELEASED_TO_POINTER.

---

## 3. Idea Types

RULE_CANDIDATE, METAPHOR, WARNING, WORKFLOW, NODE_ROUTE, PROOF, BOUNDARY, BELL, SOURCE, HOUSE_WIDE_PATTERN, TOOLING, WEIGHT_RELEASE, FOG_SIGNAL.

---

## 4. Links Ladder for Ideas

Source Link: where did it come from?
Home Link: where does it belong now?
Neighbor Link: which Suit/node handles it next?
Proof Link: what proves its state?
Return-Home Link: what closes the loop?
Bell/User-Action Link: does the user need to act?

Short form:
`source -> home -> neighbor -> proof -> return-home -> bell if needed`

---

## 5. Broad Condense / Dissect Method

When many ideas are sitting in chat:

1. COLLECT visible ideas.
2. ATOMIZE compound ideas into single points.
3. NAME each point.
4. TYPE each point.
5. STATE each point.
6. APPLY LINKS LADDER.
7. CLUSTER related ideas.
8. DECIDE promote, merge, park, block, watch, archive, or release to pointer.
9. MAP only the distilled rule.
10. RETURN HOME with state and location.

Do not create a new active rule file to avoid thinking.

---

## 6. Promotion Gate

Before promotion to map:
1. What problem does this solve?
2. Is it already covered?
3. What type is it?
4. Which Suit/node does it affect?
5. Which section of the map changes?
6. Can it be stated cleanly?
7. What gets removed/merged/shortened?
8. What proof or behavior change supports it?
9. What is its return-home state?
10. Is user action needed?

If weak, keep it here.

---

## 7. Chat Weight / Fog Idea Handling

Some ideas are not project content. They are operating-weight signals.

When the user says chat feels foggy, old, heavy, repetitive, saved already, or annoying:
- classify as WEIGHT_RELEASE or FOG_SIGNAL;
- check whether the material is already locked in a file/receipt/custody lane;
- if locked and not active, release full old weight;
- keep pointer/status only;
- ring fog bell if active decision quality is degrading.

Fog bell:
`BELL: chat fog is rising. I am dropping old saved weight and using active map + ledger + current proof only unless source recovery is needed.`

## 13. Bell Rules

Ring bell when ledger has many RAW ideas, a repeated idea appears, an idea is ready to promote, map update is in READY but not CURRENT, idea lacks source link, idea lacks return-home state, user action is needed, assistant starts making new active files instead of using map/ledger, or chat fog rises.

Bell wording:
`BELL: idea backlog needs triage before more rule work.`
`BELL: this idea is promoted in READY but not CURRENT.`
`BELL: this belongs in the ledger, not a new active rule file.`
`BELL: chat fog is rising. I am dropping old saved weight and using active map + ledger + current proof only unless source recovery is needed.`

---

## 14. Cleanup Rule

Capture. ID. Type. State. Links Ladder. Cluster. Decide. Promote only distilled rule. Merge duplicates. Archive source. Release locked old weight. Bell if user action/fog.

Do not create a new active rule file to avoid thinking.

---

## 15. Final Judge

A clean idea system means no idea is lost, no idea becomes a rule by accident, broad material has a place, map stays usable, source is preserved, old saved weight gets released, user action is clear, and every route returns home.

Final short form:
`Capture broad. Distill narrow. Promote intentionally. Hold active only. Release locked old weight. Fog gets a bell. Keep one map. Keep one ledger.`
