PURPOSE

This file labels the public GitHub house so a new reader or assistant does not mistake proof volume, test lanes, source-ore, or root pointer files for active authority.

WHAT THIS FILE IS

- A public room map.
- A front-door companion.
- A role label for the main public folders and files.
- A placement aid for public readers.

WHAT THIS FILE IS NOT

- Not an active guide body.
- Not a replacement for CURRENT_TRUTH_INDEX.txt.
- Not a replacement for ACTIVE_GUIDES.
- Not permission to promote, delete, move, overwrite, or call PASS.

HOUSE SHAPE

ROOT

- Public entry surface.
- May contain pointer/reference files that route readers to active material.
- Must not be treated as final authority without checking CURRENT_TRUTH_INDEX.txt and ACTIVE_GUIDES/.

CURRENT_TRUTH_INDEX.txt

- Orientation ledger.
- Names current active packet, local truth path, support proof roles, installed updates, and next clean action.
- Public readers must distinguish local path facts from public repo commands.

ACTIVE_GUIDES/

- Active guide packet.
- Intended active files:
  - README_START_HERE.txt
  - CLEAN_SEED_WRAP_GUIDE.txt
  - CLEAN_SEED_ALIGNMENT_CHECK.txt

AGENTS.md

- Local file-lane / Codex guidance.
- Not a fourth active guide.
- Cannot call final PASS or promote material into active truth.

PROOF_HISTORY/

- Evidence, receipts, backups, reports, and audit trail.
- Not active doctrine by volume.
- Use named proof only for the named scope.

PUBLIC_NOTES/

- Public notes, handoffs, route selectors, test packets, support cards, and public-facing explanation material.
- Not active doctrine unless deliberately promoted.

TEST_LANES/

- Experiments and proofs under test scope.
- Not active doctrine by existence.

Shed/

- Source-ore, parked material, older experiments, fallback material, and reference stock.
- Mine carefully.
- Do not install broadly.

ROOT POINTER RULE

If a guide file appears both at root and in ACTIVE_GUIDES/, the ACTIVE_GUIDES copy is the active packet copy unless CURRENT_TRUTH_INDEX.txt proves a different current authority.

Root guide files are public pointer/reference surfaces.

They route readers to the active guide packet.

They are not mirror doctrine bodies.

WORD-KEY RULE

Words are keys.

Use the most accurate word for the role:

- Active guide: can guide behavior inside its lane.
- Pointer: routes to another file.
- Reference: supports understanding.
- Evidence: supports a claim but does not command by volume.
- Source-ore: parked material that may be mined later.
- Test lane: controlled experiment area.
- Public note: support material unless deliberately promoted.

For more wording support, see:

PUBLIC_NOTES/WORD_KEYS_AND_PUBLIC_LANGUAGE_NOTES_001.txt

PASS CONDITIONS FOR THIS FIX

PASS only if:

1. A public front door exists at root.
2. The front door sends readers to CURRENT_TRUTH_INDEX.txt and ACTIVE_GUIDES/ first.
3. PROOF_HISTORY is labeled as evidence only.
4. PUBLIC_NOTES, TEST_LANES, and Shed are labeled non-authority by default.
5. Root guide files are labeled pointer/reference unless proven current.
6. Word-key support exists without replacing ACTIVE_GUIDES.
7. No active guide content is rewritten by this fix.
8. No deletion, move, cleanup, or broad proof-history reshuffle occurs.

FAIL CONDITIONS

FAIL if:

- The fix creates another competing active guide.
- The fix rewrites active doctrine without approval.
- The fix deletes, moves, or hides evidence.
- The fix claims public GitHub is the private local source of truth.
- The fix treats proof-history volume as authority.
- The word-key note becomes a hidden override.

CHECKER NOTE

The public router does not require root guide files to hash-match ACTIVE_GUIDES copies.

If they differ, that is not automatic failure.

The correct question is whether the root file is a pointer/reference surface and whether ACTIVE_GUIDES remains the named authority packet.
