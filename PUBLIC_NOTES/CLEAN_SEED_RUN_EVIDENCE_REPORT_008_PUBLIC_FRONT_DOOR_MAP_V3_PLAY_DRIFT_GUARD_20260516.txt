WHY THIS MATTERS FOR CREATIVE BUILDERS
Fast-association builders use jokes, metaphors, nicknames, and strange handles to find ideas.

That is useful.
That is also risky.

The same spark that helps discover a clean handle can also pull the project sideways if the assistant treats every playful phrase as a build signal.

This is a real threat for people who think in fast metaphors, because:
- jokes can look like commands
- names can look like doctrine
- roleplay can look like scope change
- excitement can look like approval
- a funny phrase can accidentally become a file, rule, report, or mission

PLAY-DRIFT GUARD
When play enters an active project lane, use this guard.

1. Treat joke language as play by default.
2. Do not create rules, files, reports, commands, or doctrine from jokes.
3. If a joke appears useful, park it as a handle only.
4. Before building from playful language, require a direct build instruction.
5. If play lasts more than a short exchange, name the active task again.
6. If the assistant starts making a rule list from play, stop and classify it INVALID PLAY-DRIFT.
7. Resume from the board, not from the joke.

SHORT FORM
Play is allowed.
Play is not authority.
Spark is not scope.
Joke is not command.
Metaphor is not doctrine.
Return to the board before building.
