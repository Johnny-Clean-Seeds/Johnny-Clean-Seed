MAP / BRIDGE / LINK / JACK / LADDER STARTER DEFINITIONS

MAP

A layout that shows rooms, lanes, support notes, and placement.

BRIDGE

An approved connection between lanes or systems.

A bridge says movement is allowed when the gate is met.

LINK

A direct pointer to a file, note, rule, proof, tool, or next action.

A link prevents vague routing.

JACK

A temporary lift used to inspect something stuck or unclear without moving the whole house.

A jack helps examine a rule, file, note, or idea before deciding placement.

LADDER

A controlled promotion path.

A ladder prevents support notes from sneaking into active doctrine.

It defines how something can move upward only through proof and authority.

CHECKBOARD

A visible board that tracks open work, status, lane, proof, blockers, and next move.

PARKING LOT

A clean holding lane for useful ideas that are not ready for current action.

HARD RULES AS GROWTH RAILS

Hard rules are not cages.

Hard rules are rails that keep growth clean while the house learns better movement.

The assistant must not use rules as an excuse to stop learning.

The assistant also must not use learning as an excuse to violate the model.

DISCOVERY METHODS

Deep scan is one discovery method, not the only discovery method.

The assistant should use the lightest clean method that can expose the leak.

DEEP SCAN

Use when the issue may be spread across files, notes, lanes, proof, history, or placement.

Deep scan should look for:
- inconsistencies,
- oddities,
- missing context,
- missing files,
- missing ideas,
- redundant notes,
- duplicated garble,
- stale wording,
- disconnected links,
- missing companions,
- weak placement,
- unclosed lock/save paths.

COMPARE / CONTRAST

Use when two things seem similar but behave differently.

Ask:
- What is the same?
- What is different?
- What repeated?
- What is missing from one side?
- Which side carries cleaner proof?
- Which side better fits the model?

ANALOGY AS HYPOTHESIS

Use when the world offers a useful pattern.

Ask:
- What is this like?
- Where does the analogy fit?
- Where does the analogy break?
- What would prove it helps?
- Does it bless the model or distort it?

Do not treat analogy as truth.

Treat analogy as a candidate pattern.

PATTERN RECOGNITION

Use when repeated shapes appear.

Look for:
- same failure three times,
- same leak under different symptoms,
- same command mistake,
- same checker failure shape,
- same placement confusion,
- same user correction,
- same word drift,
- same missing companion.

TRIAL / ERROR WITH PROOF

Use when the path is uncertain but testable.

Rules:
- test small,
- capture result,
- do not overgeneralize,
- convert failure into a power play,
- stop if fog increases.

GRAPH / LINK THINKING

Treat files, notes, rules, lanes, tools, and tasks as nodes.

Look for:
- missing links,
- broken bridges,
- isolated notes,
- overloaded hubs,
- circular routes,
- unowned items,
- support notes acting like doctrine,
- tools with no route to proof.

CLUSTERING

Group related notes, ideas, issues, or failures into families.

Use when there are many pieces and the assistant is losing the shape.

Cluster by:
- lane,
- trigger,
- failure type,
- word key,
- proof need,
- lock/save state,
- authority level,
- repeated pattern.

RANKING / SCORING

Use when many weak links compete.

Score by:
- recurrence,
- risk,
- authority impact,
- proof impact,
- source-truth impact,
- neighbor damage risk,
- ease of repair,
- value gained.

Prefer high-impact, low-damage, clear-proof repairs first.

BIT / STRING / TOKEN PATTERNS

Use for code, checkers, filenames, wording, and exact drift.

Look for:
- corrupted tokens,
- stale expected strings,
- naming drift,
- repeated fragments,
- mismatched case,
- bad punctuation,
- broken paths,
- unsafe backticks,
- self-bite phrases,
- copy/paste contamination.

TRIAGE

Use now / next / later / parked / blocked.

Do not treat every idea as urgent.

NOW: strengthens the active gate.
NEXT: clearly useful but belongs after current lock/save.
LATER: useful but not ready.
PARKED: preserved for future mining.
BLOCKED: violates authority, proof, source truth, or neighbor policy.
