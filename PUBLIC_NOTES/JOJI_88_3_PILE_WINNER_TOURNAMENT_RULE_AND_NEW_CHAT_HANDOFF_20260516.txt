NEW TOURNAMENT PATTERN
Name:
3-Pile Winner Tournament

Core idea:
Do not eliminate the whole list one tiny cut at a time.

Instead:
- split the field into 4 judging groups for fairness and manageability
- run the field three times
- set winners aside each time
- then run Winner Group 1 vs Winner Group 2 vs Winner Group 3

This creates a final showdown of:
6 v 6 v 6

FIELD GROUPING
Use 4 judging groups to make the list easier to judge.

Suggested groups:
1. Route / Order / Decision
2. Proof / Custody / Memory
3. Risk / Failure / Recovery
4. Growth / Option / Public / Special

The exact groups can be adjusted if needed for balance, but do not let one group become all weak public tools or all strong proof tools.

WAVE STRUCTURE

WAVE 1
Run the full current survivor field.
Pick 6 winners.
Set them aside as Winner Group 1.

WAVE 2
Run everything that did not make Winner Group 1.
Pick 6 winners.
Set them aside as Winner Group 2.

WAVE 3
Run everything that did not make Winner Group 1 or Winner Group 2.
Pick 6 winners.
Set them aside as Winner Group 3.

FINAL SHOWDOWN
Run:
Winner Group 1 vs Winner Group 2 vs Winner Group 3

That means:
6 v 6 v 6 = 18 finalists

FINAL RUNS
Final Run 1:
18 finalists -> 9 finalists

Final Run 2:
9 finalists -> 6 champions

Optional Final Run 3:
6 champions -> 3 kings

Optional Final Run 3 requires explicit user approval. Default final output is 6 champions.
