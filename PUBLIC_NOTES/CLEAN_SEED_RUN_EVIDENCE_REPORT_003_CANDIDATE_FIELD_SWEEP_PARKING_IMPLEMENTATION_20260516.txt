DUPLICATES / OVERLAPS FOUND
- Intake Buffer and Candidate Carding Gate are separate. Buffer catches raw influx; carding converts it.
- Parking Control Board and Active Working Core are separate. Core is what is active; board displays it.
- Board Refresh Rule and Run Report Ledger are separate. Ledger stores report history; refresh updates current view.
- Public/OpenAI Lot and Public-Safe Gate are separate. Lot holds candidate public material; gate approves public movement.
- Cleanup Waiting Lot and Cleanup Decision Map Gate are separate. Waiting holds possible targets; decision map authorizes exact plan.
- Bridge Map and Neighbor Probe are separate. Probe decides which bridges are needed; map records the selected bridges.
- Dropout Classifier and Missing-Idea Harvester are separate. Classifier diagnoses route failure; harvester extracts missing candidates.

MISSING ITEMS DISCOVERED IN RUN 003
1. Current Board Snapshot
A small current-state board needs to exist after each run. It should show current priority, current core, latest reports, visible lots, blocked lanes, and next gate.

2. Report Ledger Row Format
The Run Report Ledger needs a simple row format:
Report name, run question, verdict, latest state, next action, public/private status, hash/path if needed.

3. Board Staleness Check
Before using the board, check whether it matches the latest report. If not, refresh it.

4. Field-Lock Exception Lane
If a high-value outside idea enters during a locked run, it needs a place: "outside field / parked for next run," not ignored and not activated.

5. Proof Pull Rule
When the board needs proof, pull only the smallest proof item needed. Do not pull the whole proof pile.

6. Publish Visibility Marker
If a report/note is published to GitHub, the board or ledger should mark it as public-visible vs local-only.

7. Fake-PASS Block
If a script errors, later PASS lines are invalid unless the script stops. This belongs in Worker/Tool Lot and Public-Safe Gate.
