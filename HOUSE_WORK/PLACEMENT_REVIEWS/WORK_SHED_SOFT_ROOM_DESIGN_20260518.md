# Work Shed Soft Room Design

Status: SOFT PLACEMENT DESIGN
Authority: support review only; not command authority

## Active Need

The assistant needs a safe place to receive large messy idea loads and work through them without contaminating active authority, proof, source ore, or learning rules.

## Bells Triggered

- room structure
- source/support/authority boundary
- idea storage
- recursive adoption
- user-flow
- word-key clarity
- future behavior

## Selected Shape

Create a soft Work Shed under HOUSE_WORK.

Path:

HOUSE_WORK/WORK_SHED/

## Why This Is Better Than Loose Top-Level Placement

Top-level placement would make the shed look too important.

SOURCE_ORE placement would make it look like raw source only.

BRAIN_LEARNING placement would make it look like behavior doctrine.

HOUSE_WORK placement fits best because this is a work-control support room.

## Why This Is Soft

This creates signs and empty drawers.

It does not move existing files.

It does not promote ideas.

It does not rewrite authority.

It does not change active guides.

## Internal Rooms

INBOX:
mess lands here.

SORTING_BENCH:
mess gets grouped and compared here.

IDEA_CARDS:
one idea per card after sorting.

FRONT_SHELF:
hot/current helpers stay close.

PARKED_METHODS:
good but inactive helpers wait here.

OLD_WAYS:
old/superseded methods stay retrievable.

INDEXES:
maps, tags, retrieval paths, and cabinet signs.

## Fit Test

The Work Shed is good if it helps the assistant:

- capture more without losing order
- sort before deciding
- keep old ways retrievable
- keep hot ideas near the front
- avoid polluting authority rooms
- move faster with less fog
- support recursive reuse across the model

The Work Shed is bad if it becomes:

- a second SOURCE_ORE
- a second PROOF_HISTORY
- a hidden command room
- a trash pile
- a doctrine shortcut
- another hallway of loose ideas

## Current Judgment

Create the room softly.

Use it.

Feel it.

Revise only after use shows what it needs.
