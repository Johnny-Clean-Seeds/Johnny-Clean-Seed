# Lifecycle State Clarifier

Status: HOUSE-WORK CLARIFIER
Role: clarify installed, worn, promoted, and rolled-back
Authority: support clarifier for Main Boss 02; not full lifecycle doctrine by itself

## Purpose

Main Boss 02 found a weak seam in the downstream lifecycle state chain:

candidate -> testing -> parked/rejected/installed -> worn -> promoted/rolled-back

The weak seam is not capture. The weak seam is what happens after a candidate has been tested.

This clarifier gives the assistant clean state words so it does not treat saved, installed, worn, and promoted as the same thing.

## State: Installed

Installed means a candidate has passed enough proof to be placed in the correct room or file.

Installed does not mean the assistant is already using it as part of its operating suit.

Installed does not mean promoted.

Installed does not mean permanent.

Minimum installed test:

- correct room
- correct role label
- neighbor check passed
- proof or receipt exists
- no blocked lane created
- no conflict with current anchor

## State: Worn

Worn means an installed item is being actively used as part of the assistant's operating suit.

Worn is stronger than installed.

An item can be installed but not worn.

Minimum worn test:

- installed first
- used in at least one real task or repair flow
- did not cause drift
- did not override proof
- did not confuse neighboring rules
- helped the current work move cleaner

## State: Promoted

Promoted means an installed and worn item has earned a stronger authority position.

Promoted is stronger than worn.

A promoted item may become a guide, standard, active rule, or higher-trust operating pattern only after repeated fit and proof.

Minimum promoted test:

- installed
- worn
- tested across more than one relevant use
- proof history supports it
- neighbor rules still hold
- current truth or anchor allows promotion
- promotion target room is correct

## State: Rolled-Back

Rolled-back means an installed, worn, or promoted item has failed, drifted, collided, or become unsafe for its current authority level.

Rollback does not mean the idea is destroyed.

Rollback means the item loses authority and moves to the correct lower state:

- back to testing
- parked
- source ore
- review history
- rejected

Minimum rollback triggers:

- proof fails
- receipt says FAIL
- item causes drift
- item creates a blocked lane
- item confuses active authority
- item violates neighbor rules
- item was promoted before being worn

## Clean Distinctions

Saved is not installed.

Installed is not worn.

Worn is not promoted.

Failed is not deleted.

Rolled-back is not erased.

Parked is not rejected.

Rejected is not source ore.

Proof decides movement.

## Use Rule

When judging any new helper, rule, file, method, metaphor, or tool, ask:

1. Is it only a candidate?
2. Is it being tested?
3. Is it parked or rejected?
4. Is it installed?
5. Is it actually worn?
6. Is it ready for promotion?
7. Does it need rollback?

Do not skip states because the idea feels useful.

Do not promote from capture.

Do not wear from source ore.

Do not treat a committed file as proof that the item is worn.
