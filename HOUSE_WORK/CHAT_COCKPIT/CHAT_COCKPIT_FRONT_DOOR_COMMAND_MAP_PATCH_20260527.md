# Chat Cockpit Patch - Front-Door Command Map

Date: 20260527

## Live behavior patch

Before Alpha locks anything, run the Front Door Selector.

Core joint:

Front Door selects the command shape. Alpha locks the selected object. Router sends it to the right house-rule family. Gates judge it. Final Judge closes it. Safe Dissolution drops what does not belong live.

## Command shapes

- pause: stop motion.
- nxt: continue current approved sequence only.
- deep walkthrough: park the micro-thread; walk the whole house.
- lock/save: one save object; proof route.
- run gates: judge object; do not auto-save.
- think tank: broaden/refine; do not execute.
- code/script: file-first; clean-copy rule.
- porch/root: custody map before movement.
- source/transcript: source custody; no raw transcript Git.
- proof/PASS/hash: receipt/proof lane.
- mule: handoff/review boundary.

## Object declaration

Alpha receives one selected object:

- current micro-task;
- whole house;
- save object;
- source object;
- proof object;
- root/porch object;
- tool/code object;
- blocked/pause state.

## Alpha lock

After selection, Alpha locks:

- object;
- source;
- mode;
- lane;
- boundary;
- proof need;
- what must not change.

## Boundary

This is a cockpit/router support map only.

It does not rewrite ACTIVE_GUIDES, CURRENT_TRUTH_INDEX, doctrine, source custody, porch cleanup, save proof, mule handoff, or the original router parent.

## Proof condition

The proof is live routing behavior:

- "deep walkthrough" parks the current micro-thread and walks the whole house;
- "nxt" continues only the current approved sequence;
- "lock/save" saves one object with proof.

## Short form

Door first. Alpha second. Router third.
