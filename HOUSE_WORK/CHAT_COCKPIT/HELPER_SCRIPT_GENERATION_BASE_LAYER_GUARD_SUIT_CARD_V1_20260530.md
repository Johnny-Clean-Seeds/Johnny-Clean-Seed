## Wear line

Generate helpers from safe primitives first. Task logic second. Proof closes.

## Wear when

Wear this before creating, repairing, or running a local helper script, especially after parser, binding, manifest, receipt, Git staging, dirty-state, or command-surface failures.

## Active checks

- Is a helper needed, or is this a judgment task?
- Did Code Gate parse and run the target path actually needed?
- Are report-writing primitives safe for empty collections and null text?
- Are variable names clear of PowerShell automatic/reserved variables?
- Are markdown fences safe for PowerShell strings?
- Are command outputs assigned to variables before operators are applied?
- Are manifest and receipt created in the correct order?
- Is staged-set verification exact?
- Did a failure become a breadcrumb trail instead of an isolated typo?

## Blocked moves

- no direct run after parser FAIL;
- no one-off repair when the scent repeats;
- no helper generator pattern that repeats `$Matches`, backtick strings, empty collection binding, or manifest-before-manifest;
- no broad audit from triage output;
- no doctrine, ACTIVE_GUIDES, CURRENT_TRUTH_INDEX, watcher, automation, implementation, or Whirlpool unlock.

## Close rule

A generated helper route closes only when the helper parses, runs the intended path, reports cleanly, preserves boundary, and does not add a new lower-layer breadcrumb.
