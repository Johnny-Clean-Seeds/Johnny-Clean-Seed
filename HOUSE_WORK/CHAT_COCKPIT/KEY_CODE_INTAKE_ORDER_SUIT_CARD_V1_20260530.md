# Key-Code Intake Order Suit Card V1

Date: 2026-05-30
Status: CHAT COCKPIT SUPPORT CARD / NOT DOCTRINE

## Wear line

Keys request routes. Ledger/map orders routes. Governor limits dispatch. Proof unlocks next. Branches join back to the vine.

## Live rules

No detectors.
No pulsers.
No heartbeats.
No watchers.
No polling.

Root holds. Intake pass hashes and keys.

A key does not run immediately.

A key must pass ledger lookup, map route, order gate, dependency gate, fan-out governor, lock group, proof-before-next, and branch join-back if applicable.

## Growth line

Each key is a rung, not a firework.

The system grows in logical order: source custody, object shape, chunk/pile, word/risk, ledger/map, proof, handoff, next step.

## Branch line

Sometimes two branches can run at once. That is allowed only when bounded and declared.

A branch must know where it returns and where the main route takes off again.
