# Chat Suit Card — Lower-Level Error Powerplay Freeze V1

Date: 2026-05-30
SuitKey: LOWER-LEVEL-ERROR-POWERPLAY-FREEZE-SUIT-V1
RuleKey: LOWER-LEVEL-ERROR-POWERPLAY-FREEZE-RULE-V1
Status: CURRENT CHAT SUIT / ERROR ROUTING SUPPORT / NOT DOCTRINE

## Wear when

Wear this suit when a save, helper, script, proof route, Git stage, Chat Drop action, Code Gate run, operator command, manifest, receipt, or local execution surface fails while the active object is higher-level.

## House links

- BRAIN_LEARNING: lower-level freeze rule.
- WORK_SHED / SORTING_BENCH: fit report and audit seed.
- PROOF_HISTORY: receipt and manifest.
- CHAT_COCKPIT: active suit card.
- GRANDMASTER_NODE_ROOT_LEDGER: node/feeler return route for base-layer failures.
- POWERSHELL_CODE_GATE: parse/run boundary and warning surface.

## Active checks

- What is the upper object?
- What lower layer failed?
- Is this a repeated family?
- Did the old route try to cover the base problem?
- What proof would show the lower layer is repaired?
- What condition lets the upper task resume?

## Allowed moves

- freeze the upper task;
- preserve evidence;
- classify lower-layer failure;
- repair the lower layer;
- run proof;
- resume only with a clean return point.

## Blocked moves

- do not bulldoze the upper task;
- do not call a direct-run failure a harmless script typo if the mechanism repeats;
- do not treat Code Gate probe PASS as job PASS;
- do not force-add ignored paths without lane/hash staging;
- do not create a new helper layer to hide the failing base layer;
- do not perform a broad audit unless repetition evidence triggers it.

## Final Judge

PASS only if the lower layer is named, fixed, proved, and the upper task resumes from a known clean point.

Short form:

Freeze above. Diagnose below. Repair the base. Resume with proof.
