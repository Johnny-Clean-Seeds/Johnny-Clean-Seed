# Idea Filing Cabinet Soft Index

Status: SOFT SUPPORT INDEX
Authority: support only; not command authority

## Purpose

This index defines a filing-cabinet shape for ideas, helpers, methods, old ways, and adopted patterns.

It is soft.

It does not move existing files.

It gives retrieval logic before hard structure.

## Cabinet Shape

Cabinet:
All ideas and methods.

Drawers:
Broad families.

Folders:
Grouped topics or method families.

Cards:
Atomic ideas.

Tags:
Cross-cutting retrieval keys.

Front Shelf:
Current or frequently used ideas.

Archive:
Old, superseded, or low-use ideas that may matter later.

## Drawers

### Drawer 1 — Current Operating Helpers

Holds:
Rules and checkers used in current work.

Examples:
Bell Logic Gate.
Word-Key Clarity Checker.
Weigh Options Before Code.
Trace-Gated Boss Control.

Default location now:
BRAIN_LEARNING and HOUSE_WORK support indexes.

### Drawer 2 — Candidate Helpers

Holds:
Ideas that may help but are not adopted.

Examples:
algorithms, bitstrings, filing methods, layout methods, graph methods.

Default location now:
SOURCE_ORE and Soft Suit style parking.

### Drawer 3 — Old Ways / Superseded Methods

Holds:
Methods changed out of current use but not thrown away.

Required fields:
old job, why changed, possible future fit, current status.

Default location now:
SOURCE_ORE, PROOF_HISTORY, or future archive shelf.

### Drawer 4 — Proof / Receipt Methods

Holds:
Proof patterns, receipt formats, save gates, pass/fail checks.

Default location now:
PROOF_HISTORY and BRAIN_LEARNING.

### Drawer 5 — Placement / Room Methods

Holds:
room maps, role indexes, system links, middle-room reviews, filing cabinet designs.

Default location now:
HOUSE_WORK/INDEXES and HOUSE_WORK/PLACEMENT_REVIEWS.

### Drawer 6 — Adoption / Pattern Transfer

Holds:
how methods are tested, adopted, parked, changed, or reused across lanes.

Default location now:
BRAIN_LEARNING and SOURCE_ORE.

## Front Shelf Rule

A thing belongs on the front shelf when it is:

- current
- used often
- changed often
- needed for next work
- a gate/checker that protects work
- a method currently under test

Front shelf means fast retrieval, not authority.

## Card Fields

Each idea card should eventually carry:

- Name
- Drawer
- Folder
- Status
- Source
- Job
- Fit condition
- Neighbor risks
- Lifecycle state
- Tags
- Next action
- Proof needed

## Organization Shapes Allowed

Use the shape that fits the task:

- hierarchy
- tags
- facets
- matrix
- sequence
- graph/link map
- timeline
- color/shape signs
- status labels

No single organization shape owns the whole model.

Choose the lowest-force shape that improves retrieval and proof.
