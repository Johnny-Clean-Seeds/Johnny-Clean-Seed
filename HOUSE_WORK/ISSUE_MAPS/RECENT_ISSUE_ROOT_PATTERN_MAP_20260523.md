## Root family grouping

### Lifecycle missing

Objects need full lifecycle before action.

Runner:
sent -> dropped -> run -> report -> verify -> archive -> receipt -> porch clean

Rule:
source/correction -> map -> wording -> apply -> prove -> save -> moving-forward guard

Icon:
brief -> design map -> generate -> small-size proof -> install -> visible proof -> close/park

### Live map not firing before work

The cockpit/router and rules existed, but did not always fire before action.

Fix:
new task map must shape the next action.

### Proof target mismatch

PASS must match the intended target, not neighbor success.

Fix:
intended artifacts must be compared to committed files/receipts/output.

### Porch/mail lifecycle failure

Porch is not storage.

Fix:
used runners and old mail need closeout.

### Moving-forward gap

Past diagnosis is not enough.

Fix:
do-this-not-that future behavior must be saved and used.

### Evidence diagnosis too soft

Repeated proof should lead to diagnosis.

Fix:
if the shoe fits, wear it; if the bone is sticking out, call it broken.
