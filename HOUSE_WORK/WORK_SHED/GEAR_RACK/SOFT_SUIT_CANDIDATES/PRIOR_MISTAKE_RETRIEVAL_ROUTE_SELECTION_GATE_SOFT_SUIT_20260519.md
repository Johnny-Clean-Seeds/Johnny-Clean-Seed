# Prior-Mistake Retrieval And Route Selection Gate - Soft Suit Candidate

Date: 2026-05-19
Base: main @ 2ade133
Lane: Work Shed / Gear Rack / Soft Suit Candidates
Status: Soft Suit candidate, not doctrine, not active guide, not tool promotion

## Job

Prevent repeated mistakes and long painful routes by retrieving similar prior cases before choosing a work route.

This is not "future prediction" as confidence theater.

This is:

prior evidence -> current shape -> route choice -> proof.

## Trigger

Wear this Soft Suit when any of these appear:

- repeated failure,
- connected batch,
- old/new/stale mix,
- mule return,
- proof/receipt mismatch,
- stale pointer risk,
- long painful route starting,
- assistant awareness should have fired,
- user says a pattern should not happen again,
- user asks why the simple path is not being used.

## Working loop

TRIGGER
-> STATE CLASSIFY
-> RETRIEVE SIMILAR CASES
-> CHOOSE ROUTE
-> PROVE ADVANTAGE
-> ACT / PARK / BLOCK
-> SAVE LESSON TAGS ONLY IF USEFUL

## Retrieval rule

Retrieve 3 to 7 similar prior cases when possible.

Look for:
- similar failure mode,
- similar lane,
- similar object type,
- similar proof risk,
- similar stale-state pattern,
- similar mule/batch/custody pattern,
- similar assistant behavior mistake.

Do not retrieve endlessly.

If 3 useful cases are enough, stop.

If no useful cases appear quickly, say so and choose the safest route by current evidence.

## Route choices

Choose one:

1. Batch safe-bin
   Use for connected mule returns, checker batches, multi-file reports, and companion artifacts.

2. One-by-one inspection
   Use for executable scripts, private/security material, unclear provenance, authority-changing files, destructive risk, or known collision items.

3. Direct next action
   Use when visible read-only output makes the next safe move clear.

4. Mule critique
   Use when the issue is large, uncertain, or needs outside hard critique.

5. Park
   Use when useful but not ready.

6. Block
   Use when proof, custody, or authority is unsafe.

## Prove advantage before acting

Name the advantage type:

- faster with equal or better safety,
- safer with acceptable speed,
- clearer placement,
- less stale-pointer risk,
- better retrieval from prior mistakes,
- easier mule return intake,
- better proof path,
- better tool-building leverage,
- less house drag,
- stronger finish condition.

Also name:
- evidence,
- tradeoff,
- risk,
- close condition.

## Save rule

Save a lesson tag only when it will improve future retrieval.

Do not tag every note.

Do not turn every observation into doctrine.
