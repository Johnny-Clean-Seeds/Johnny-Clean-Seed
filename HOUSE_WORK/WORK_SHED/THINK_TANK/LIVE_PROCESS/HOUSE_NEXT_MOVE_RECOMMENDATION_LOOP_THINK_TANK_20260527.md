## Center object

The house should learn to recommend next clean moves without self-authorizing execution.

## Think Tank finding

The missing middle is recommendation intelligence.

A clean house should not only store lessons.

It should notice current state, open issues, proof gaps, residue, parked objects, and possible next steps.

It should then generate candidate moves, gate them, rank them, and present a compact next-clean-move list.

## Outside model comparison

This matches improvement and agent-control patterns:

- PDSA / PDCA: plan, do, study/check, act.
- OODA: observe, orient, decide, act.
- MAPE-K: monitor, analyze, plan, execute, with knowledge underneath.
- ReAct: reasoning and action update each other through observations.
- Tree of Thoughts: explore candidates and evaluate before choosing.
- Reflexion: use feedback/reflection to improve future trials.
- Generative agents: memory, reflection, and planning shape future behavior.

House translation:

observe -> orient -> generate -> gate -> rank -> recommend -> await authority -> execute only if approved -> learn from result.

## Critical boundary

This is not autonomy.

It is recommendation.

The house may propose.

The gates decide the recommended route.

The assistant does not execute without authority.

## Recommendation output

The live output should be compact:

NEXT CLEAN MOVE:
what to do and why.

BACKUP MOVE:
what to do if blocked.

PARKED IDEA:
useful but not active.

BLOCKED MOVE:
do not do and why.

## Data-saving connection

This loop must obey Safe Dissolution.

Do not output giant lists.

Do not carry the furniture.

Recommend the next clean key.

## Clean phrase

House proposes. Gates judge. Assistant obeys.
