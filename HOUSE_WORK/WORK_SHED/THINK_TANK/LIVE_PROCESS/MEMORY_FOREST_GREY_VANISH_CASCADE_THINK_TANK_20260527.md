## Center object

Old chat retention without carrying old chat weight.

## Finding

The useful idea is not ordinary chunking.

Chunking preserves text pieces.
Summarizing lightens text but can flatten relations.
A memory forest preserves meaning shape.

## Comparison

Chunking:
good for raw recall, weak for living structure.

Summarization:
lightweight, but risks flattening important relationships.

Consolidation:
merges repeated ideas under shared roots.

Tree memory:
keeps parent/child idea structure.

Graph/cross-link memory:
handles large trees that touch in several ways.

Safe dissolution:
lets old raw chat grey or vanish from live carry after safe capture.

Redundancy/Waste:
decides merge, pointer-only, park, blocked, or proof-needed.

Proof custody:
blocks unsafe vanishing of evidence.

## Clean process

Build multiple trees, not one giant chunk pile.

Each tree needs:

- root name;
- mission;
- branch list;
- cross-links;
- proof pointers;
- decay state;
- return trigger;
- blocked items.

## Why it matters

The assistant should not keep old chat as live bulk.

The assistant should keep the map needed to retrieve meaning.

## Not deletion

Vanishing means no longer carried live.

Evidence still lives in proof custody, source custody, receipts, or saved files.

## Test needed

Do not promote broadly until tested on a real recent trail.

First test should be the recent helper-script/proof-custody/chat-weight failure trail.

## Short form

Tree the meaning. Grey the residue. Keep proof safe.
