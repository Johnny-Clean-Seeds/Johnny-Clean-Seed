# Assistant Mule Exchange Bridge

Home: HOUSE_WORK / WORK_SHED / MULE_WORKSHOP / BRIDGES / ASSISTANT_MULE_EXCHANGE_BRIDGE
Status: ACTIVE PROJECT STRUCTURE / EXCHANGE ROOM

## Purpose

Make assistant-mule file exchange direct, clean, and house-shaped.

This bridge answers the current problem: the assistant and mule need to interchange files without scattering packets across chat, Desktop, or stale 123-only tools.

## Rooms

ASSISTANT_TO_MULE
Requests, handoffs, packets, and instructions prepared by the assistant for the mule.

MULE_TO_ASSISTANT
Returns, reports, proofs, deltas, and proposed fixes from the mule for assistant intake.

SHARED_DESK
Temporary shared staging for one active exchange.

REQUEST_CARDS
Reusable request-card templates.

RETURN_CARDS
Reusable return-card templates.

FOOTPRINT_LEDGERS
What was read, written, skipped, blocked, returned, or archived.

STAGING
Temporary prep before a file becomes an active request or return.

BLOCKED_OR_DENIED
Requests or returns blocked by authority, missing proof, wrong lane, or stale state.

ARCHIVE
Old completed exchange packets.

## Core sentence

The mule does not need a tunnel if he has a room. The assistant and mule need a shared workroom with request cards, return trays, footprints, and cleanup duty.
