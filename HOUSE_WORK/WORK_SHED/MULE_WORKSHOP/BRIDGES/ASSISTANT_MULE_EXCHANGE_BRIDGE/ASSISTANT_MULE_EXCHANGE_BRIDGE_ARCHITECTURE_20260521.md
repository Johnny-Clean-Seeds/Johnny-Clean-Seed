# Assistant Mule Exchange Bridge Architecture

Date: 2026-05-21
Status: REPAIR ARCHITECTURE / PROJECT PLACEMENT

## Correction

The previous idea to place the front door bridge mainly in 123 was wrong for this stage.

123 is useful porch and storage, but the assistant-mule exchange belongs in Mule Workshop.

Reason:
The mule is now treated as a real guest. Guests need rooms, rules, desks, ledgers, return trays, and cleanup duties.

## Model

Mr.Kleen = house / brain.
Mule Workshop = guest room and workroom.
Assistant Mule Exchange Bridge = shared workroom between assistant and mule.
123 = porch, storage unit, and outside drop zone.
Mule = real guest worker.
Assistant = house-aware operator.
Receipts and ledgers = footprints.

## Exchange flow

ASSISTANT REQUEST -> REQUEST CARD -> ASSISTANT_TO_MULE -> MULE WORK -> MULE_TO_ASSISTANT -> RETURN CARD -> FOOTPRINT LEDGER -> HOST INTAKE / DISPOSITION -> ARCHIVE OR SAVE.

## What this fixes

- assistant and mule can interchange files cleanly;
- mule gets a real room instead of being a packet ghost;
- returns stop floating loose;
- request/return boundaries become inspectable;
- old work can be archived instead of haunting current work;
- file exchange gets traceability before adoption.

## Boundary
