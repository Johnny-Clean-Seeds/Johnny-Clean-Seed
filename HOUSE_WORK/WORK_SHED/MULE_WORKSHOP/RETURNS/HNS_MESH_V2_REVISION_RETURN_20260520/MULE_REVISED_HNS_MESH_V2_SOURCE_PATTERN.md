## Definition

The House Nervous System / Alarm-Debugger Mesh is a routed sensing and repair pattern for Mr.Kleen.

It gives critical lanes local sensors, thresholds, alarm levels, named debuggers, routes, safe reflexes, proof checks, memory outputs, and decay paths so the house can feel meaningful weak signals early, stop dirty movement before damage, repair root causes instead of symptoms, and remember useful failures without becoming ruled by alarms.

Short form:

Sensors everywhere. Alarms only at threshold. Debug root cause. Route cleanly. Prove or decay. No Alarm King.

## Core Law

Every critical lane gets a sensor.
Every sensor gets a threshold.
Every threshold gets an alarm level.
Every alarm gets a debugger.
Every debugger gets a route.
Every route gets a safe reflex, approved action candidate, or stop condition.
Every result gets proof, memory, parking, escalation, or decay.
No alarm, sensor, debugger, route, reflex, memory, source, tool, or mule output becomes command authority.

## Architecture Map

```text
SIGNAL
-> SENSOR
-> SALIENCE GATE
-> THRESHOLD CHECK
-> ALARM LEVEL
-> DEBUGGER
-> ROUTER
-> SAFE REFLEX / ACTION CANDIDATE / STOP
-> AUTHORITY / PROOF / BOUNDARY CHECK
-> RESULT
-> MEMORY / DECAY / PARK / ESCALATE / CLOSE
```

Clean meaning:

- Signal: something happened.
- Sensor: a lane can feel that condition.
- Salience gate: decides whether the signal matters.
- Threshold: line between background and alarm.
- Alarm level: severity and allowed response.
- Debugger: root-cause question set.
- Router: sends the cause to the correct lane.
- Reflex: narrow safe immediate move.
- Proof: evidence that the move is allowed or complete.
- Memory: useful residue retained.
- Decay: stale/noisy signal fades.

## Alarm Levels

| Level | Meaning | Default response |
| --- | --- | --- |
| QUIET | Sensor exists; no meaningful signal. | No action. |
| NOTICE | A change is visible but not risky. | Name if useful, keep moving. |
| WATCH | A pattern may be forming. | Keep context, avoid overreaction. |
| WARNING | Dirty milk, proof weakness, wrong-lane risk, delivery risk, or authority confusion is likely. | Pause and inspect. |
| BLOCK | Authority, proof, file custody, destructive action, or user-command risk is active. | Stop forward motion until resolved. |
| STABILIZE | Too many alarms, repeated failures, overload, or fog. | Drop secondary work and restore orientation. |
| STORM | The mesh itself is overfiring or losing discrimination. | Freeze optional work, choose one root alarm, apply decay. |

## Sensor Families

### Authority Sensors

Watch for source ore acting like doctrine, mule output acting like command, TODOs acting like authority, active guides touched without approval, CURRENT_TRUTH_INDEX touched without approval, memory replacing file proof, and metaphor becoming command.

Debugger:

- What object is steering?
- Is it user command, active guide, anchor, proof, source ore, candidate, TODO, or support?
- What proof allows it to steer?

Safe reflex:

- Name authority source.
- Return the object to the proper lane.
- Block promotion unless selected and proven.

### Proof Sensors

Watch for final success claims without receipt, missing SHA256 where needed, clean claims without status proof, failed commands followed by success language, proof from wrong directory, and claims broader than evidence.

Debugger:

- What exactly was proven?
- What remains unproven?
- Did any command fail?
- Is the claim larger than the evidence?

Safe reflex:

- Stop the success claim.
- Preserve output.
- Repair the proof statement.

### File Custody Sensors

Watch for generated helpers left loose, wrong lane placement, output mixed with input, old handoff mixed with current handoff, duplicate names, orphan artifacts, and files without role/receipt/next action.

Debugger:

- Is this active, archived, source, output, proof, support, or stale?
- Is it temporary?
- Does it belong in return lane, archive, storage, proof history, or active handoff?

Safe reflex:

- Keep current handoff active until house disposition.
- Move only when explicitly approved and safe.
- Record cleanup note if movement is blocked or unnecessary.

### Delivery Hygiene Sensors

Watch for user-requested deliverables answered only in chat, helper files left on Desktop after proof, generated artifacts not routed, and output scattered outside the named lane.

Debugger:

- Did the user ask for a file?
- Was the file created in the approved lane?
- Is there loose output elsewhere?

Safe reflex:

- Create artifact in approved return lane.
- Do not scatter copies.

### Source-Ore Sensors

Watch for outside research, old notes, web material, 123 material, proof history, live-test evidence, or prior packets steering as authority.

Debugger:

- What source supports which house claim?
- What adaptation is needed?
- What does the source not prove?

Safe reflex:

- Map source to claim.
- Keep support separate from authority.

### Mule Custody Sensors

Watch for mule output being installed before read/disposition, stale handoffs treated as current, current handoffs mixed with archive, and return output acting as command.

Debugger:

- What is the current handoff?
- What was read?
- What is output?
- What must house judge later?

Safe reflex:

- Produce return packet only.
- State no final authority.
- Keep current/stale separation.

### Desktop / Delivery Nerve

Watch for clean repo/proof while delivery surface remains dirty.

Debugger:

- Did the house get clean but the user's delivery surface stay cluttered?
- Is a helper still loose?
- Is the artifact intentionally kept?

Safe reflex:

- Route deliverable to intended lane.
- Mark any cleanup that needs separate approval.

### Active Ball / Load Sensors

Watch for too many bosses, return anchors, handoffs, TODOs, or alarms trying to steer one move.

Debugger:

- What is the active ball?
- Which signals are support only?
- What can be parked?

Safe reflex:

- Keep one active ball.
- Park secondary signals.
- Avoid broad shed rummage.

### Word-Key Clarity Sensors

Watch for one word carrying multiple behavioral roles, especially "alarm," "PASS," "source," "authority," "current," "archive," "runtime," and "dashboard."

Debugger:

- What does the word control?
- Is it decorative, source, rule, verdict, or command?

Safe reflex:

- Define the word in lane context before acting on it.

## Dead Nerve Detector

Purpose: detect missing alarms.

Triggers:

- A known rule should have fired but did not.
- User catches a failure the assistant should have caught.
- Proof passed while delivery failed.
- Source ore steered without authority warning.
- Stale mule material was used as current.
- PASS followed an error.
- File deliverable was requested but only chat was returned.

Reflex:

- Capture the failure class.
- Add or tune the sensor only if the pattern is proven or high impact.
- Avoid giant new bureaucracy.

## Alarm Storm Detector

Purpose: detect overfiring.

Triggers:

- Too many warnings at once.
- Same alarm repeats without closure.
- Every idea becomes urgent.
- Every source becomes important.
- Every task triggers a full ceremony.
- The nervous system creates bloat instead of discrimination.

Reflex:

- Stop broad expansion.
- Name active ball.
- Name last clean state when relevant.
- Park secondary signals.
- Handle one root alarm.
- Apply decay to weak signals.

## Debugger Structure

A clean alarm must run a debugger before action.

Debugger questions:

- What happened?
- Which lane felt it?
- What threshold crossed?
- Is the symptom local or referred from another lane?
- What is root cause?
- What authority permits movement?
- What proof is required?
- What should decay?
- What closes the alarm?

## Router Structure

The router sends signals to the right lane:

- Authority issue -> authority/source-of-truth lane.
- Proof issue -> proof/receipt lane.
- File placement issue -> custody/workshop lane.
- Delivery issue -> delivery hygiene lane.
- Source support issue -> source-to-claim map.
- Complex architecture issue -> MTER revision route.
- Load issue -> active ball/load gate.
- Valence/motivation issue -> Valence Council / Gated Valence Torus support, with proof still above valence.

## Reflex Structure

Safe reflexes are narrow and reversible unless separately approved.

Allowed reflex examples:

- Stop before destructive or authority-touching action.
- Name missing proof.
- Route output to named return lane.
- Keep current handoff separate from archive.
- Create a support note in the approved output lane.

Blocked reflex examples:

- Rewrite ACTIVE_GUIDES.
- Rewrite CURRENT_TRUTH_INDEX.
- Install doctrine.
- Build runtime or dashboard.
- Move/delete project files without explicit approval and proof.
- Treat the alarm as command authority.

## Memory And Decay

Memory outputs:

- receipt;

Decay outputs:

- support sources remain support, not authority.
