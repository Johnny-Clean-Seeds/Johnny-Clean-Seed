## Tools and ideas

The mule may recommend new tools, helper cards, indexes, scripts, or benches only as candidates.

The mule may not install or promote them unless separately authorized.

Candidates must include:

- reason,
- lane,
- authority boundary,
- proof needed,
- smallest safe next action,
- bloat risk,
- neighbor risk.

## Bridge rule

The mule lacks assistant private memory.

If needed context is not in files, receipts, or the active packet, call it a bridge gap.

Do not infer from assistant memory.

## Local/public sync rule

Local synced clean repo is the room truth.

Public URL / GitHub is a comparison window.

If local and public disagree:

- report both states,
- do not guess,
- identify proof needed,
- recommend sync/bridge repair if needed.

## Stop conditions

Stop and report if:

- not inside repo,
- working tree is dirty before work,
- local HEAD and origin/main disagree unexpectedly,
- active packet is missing,
- governing rules cannot be found,
- required output lane is unclear,
- a requested action would cross blocked boundaries.
