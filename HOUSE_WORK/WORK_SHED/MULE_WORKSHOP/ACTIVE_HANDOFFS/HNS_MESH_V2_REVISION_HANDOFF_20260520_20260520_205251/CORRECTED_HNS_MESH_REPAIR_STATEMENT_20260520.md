### 6.8 Mule Workshop Structure

The mule should have a workshop with a clean current/old separation.

Suggested structure:

```text
MULE_WORKSHOP
-> ACTIVE_HANDOFFS
   current handoff packets the mule must read/use now

-> ACTIVE_DESK
   current working source packets and live job material

-> RETURNS
   completed mule outputs

-> STORAGE_SHED / HANDOFF_ARCHIVE
   old handoffs, old packets, stale jobs, superseded source bundles

-> INDEX
   current active handoff pointer and archive locator
```

The mule should have bounded freedom to shape his workshop.

He should not be micromanaged into a brittle layout if a better local workshop shape helps him work.

But he must preserve custody:

- current vs old;
- input vs output;
- source vs authority;
- active vs archived;
- handoff vs return;
- support vs command.
