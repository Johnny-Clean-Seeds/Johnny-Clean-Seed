# Batch Safe Bin / Trial Suit Method

Date: 2026-05-19.
Room: Incoming File Parking / Work Shed.
State: method note.

## Job

This method controls large batch intake without creating fake ghosts.

## Safe Bin

The safe bin preserves the batch without treating it as active truth.

Hold:
- source,
- arrival context,
- file list,
- hashes when useful,
- lane guesses,
- dependency guesses,
- risk flags,
- current decision state.

## Trial Suit

The trial suit tests the batch as a working set.

Test:
- labels,
- companion files,
- order dependencies,
- proof path,
- conflict points,
- old-brain fit,
- new-brain fit,
- install chunk size.

## Install Strategy

Install as many related good files together as possible when the set proves fit.

One-at-a-time handling is reserved for:
- executable scripts,
- private/security-sensitive material,
- unclear provenance,
- files causing a specific conflict,
- high-risk authority changes.

## Output

Each batch review ends with one of:
- install now,
- install as clean chunk later,
- park,
- inspect more,
- reject with reason,
- isolate for safety.
