# Work Shed

Status: SOFT ROOM
Authority: support only; not command authority

## Purpose

The Work Shed is a controlled messy workroom for the assistant.

It receives big idea loads, rough material, candidate methods, old ways, and working piles without polluting active rooms.

The Work Shed is allowed to be messy during intake.

It must be clean in sorting.

## Core Rule

The Work Shed can receive mess.

The Work Shed cannot command.

The Work Shed cannot promote.

The Work Shed cannot become doctrine.

The Work Shed must sort before anything leaves.

## Room Map

INBOX:
Raw drops, messy loads, quick captures, unsorted ideas.

SORTING_BENCH:
Temporary work area where piles are grouped, compared, and clarified.

IDEA_CARDS:
Atomic idea cards after rough sorting.

FRONT_SHELF:
Current, hot, often-used, or frequently changed helpers.

PARKED_METHODS:
Useful methods not active now.

OLD_WAYS:
Superseded or old methods kept because they may fit later.

INDEXES:
Maps, cabinet indexes, tags, retrieval paths, and room signs.

## Exit Rule

Nothing leaves the Work Shed until it has:

- name
- job
- source
- status
- lane
- neighbor risk
- lifecycle state
- next action
- proof need

## Blocked

Do not treat Work Shed material as active authority.

Do not promote Work Shed material directly.

Do not install Work Shed ideas as doctrine.

Do not use Work Shed mess as proof.

Do not let the Work Shed replace SOURCE_ORE, PROOF_HISTORY, ACTIVE_ANCHOR, TODO, or BRAIN_LEARNING.

## Feeling Test

This room is soft.

If it helps the assistant work cleaner, it can be strengthened later.

If it creates fog, it can be revised, parked, or reshaped.

Proof decides.
