# Helper Organic Pathing Support Handoff

Date: 2026-05-30
Status: HELPER HANDOFF / SUPPORT / NOT DOCTRINE
WorkKey: ORGANIC-PATHING-SOFT-SUIT-INTAKE-GATE-JOIN-20260530-V1

## Job

Ensure helpers are helping.

A helper is not helping by adding a large report, loose files, or another concept pile.
A helper helps when it reduces path fog and returns usable path evidence.

## Helper task

For any route, intake object, or pathing candidate, return:

1. Active route checked.
2. Source node and target node.
3. Relation type.
4. Required toolbelt.
5. Load shelf.
6. Proof/source pointer.
7. Return path.
8. Path health score.
9. Dirty shortcut or desire path verdict.
10. Key/ledger/map join needed.
11. Boundary held.
12. Next condition.

## Required helper verdicts

Use these, not vague prose:

- ROUTE_HEALTH_PASS
- ROUTE_HEALTH_WATCH
- ROUTE_NEEDS_REPAIR
- DIRTY_SHORTCUT_BLOCK
- DESIRE_PATH_PAVE_CANDIDATE
- INTAKE_GATE_JOIN_NEEDED
- HELPER_OUTPUT_NOT_USABLE

## Helper failure signs

- no return path;
- no proof/source pointer;
- no source/target nodes;
- no toolbelt;
- no load shelf;
- no boundary;
- no next condition;
- adds concept weight without key/ledger/map join;
- treats source ore, Chat Drop, or proof receipt as authority.

## Intake Gate helper relation

Helpers should feed the Intake Gate by classifying incoming material into:

- intake;
- digest;
- circulate;
- use;
- store;
- excrete;
- heal;
- grow.

The helper should not decide promotion. It supplies route evidence and join candidates.

## Boundary

Helper output is support only until judged by the house. No doctrine, ACTIVE_GUIDES, CURRENT_TRUTH_INDEX, broad refactor, delete, move, automation, watcher, or universal map.
