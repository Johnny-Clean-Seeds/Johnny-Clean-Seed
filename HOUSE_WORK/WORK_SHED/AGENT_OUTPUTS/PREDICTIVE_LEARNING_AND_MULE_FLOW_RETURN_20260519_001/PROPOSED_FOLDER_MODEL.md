# Proposed Folder Model

Status: candidate model only.
Authority: not doctrine, not active guide material.

## Big Issue Lab Root

Recommended root:

HOUSE_WORK\WORK_SHED\BIG_ISSUE_LABS\

Use a lab only when a problem needs multi-pass review, mule critique, or structured comparison. Do not create a lab for normal small tasks.

## Lab Folder Shape

Example:

HOUSE_WORK\WORK_SHED\BIG_ISSUE_LABS\PREDICTIVE_LEARNING_AND_MULE_FLOW_20260519\

Required files:

1. 00_MANIFEST.md
   - file list, role, source status, read order, active target, boundary.

2. 01_CURRENT_STATE.md
   - current base, active ball, known mismatches, protected files, blocked moves.

3. 02_PRIOR_CASES.md
   - prior failures, wins, receipts, and parked candidates with lesson tags.

4. 03_PROBLEM_MAP.md
   - parent problem, child symptoms, ghost-boss risks, proof needs.

5. 04_CANDIDATE_MODEL.md
   - best current proposal before mule critique.

6. 05_MULE_HANDOFF.md
   - role, scope, questions, proof contract, return folder contract.

7. 06_RETURN_INTAKE_PLAN.md
   - how the mule return will be safe-binned, tested, parked, adapted, or rejected.

8. 07_CLOSEOUT.md
   - final status: adopted, adapted, parked, rejected-as-active, blocked, or needs later proof.

Optional folders:

- EVIDENCE\
- SOURCE_ORE\
- MULE_RETURNS\
- DO_NOT_USE\

## Mule Return Folder Contract

Every mule return should land inside one folder:

HOUSE_WORK\WORK_SHED\AGENT_OUTPUTS\[RETURN_NAME]\

Required:

- MANIFEST.md first.
- Main report second.
- Candidate files separated from evidence files.
- Optional support in OPTIONAL_SUPPORT.
- No active guide edits.
- No tool promotion.
- No command authority.

## Lesson Index Folder

Recommended later folder:

HOUSE_WORK\WORK_SHED\LESSON_INDEX\

Files:

- LESSON_INDEX.md
- CASES\
- TAG_SCHEMA.md
- RETRIEVAL_TESTS\

Case file shape:

LESSON_YYYYMMDD_SHORT_NAME.md

Required sections:

- Case ID
- Source path
- Hash or proof anchor
- Failure/win mode
- Trigger for future retrieval
- Similarity tags
- Route chosen
- Result
- What to do next time
- What not to infer
- Current status

## Placement Rules

- Big Issue Labs hold active structured review, not permanent doctrine.
- Lesson Index holds reusable retrieval lessons, not full source dumps.
- Agent Outputs hold return packages, not active commands.
- Source-ore remains source-ore unless separately promoted.
- Do-not-use material stays visible but cannot steer.

## Close Conditions

A lab closes when:

- current problem is named,
- prior cases are indexed or explicitly skipped,
- candidate is judged,
- mule output is received or no mule needed is stated,
- next action is adopt, adapt, test, park, reject, or block,
- no active guide or truth index edit is implied without explicit approval.
