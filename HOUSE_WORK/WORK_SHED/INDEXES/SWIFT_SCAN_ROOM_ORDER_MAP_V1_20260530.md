## Purpose

This map keeps Swift Scan from becoming a random room walk. Each room has a reason-before-entry, a job, and an output.

## Order

| Step | Room | Reason before entry | Output |
|---|---|---|---|
| 1 | Stop / Set Object | no useful walk without object clarity | locked object |
| 2 | Wash Live Load | heavy chat/file load distorts judgment | active carry + stale/proof-only list |
| 3 | Situation Awareness / Freshness | see current state before reasoning | current pointer |
| 4 | Proof / AAR | proof must be named before promotion | proven / not-proven split |
| 5 | Boundary / Promotion Pressure | repeated success creates false permission | blocked lanes |
| 6 | Premortem / Failure Forecast | test next action before doing it | risk check |
| 7 | Think Tank / Method Match | borrow useful method keys only | adopted keys |
| 8 | Creative Room / Synthesis | make method house-native | clean system shape |
| 9 | Lost Leftovers Sweep | compression can lose guards | recover/drop list |
| 10 | Decision Record / Map | one compact record beats repeated file load | context/decision/consequence |
| 11 | Final Judge / Omega | close without fake motion | stop/move/save/park/review |

## Swift Scan short form

Stop the pile. Carry one pointer. Review proof. Check freshness. Block false promotion. Forecast failure. Pull useful methods. Use Creative Room to make one clean shape. Recover leftovers. Record one decision. Move only if the next move has a real job.
