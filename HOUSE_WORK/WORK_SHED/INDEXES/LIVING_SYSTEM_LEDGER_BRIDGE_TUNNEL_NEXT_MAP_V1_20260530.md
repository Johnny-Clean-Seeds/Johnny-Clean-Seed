# Living System Ledger Bridge Tunnel Next Map V1

Status: NEXT OBJECT MAP / NOT IMPLEMENTED

## Problem

Ledgers, maps, keys, proof receipts, Chat Drop copies, and room notes exist, but the cross-links are not clean enough. Workers need clearer direction to the files and to the current Chat Drop rules.

## Desired shape

A web-style linked system where bridges and tunnels connect objects that belong together.

Bridge/tunnel directions:
- UP: object to parent rule, higher index, house map, or system principle.
- DOWN: object to proof, receipt, implementation, template, or row-level evidence.
- SIDE: object to sibling rule, related ledger, neighboring room, or parallel process.
- BACK: object to source ore, original spur, user correction, or prior failed route.
- FORWARD: object to next allowed action, return trigger, or blocked-lane condition.

## Bridge card fields

- BridgeId
- SourceObject
- SourcePath
- TargetObject
- TargetPath
- Direction
- RelationType
- WhyLinked

Make bridges before moving files. Link before cleanup. Classify stale copies before deleting. Preserve proof custody.
