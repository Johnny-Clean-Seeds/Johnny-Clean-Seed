## Living rules

No Session / No Gate:
if operator session is not open, gate shell cannot work.

No Packet / No Work:
gate shell acts only on a keyed packet.

No Return Surface / No Packet:
packet without return surface is invalid.

No Receipt / No Close:
packet is not done until receipt exists.

No Hidden Transcript Reading:
gate shell does not read operator prompt as default sensor.

Terminal-Safe Copyback:
terminal output must be minimal and safe to accidentally paste.

Gate-for-Gates Does Not Do All Work:
router routes; specialist gates work.

State Before Action:
gate action requires valid session, packet, route, and lock state.

Close Cascades:
operator close forces gate close/park/receipt.

Dead Letters Are Not Trash:
invalid/failed packets are preserved for review.
