# Git-to-Assistant Pointer Route Map V1

Date: 2026-05-30
Status: SUPPORT MAP / NO-BLEED POINTER ROUTE

## Route

Concept grows
-> Git durable anchor updates if concept/rule/proof changes
-> AssistantLocal chat-load pointer updates if assistant needs re-entry/load
-> UserLocal receives a bridge file only if user-facing purpose is explicit
-> Chat receives minimal live key only
-> receipt records hashes and stale checks

## No-bleed route rules

Git -> AssistantLocal:
allowed as pointer map/load card.

AssistantLocal -> Git:
allowed as references to anchors, not authority.

UserLocal -> Git:
allowed through explicit packet/drop/receipt/save route.

Git -> UserLocal:
only explicit user-facing bridge/drop-copy/pickup when purpose named.

AssistantLocal -> UserLocal:
blocked by default.

Chat -> any surface:
only through explicit save/update route.

## Stale checks

If Git HEAD changes, AssistantLocal load card may be stale.
If Git anchor hash changes, AssistantLocal pointer must refresh.
If UserLocal bridge file exists without purpose/receipt, review before use.
If Chat carries full content instead of pointer, unload to ledger.
