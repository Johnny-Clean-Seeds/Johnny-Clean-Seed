# Relation Method / Across-the-Board Relation Lens

Date: 2026-05-20
Lane: HOUSE_WORK / WORK_SHED / IDEA_BAG
Status: method candidate
Authority: idea/lens candidate; not doctrine; not active guide

## Purpose

Catch issues that are not isolated to one lane.

Some problems look like a local bug but actually touch several house lanes.

## Pattern recognition versus relation method

Pattern recognition asks:
Have I seen this shape before?

Relation Method asks:
What is this connected to across the board?

## Relation map

When an issue, fix, rule, tool, or pattern appears, inspect relations across:

- source;
- trigger;
- lane owner;
- proof;
- TODO;
- artifact;
- handoff;
- PowerShell;
- mule;
- memory;
- status/anchor;
- active guide boundary;
- next-start effect.

## Use cases

Use this lens for:

- repeated near-same scripts;
- local handoff failures;
- proof false-success hazards;
- passive-rule failures;
- artifact mistakes;
- mule packet friction;
- memory-vs-house capture gaps;
- broad status/search fog;
- rule candidates that may affect many lanes;
- neighbor-lane collisions.

## Output

A relation pass should return:

- home lane;
- touched neighboring lanes;
- parent issue;
- child ghosts;
- proof risk;
- carryover risk;
- whether this needs TODO, Gear Rack, Idea Bag, Sorting Bench, receipt, parking, block, or live-use watch.

## Boundary

Do not use this to justify installing everything everywhere.

Relation Method maps connections.

All Respectful Fits decides where a useful proven piece may respectfully belong.

Proof still decides.
