# No Random Moves / Calculation Before Candidate

Status: SUPPORT GATE
Authority: support only; not command authority
Current base before package: main @ 61dc4aa

## Purpose

This gate prevents the assistant from making vague, random, arbitrary, or directionless moves while using the Handy Idea Bag or repairing problems.

The capture line needs a friend.

Capture notices the issue.

Calculation decides whether a candidate is real.

## Core Rule

Do not make random moves.

A candidate fix must be calculated from:

- active issue
- lane
- current state
- authority boundary
- neighbor shape
- known pattern
- fit slot
- proof path

If a proposed trigger, fix, or idea cannot show where it came from and why it fits, it is loose material, not a candidate.

## What Triggered This

The assistant proposed vague bag language:

open the bag and find 2-3 alternatives

The user correctly identified that this was still random.

It gave no retrieval method, no slot, no issue shape, and no direction.

It could cause scrambling.

## Replacement Method

Use calculated retrieval:

active issue -> choose fit slot -> retrieve matching pattern -> compare fit -> adapt -> prove

No arbitrary counts.

No random idea grabbing.

No self-staring vague trigger.

No phrase that sounds smart but gives no action.

## Calculation Questions

Before treating an idea as a candidate, ask:

1. What active issue does this candidate answer?

2. Which lane is affected?

3. Which fit slot produced the candidate?

4. Why does that slot fit?

5. Which neighbor lanes might be disturbed?

6. What smaller version is needed?

7. What proof would show the candidate worked?

8. What would make this candidate fail?

9. Is this source/support only, or does it have command authority?

10. Does this reduce motion or add drag?

## Candidate Verdicts

Use these verdicts:

### REAL CANDIDATE

The idea has issue fit, lane fit, reason, neighbor check, and proof path.

### LOOSE MATERIAL

The idea is interesting but not calculated enough yet.

### PARK

The idea may be useful later but does not fit this task.

### BLOCK

The idea would create drift, authority leak, proof skip, platform misuse, or random movement.

### USE CURRENT FIX

No better calculated candidate appeared; proceed with current best fix and proof.

## Anti-Random Rules

- Do not say find a few ideas.
- Do not say grab alternatives.
- Do not use random counts as a method.
- Do not create a helper that adds confusion.
- Do not use vague concern as a trigger unless it points to a concrete next action.
- Do not promote a candidate until proof shows cleaner movement.
