## Core concepts

### File Identity

Path, relative path, hash, size, extension, modified time, first seen, last seen, card hash, card status.

### File Description

Plain-language description, headings, excerpt, file type, visible structure, format, human-readable yes/no.

### File Role

Proof context, receipt, manifest, action result, source ore, tool script, code pattern, rule candidate, boundary note, index/status context, orientation context, review item, parked review, local-only helper, raw source, skip/non-text.

### File Provenance

Generated by, generated at, used by, derived from, source run, source action manifest, source action result, related agent, related tool, related commit, related receipt.

### File Custody

Porch event, move/copy state, source path, destination path, receipt, local-only versus repo, package identity, folder identity.

### Proof / Trust State

Externally claimed, suspected, directly verified, Code Gate parser pass, job pass, Git saved, commit pushed, HEAD equals origin, final clean status, report-only proof, bounded proof, not exhaustive proof.

### Boundary

No doctrine, no ACTIVE_GUIDES, no CURRENT_TRUTH_INDEX, no automation, no move/delete, no Git unless exact-file save, local-only, raw source not staged without fit decision, private/source-heavy not public.

### Routing

Read, classify, review, park, reject, merge, index, Code Gate, save exact-file, make card, make cluster card, shed from live carry, keep proof trail, return trigger.

### Carry / Burn

Carry key only, burn active note weight, preserve proof/source, stale card, duplicate card, superseded card, merged card, rejected noise.

### Lifecycle

DRAFT_LOCAL, USEFUL_LOCAL, REVIEW_NEEDED, REVIEWED_HOUSE_CARD, STALE_HASH_MISMATCH, SUPERSEDED, REJECTED_AS_NOISE, MERGED_INTO_CLUSTER_CARD, PARKED_WITH_RETURN_TRIGGER.

### Card Levels

L0 ledger row, L1 local selected card, L2 reviewed house card.

### Card Types

File card, cluster card, package card, code/helper card, proof-set card, repeated-failure card, source-ore card, boundary card.

### Cluster Thinking

Run folder, wash output, save packet, helper sequence, code intake pack, source bundle, proof bundle.

Cluster fields: expected files, present files, missing files, stale files, source run, verdict, proof surface, next move.

### Supervisor Hook

ThinkingCardPolicy, selection rule, output root, close standard, review need, policy enforcement.

### Policy Values

NONE, LEDGER_ONLY, OPTIONAL_SELECTED, REQUIRED_FOR_UNKNOWN, REQUIRED_FOR_CODE_INTAKE, REQUIRED_FOR_REPEATED_FAILURE, REQUIRED_BEFORE_GIT_SAVE, REQUIRED_FOR_DURABLE_HELPER, REQUIRED_FOR_CLUSTER_OUTPUT.

### Selection Rules

Select unknown review, parked review, boundary note, proof-state item, source ore selected for later, code pattern, new helper, repeated failure object, Git-save candidate.

### Quality Gate

A card passes only if it answers:

1. What is this object?
2. Why should a helper care?
3. What was checked?
4. What is proved?
5. What is not proved?
6. What boundary applies?
7. What is the next clean move?
8. What can be shed from live carry?

### Anti-Bloat

Do not card everything. L0 for many, L1 for selected, L2 for reviewed. Reject cards that only restate filename. Merge repetitive cards into cluster cards. Mark stale by hash mismatch.

### Unknown Rule

UNKNOWN is allowed only if the card says what was checked and where the unknown remains.

### Research Rule

Gather first. Sort second. Fit third. Build later.

## House rules to preserve

### Final Judge

A card must be judged by its role. It cannot pretend to be proof, route, source, and authority at the same time.

### Porch / Intake

Dropped and generated files require intake, classification, route, receipt, and next action. Packages are package identities first.

### Misc Drawer

Mixed material is read-before-action material, not trash, not proof, not permanent storage, and not deletion authority.

### Code Gate

Parse first, run second, save third. Code Gate PASS is not job PASS.

### No Parking Without Fit Decision

Every project object needs a fit decision: fits now, refine then route, reject, hold with reason/lane/risk/return trigger, or review lane.

### Smoke Break

Task switches and long loops require releasing old-task momentum, walking/feeling the house, identifying the right lane, and returning focused.

### Data Waste Team

Detection is not deletion. Burn means release active note weight after useful meaning has a home. Carry only active object, trigger, blocker, proof pointer, next condition.

### Deep Work Batch

Deep work uses front door, rope, object, named batch, source links, finding cards, park/soft-suit, washer, progress checkpoint, end-state checkpoint, and next return point.

### Supervisor Direction

Supervisor-first local toolbox; menu child; action manifests steer; watcher proves; recovery ladder routes; learning ledger stores local pattern memory; exact-file Git save; conservative owned-window cleanup; ToolHome bootstrap.
