# Lane-Locked Manifest Staging — History, Fit, and Pull-Back Report — 20260528

Status: FIT REPORT / HISTORY CROSS-EXAM / LOCK-SAVE SUPPORT
Authority: not doctrine; not ACTIVE_GUIDES; not CURRENT_TRUTH_INDEX

## Active problem

A lock-save script failed because `git add -- HOUSE_WORK/WORK_SHED/...` hit an ignored-path wall. The failure was clean because it stopped and did not claim save success. The repair cannot be blind `git add -f`. The repair must be a lane-locked manifest exception.

## House-history cross-exam

This mechanism is not a foreign patch. It is the convergence of older house ideas:

- Abort Latch / fail-closed behavior: stop on dirty route instead of pretending success.
- Code Gate: parse first, risk-classify, run second, and remember Code Gate PASS is not job PASS.
- No Parking Without Fit: every object needs a role, lane, reason, and return trigger.
- Porch / YOU GOT MAIL: custody requires object name, source, classification, route, action, destination, receipt, and next action.
- Data Waste / carry the key: after proof exists, carry save ID, manifest hash, receipt path, commit hash, and end-state label instead of the full furniture pile.
- A/B Checkpoint Truth: separate progress from end-state so a choke cannot disguise itself as clean completion.

## Pull back out

Pull back Abort Latch as stop behavior. A nonzero Git save step blocks until routed.

Pull back Code Gate warning classification: `FORCE_ADD_REQUIRES_MANIFEST_KEY`.

Pull back Porch fields into manifest entries.

Pull back No Parking fit decisions into every manifest entry.

Pull back Data Waste after save: chat carries proof keys, not the whole manifest body.

Pull back A/B end-state labels for every save attempt.

## Do not repeat

Do not solve wrong-lane evidence with stronger staging machinery. An ignored path may be a wrong-lane signal.

Do not build a giant universal save engine before one narrow repaired save proves itself.

Do not let Code Gate become a fake blessing. Code Gate PASS is not job PASS.

Do not let the manifest become a throne. It records custody and scope; it does not decide quality or authority.

Do not treat hash as magic. A bad file can hash perfectly.

Do not write receipt language that outruns Git proof.

Do not force-add folders or wildcards.
