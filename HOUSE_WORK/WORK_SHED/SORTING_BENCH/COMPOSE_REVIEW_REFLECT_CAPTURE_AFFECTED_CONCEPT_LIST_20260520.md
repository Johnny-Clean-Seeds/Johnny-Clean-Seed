# Compose-Review-Reflect-Capture Affected Concept List

Date: 2026-05-20

## Status

SORTING BENCH CAPTURE / AFFECTED CONCEPT LIST.

## Why This Exists

The user identified that when the assistant composes a thought, prompt, handoff, or long code block, the review point itself can reveal missing structure.

That is a capture moment.

The assistant must not leave that structure only in chat.

## Core Lifecycle

The corrected lifecycle is:

1. Compose.
2. Review.
3. Reflect.
4. Identify newly revealed structure.
5. Capture or park it.
6. Wear parked material as Soft Suit candidate when fitting.
7. Prove behavior through live use.
8. Mark safe only when proof supports it.
9. Inject only when the correct save/update/code moment naturally arrives.

## Affected Concepts / Bases

### 1. Long Code Draft-Review-Send Gate

The long-code gate reviews code before sending.

This new rule adds a second question:

What structure did the drafted code reveal that must be captured?

### 2. Reusable Script Skeleton Pull Rule

Skeleton pull asks whether a cousin script already exists.

This new rule adds:

If a new cousin pattern appears during drafting, park it as a skeleton candidate after the current task is safe.

### 3. Soft Suit Trigger Event Proof-Before-Mark Rule

Trigger events need proof before marking.

This new rule clarifies:

A parked idea can be worn as Soft Suit candidate before it is marked safe, but trigger-event marking still requires proof.

### 4. Discovery Capture Interrupt Rule

Discovery capture interrupts live work when a pattern appears.

This new rule adds a specific discovery window:

the draft/review holding point before sending.

### 5. After-Fix Carryover Latch

After a fix, the lesson must improve the next start.

This new rule adds:

after a composed artifact, the revealed structure must also be carried forward.

### 6. Filing Cabinet / Gear Rack

The draft may reveal a reusable workflow object.

Capture it as Gear Rack / Filing Cabinet candidate when recurring.

### 7. Relation Method / Across-The-Board Lens

The reflection must check all affected bases and neighbors instead of only the visible artifact.

### 8. Parking Lane

Parking is active holding, not neglect.

A parked idea has lane, status, proof need, and future injection condition.

### 9. Soft Suit Candidate Behavior

Parked material may guide live behavior immediately as a candidate suit.

But it cannot claim safety, promotion, or doctrine until proven.

### 10. Injection Timing

Injection happens when the correct save/update/code moment naturally appears.

It does not happen from excitement or because the idea sounds strong.

### 11. Proof Before Promotion

Proof still controls promotion.

No proof, no safe mark.

No proof, no gate extraction.

No proof, no doctrine.

### 12. Active Anchor / Next Start

If the captured idea affects the current next move, the anchor must reflect the correct next-start boundary.

### 13. Status Ledger

The status ledger can record the save and meaning, but it does not become authority.

### 14. TODO

If the idea needs future live use, a TODO should be created or updated.

### 15. Growth Cycle Unload

The Growth Cycle unload exposed this problem because a good explanation produced new structure.

This rule prevents that structure from being lost.

### 16. Mule Return Disposition

Mule return unpacking should identify not only mule conclusions but also new house structures revealed during assistant review.

### 17. Trigger-Event Marking

The previous premature trigger-event code proved why reflection and proof order matter.

### 18. Prompt Composition Itself

Prompts are artifacts.

A prompt can reveal missing rules, affected concepts, and future skeletons.

The assistant must review prompts as objects, not only send them.

## Proof Needed Later

Future proof should show that the assistant:

- reviewed a draft before sending;
- noticed a newly revealed structure;
- captured or parked it;
- avoided over-injection;
- used the parked candidate safely;
- waited for a correct save/update/code moment before injection;
- did not promote without proof.
