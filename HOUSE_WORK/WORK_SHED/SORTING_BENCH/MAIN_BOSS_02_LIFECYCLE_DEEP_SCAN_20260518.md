## Initial Weak-Link Findings

### Weak Link 1 — Lifecycle words exist across the house, but a single state machine is not yet proven.

Why it matters:
Lifecycle terms can appear clean individually but still fail as a system if transitions are not defined.

Need:
A lifecycle state machine must define states, allowed transitions, blocked transitions, proof needed for each transition, and rollback/failure path.

### Weak Link 2 — Source/support/authority state can be confused with lifecycle state.

Why it matters:
A source-only item can be captured, parked, tested, or rejected, but it still cannot command unless promoted through the right gate.

Need:
Lifecycle state must sit beside authority state, not replace it.

### Weak Link 3 — Installed, worn, and promoted need sharper distinction.

Why it matters:
Installed can mean saved into a file. Worn means used in real work. Promoted means given stronger standing after proof. These cannot collapse into one word.

Need:
Define each separately and require proof before moving upward.

### Weak Link 4 — Rollback / rolled-back needs an explicit path.

Why it matters:
If a helper or rule fails after use, the house needs a clean return path that preserves proof/history and does not erase the failed attempt.

Need:
Rollback should mean supersede, park, or revert through receipt, not hide or delete history.

### Weak Link 5 — Front shelf is a retrieval state, not a lifecycle promotion.

Why it matters:
A Work Shed front-shelf helper is easy to reach but still support-only.

Need:
Lifecycle machine must keep retrieval closeness separate from authority.

## Candidate State Machine Draft — Inspection Only

raw:
Unprocessed input. Cannot command.

captured:
Saved in a safe lane. Cannot command by capture alone.

candidate:
Named possible helper/rule/fix. Needs fit judgment.

testing:
Being used in limited scope. Needs proof.

parked:
Useful but not active now. Retrievable later.

rejected:
Does not fit current need/core. Preserve reason if useful.

installed:
Saved into a proper file/lane. Not automatically proven in use.

worn:
Used in real work. Can reveal fit or failure.

promoted:
Given stronger role after proof and neighbor checks.

rolled-back:
Removed from current use, superseded, reverted, or parked after failure or better fit appears.

archived:
Historical. Enter only for evidence, rollback, or comparison.

superseded:
Replaced by a newer proof path. Old record remains visible.
