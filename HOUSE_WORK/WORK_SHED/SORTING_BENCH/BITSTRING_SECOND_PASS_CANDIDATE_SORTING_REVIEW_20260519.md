## Sorting rule for this pass

A candidate ranks higher when it:
- helps the assistant hold papers in order,
- protects proof/source/history boundaries,
- improves lifecycle state clarity,
- reduces false confidence,
- strengthens authority routing,
- avoids fake compression,
- and can be tested in a small Work Shed object.

A candidate ranks lower when it:
- wants a full system install too early,
- requires runtime automation now,
- creates heavy ceremony,
- duplicates existing Clean Seed doctrine,
- or turns bitstrings into magic instead of useful representation discipline.
