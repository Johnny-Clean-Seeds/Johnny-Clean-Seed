# Desktop Helper Left Behind - Delivery Failure Capture

Date: 2026-05-20
Lane: HOUSE_WORK / WORK_SHED / SORTING_BENCH
Status: FAILURE CAPTURE / RULE REPAIR
Base before save: 11bf30e

## What happened

The assistant created a one-time Desktop PowerShell helper for the Valence Council / Gated Valence Torus save.

The helper successfully saved the source-pattern candidate and produced clean git proof.

But the helper file remained on the user's Desktop after serving its purpose.

The user had to point out that AI is awful at leaving files around and that the proof was in the pudding.

## Root failure

The assistant treated cleanup as an afterthought instead of part of the helper's successful proof path.

This caused a split flow:

save helper
-> proof
-> user notices Desktop clutter
-> separate cleanup command

That is not clean delivery.

## Corrected behavior

Future temporary Desktop helpers must self-clean after successful proof, or be routed into Mr.Kleen as tool/tool-candidate material with receipt.

Desktop is not a parking lane.

## Scope

This repair does not change doctrine.
This repair does not rewrite ACTIVE_GUIDES.
This repair does not rewrite CURRENT_TRUTH_INDEX.
This repair does not install automation or a runtime.

It saves a behavior rule for delivery hygiene.
