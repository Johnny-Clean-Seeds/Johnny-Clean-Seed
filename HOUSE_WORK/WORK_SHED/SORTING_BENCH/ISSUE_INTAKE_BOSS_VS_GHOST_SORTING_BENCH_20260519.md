# Issue Intake / Boss-vs-Ghost Sorting Bench

Status: living Work Shed object
Date: 2026-05-19

Purpose:
When many issues arrive at once, set them calmly to the side before fixing.

This prevents frantic repair, blob-saving, file risk, and ghost-boss chasing.

Trigger:
Use this bench when:
- the user sends multiple issues at once,
- the user asks "why are we falling apart?",
- rule-trigger failure is suspected,
- several saved rules are not firing,
- file safety feels exposed,
- or the assistant is tempted to fix before sorting.

Intake fields:

## Raw pile
What came in?

## Separate pieces
List the distinct ideas/issues.

## Symptoms
What is visibly going wrong?

## Candidate real bosses
What may be the actual parent issue?

## Candidate ghost bosses
What looks scary but may only be a symptom, stale edge, or wrong-room problem?

## Proof needs
What must be checked before calling it real?

## File risks
What files, folders, branches, receipts, or active worker lanes could be disturbed?

## Parking items
What should be safely held but not fixed now?

## Next action
Choose one:
- fix current boss,
- park with bookmark,
- run proof check,
- save/lock,
- pause,
- or ask one clarifying question.

Gear to wear:
Pink Pony Diner Apron.
Active Ball Load Gauge.
Gear Rack.
Aha Origin Tagging Lens if a new fix appears.

Boundary:
This bench is not doctrine.
It is a calm sorting tool for high-density issue moments.
