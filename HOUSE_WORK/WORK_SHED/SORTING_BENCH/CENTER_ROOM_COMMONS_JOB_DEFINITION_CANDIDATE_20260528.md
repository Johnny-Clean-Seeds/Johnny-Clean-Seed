## 1. Room Name

CENTER ROOM / COMMONS

Short form:

Where the house remembers why it is awake.

## 2. Why This Room Exists

The house has strong gates, receipts, proof, paths, lockers, helpers, and rules.

That is good, but it can make the house feel like a courthouse or machine shop if there is no central calm surface.

The Center Room exists to answer:

- What is alive right now?
- What is quiet right now?
- What is the house for today?
- What should I not carry?
- Where do I go if I am lost?
- What is the smallest honest next move?

It is not a dashboard, command center, doctrine file, proof index, or todo board.

## 3. Core Job

The Center Room should provide a small, human-readable current-state surface.

It should carry:

- current clean anchor
- active object, if any
- quiet/parked load pointer
- rule pressure currently worn
- next clean move
- stop conditions
- where to look if lost

It should not carry every detail.

The Break Room Locker carries parked loads. Proof History carries receipts. Assistant Path carries orientation paths. Tool Shed carries helpers. The Center Room points to those rooms instead of duplicating them.

## 4. What Belongs Here

- current anchor
- active object
- current house mood / stance
- current rule pressure
- current open lane
- current quiet lane
- current stop condition
- pointer to Break Room Locker
- pointer to latest proof receipt or status card
- one next clean move

## 5. What Does Not Belong Here

- full proof index
- full helper registry
- all parked issues
- all receipts
- all code
- all doctrine
- long chat summaries
- raw logs
- mule dumps
- debugger reports

If it gets heavy, it belongs in another room.

## 6. Think Tank + Opposition Cross-Check

Think Tank pressure:

Move the thinking burden out of live chat and into the house.

Center Room fit:

It reduces live-chat burden by giving the house one visible center point.

Opposite pattern rejected:

Courthouse-house: every movement requires a formal hearing, receipt, and gate.

Clean Seed inversion:

Center Room gives calm orientation before gate selection.

Opposite pattern rejected:

Dashboard-house: everything becomes a live control panel.

Clean Seed inversion:

Center Room is a small current-state surface, not a dashboard.

Opposite pattern rejected:

Memory fog-house: context stays in chat and slowly turns into fog.

Clean Seed inversion:

Center Room carries only the current key, while Break Room Locker carries parked load.

## 7. Relationship To Break Room Locker

Break Room Locker holds unresolved load.

Center Room holds current orientation.

Locker is for what should not be carried live.

Center is for what must be remembered live.

The Center Room should include a pointer like:

Parked load: see HOUSE_WORK/BREAK_ROOM_LOCKER/LOCKER_INDEX.md

## 8. Relationship To Assistant Path

Assistant Path helps a fresh reader enter and understand the current path system.

Center Room helps the current worker regain center before choosing work.

Assistant Path answers: Where do I start?

Center Room answers: What is awake right now?

## 9. Relationship To Proof History

Proof History stores proof.

Center Room points to the latest relevant proof, but does not become the proof library.

The Center Room should never claim proof without a pointer.

## 11. Return Triggers

Use the Center Room when:

- user says walk the house
- user says smoke break
- user asks what is on the plate
- user asks what the house feels like
- assistant is carrying too much chat load
- work switches lanes
- new session needs orientation
- no active object is selected
