# Frame-to-Script Merge and Neighbor Lanes

Date: 2026-05-30
Status: MERGE MAP / SUPPORT ONLY / NOT PROMOTED

## Why this exists

Frame-to-Script Candidate System should not become an isolated concept pile.

It should merge with already saved neighbors while staying support architecture.

## Merge neighbors

### Key / Ledger / Map Driver

Shared language:
keys identify/request, ledgers hold state, maps route, events prove motion, final packets close after proof.

Merge need:
Frame contracts should become one kind of keyed route object.

### Intake Gate System

Shared language:
incoming object must be classified, bounded, and routed before action.

Merge need:
Frame-to-candidate is an intake route for script-candidate requests.

### Key-Code Manual Test Evidence

Shared language:
real hash, key assignment, ledger route, map route, governor, branch join-back, proof-before-next.

Merge need:
Frame-to-candidate manual test should reuse that discipline.

### Code Gate / PowerShell Gate

Shared language:
parser PASS is not job PASS; Code Gate tests candidate shape/risk, not final usefulness.

Merge need:
Candidate lifecycle must keep Code Gate result separate from job proof and placement.

## Open lane rule

Do not promote the merged concept yet.

First prove:
manual frame-to-candidate contract test.

Then consider:
candidate writer helper only if the manual test shows the frame contract is sufficient.
