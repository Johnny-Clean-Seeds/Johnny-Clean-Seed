## Trigger / difference signal

A trigger is a difference signal.

It appears as need, friction, threat, opportunity, user command, curiosity, memory echo, contamination risk, or growth-stage pressure.

The trigger is not action. Action is the end movement.

Trigger asks whether anything needs to be noticed, valued, protected, changed, grown, rejected, or acted on.

Sharper definition:

A trigger is input that creates a gap against need, expectation, viability, goal, memory, boundary, or growth-stage pressure.

## Relevance / salience gate

## Brake / decay / strengthen / weaken

No feedback loop is clean unless it has a route, a brake or decay path, and a stop condition.

Strengthen when a pathway is proofed, useful, bounded, and repeatable.

Weaken when a pathway creates bloat, fog, overreach, false certainty, or unstable motivation.

Brake when acceleration starts outrunning proof, authority, boundary, or current command.

Decay when a signal is unproved, stale, weak, unused, or only locally useful.

Park when the idea is useful but not stage-ready.

Block when the route violates authority, proof, boundary, No Rule King, or active user command.
