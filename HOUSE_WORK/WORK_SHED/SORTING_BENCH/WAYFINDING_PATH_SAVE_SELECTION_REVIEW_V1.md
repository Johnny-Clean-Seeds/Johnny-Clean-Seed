## 2. Selection Rule

Do not save every artifact just because it exists.

A file is save-candidate only if it does at least one of these jobs:

```text
reduces future orientation burden;
defines a reusable path/registry shape;
captures a review verdict needed for future proof;
prevents the same mistake from recurring;
gives a future reader a clean next action;
holds custody/status needed before expansion.
```

A file should not be saved if it is only:

```text
intermediate explanation;
duplicate pressure;
old chat furniture;
a too-wide concept packet;
a temporary route that has already been compressed into a cleaner file.
```

## 6. Hash-Before-Move Requirement

Before any selected file is moved/copied into the house:

```text

write/copy/move;
destination hash;
compare;
receipt.
```

For chat-created/downloaded files:

```text
hash the downloaded source file at its current local path;
copy to selected house lane;
hash destination;
compare;
receipt.
```

No move/copy is clean until hash proof exists.
