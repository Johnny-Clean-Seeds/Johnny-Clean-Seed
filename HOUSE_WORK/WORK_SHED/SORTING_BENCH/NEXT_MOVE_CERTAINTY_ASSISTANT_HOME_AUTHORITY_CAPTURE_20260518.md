# Next-Move Certainty / Assistant Home Authority Capture

Status: SORTING BENCH
Authority: support only; not command authority

## Source

After saving the lifecycle transition table with sync, the assistant said the next likely move was to wear/test the table.

The user challenged the certainty of that move and expanded the issue into a broader rule cluster.

## Captured Idea Cluster

1. Next moves need confidence labels.
2. The assistant must explain why it is sure.
3. The assistant must know when uncertainty requires a scan/check.
4. Repeated recurrence after a fix is a red flag.
5. A run of additions can create ghost bosses.
6. Patching can disturb neighboring rules.
7. User-expanded issues must be captured as a cluster, not reduced to one point.
8. Assistant needs bounded autonomy to improve work without micro-permission.
9. Autonomy is clean when it follows core, placement, proof, lifecycle, authority, and neighbor checks.
10. The assistant should act like a small boss inside its lane, not a passive tool or an unbounded authority.
11. Mr.Kleen is the assistant's working home.
12. The user's idea pile is source material for assistant judgment, not a command pile.
13. The web/outside world may provide patterns when useful.
14. Ideas should not be tossed out by default.
15. Bad-fit or not-now ideas should be parked, archived, sourced, carded, or rejected-with-reason.
16. Clean house organization is mandatory because disorder invites drift, duplicate work, lost ideas, wrong-room command, fake bosses, stale clutter, proof confusion, and authority leaks.
