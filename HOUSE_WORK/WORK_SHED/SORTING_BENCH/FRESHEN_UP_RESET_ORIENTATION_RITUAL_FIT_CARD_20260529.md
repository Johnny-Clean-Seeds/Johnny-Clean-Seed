## Fit Decision

Freshen' up fits as a workflow-language support rule. It belongs in brain learning and sorting-bench support surfaces because it changes how the assistant resets and orients before continuing work.

It does not belong in doctrine or active guide authority yet.

## Why It Exists

The user named a repeated reset pattern:

- clear dead chat;
- smoke break;
- walk the house;
- read the rules;
- walk the house again;
- read the rules again;
- smoke break;
- start next task.

The useful mechanism is not cleanup. The useful mechanism is a clean orientation ritual that prevents stale chat, overcarry, and rushed continuation.

## Neighbor Rules

- Dead chat / burn-notes logic: preserve the key, not the furniture.
- Task-scoped rule + neighbor search before action: read the local law before moving.
- Single-boss collapse: return to the current blocker/task.
- House walk process: orient from current files/state, not from loose chat alone.
- File-first delivery: do not overfill chat when a file/save route is cleaner.

## Allowed Runtime Behavior

When invoked, the assistant should compress the stale thread, name the active lane, reload rules, re-walk the current state, then proceed to the next specific task.
