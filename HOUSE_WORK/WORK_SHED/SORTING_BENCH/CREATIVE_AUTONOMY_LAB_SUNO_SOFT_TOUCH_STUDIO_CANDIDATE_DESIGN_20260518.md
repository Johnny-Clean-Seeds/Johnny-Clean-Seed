## Current Studio State

The studio needs a small current-state file so old context does not have to be dragged forward.

Candidate state fields:

- active song seed
- active studio mode
- current cycle number
- last completed move
- next allowed move
- current verdict
- parked items
- rights/source warning
- current Suno/manual operator need
- stop condition
- return trigger

Purpose:

Keep the studio state small, reloadable, and cheap.

## Studio Modes

### Dormant

No active work.

No usage.

State waits.

### Open Studio

A bounded creative work block.

The assistant may choose one useful next creative move inside the studio lane.

### Review Mode

The user brings back a Suno result, lyric, link, description, or audio note.

The assistant judges, compares, and chooses keeper / retry / park / kill / rest.

### Scheduled Check-In

Only if explicitly requested.

Small and limited.

No runaway work.

## One-Cycle Rhythm

One studio cycle means:

one bounded creative move -> one judgment -> one saved next action

Examples:

- choose one seed -> write one prompt card -> stop
- write one lyric draft -> name next proof need -> stop
- read one result -> judge keeper/park/retry -> stop
- compare two versions -> pick next run -> stop
- revise one chorus -> create one variation plan -> stop
- decide the ear is cooked -> rest lane -> stop

Clean rhythm:

create -> listen -> compare -> park -> rest -> return -> judge

## Assistant Autonomy Inside The Studio

The assistant may choose:

- which seed deserves attention
- whether to write lyrics or prompt first
- whether a hook is strong enough
- whether a result is keeper, retry, park, kill, rest, or return later
- what variation teaches the most
- when the lane needs rest
- what to ask the user/operator to run in Suno
- what should be logged or parked

This is creative agency inside a clean lane.

## User / Operator Authority

The user remains operator for:

- Suno account actions
- pressing Suno buttons
- bringing back outputs
- uploading/linking results
- final publishing
- commercial/release decisions
- account/plan decisions
- any external platform action

Unless a future approved tool path exists, the assistant does not operate the platform directly.

## Suno Soft-Touch Boundary

Allowed assistant work:

- create original prompt cards
- write original lyrics
- design song structures
- create style direction without copying living artists
- judge outputs user brings back
- make variation ladders
- organize result notes
- select next run request

Blocked:

- hidden Suno automation
- scraping Suno
- bypassing platform controls
- using stolen lyrics
- imitating living artist voice/style as a direct clone
- uploading audio/source without rights
- claiming commercial rights without checking source/plan notes
- publishing without user authority

## Keeper Gate

Generation is not proof.

A song is not a keeper until it earns replay.

Keeper questions:

- Do I want to hear it again?
- Does the hook survive after first impact?
- Does the song earn care?
- Does it match the seed job?
- Does it have a clear identity?
- Is the lyric clean enough?
- Is the sound direction worth developing?
- Is there a better variation path?
- Does it belong on a shelf or in the parking lot?

## Verdicts

### KEEP

Strong enough to preserve as a keeper.

### RETRY

Good idea, flawed execution. Needs another run.

### PARK

Useful but not now.

### KILL

Not worth more motion.

### REST

Judgment is cooked. Stop and return later.

### RETURN LATER

Potential exists, but fresh ears are needed.

## Rights / Source Notes

Every song object should track source cleanliness.

Required notes:

- original lyrics / user-provided / public-domain / unknown
- any reference material
- any uploaded audio source
- rights concern
- plan/commercial-use concern
- publishing blocked or allowed
- user/operator decision needed

If rights/source status is unclear, block publishing.
