## Cycle flow frame

### Work Entry Cycle

Purpose:
Stop action from starting in the wrong state.

Flow:
state -> custody -> lane -> rule confirmation -> completion standard -> action -> stop

### Idea Capture Cycle

Purpose:
Prevent dense user ideas from being lost or flattened.

Flow:
raw idea -> group -> parent boss/ghost split -> lane -> TODO/park/apply

### TODO Cycle

Purpose:
Make TODO actionable without fake priority.

Flow:
parent boss -> children/ghosts -> selected active item -> proof -> close/park

### Mule Cycle

Purpose:
Use mule as sharpener, not supervisor.

Flow:
packet -> worker return -> intake -> dedupe against current house -> disposition -> no inertia

### Proof Cycle

Purpose:
Separate done from PASS.

Flow:
write -> hash -> receipt -> commit -> push -> fetch -> HEAD/origin match -> clean status

### Learning Cycle

Purpose:
Turn problems into clean growth.

Flow:
failure -> diagnosis -> rule candidate -> save if selected -> live-use proof
