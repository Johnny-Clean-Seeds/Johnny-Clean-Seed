# System Links Room

Status: ACTIVE SUPPORT ROOM
Role: prove each system group is internally tied together before broader cross-group linking
Authority: support room only; not command authority

## Core Rule

Do not turn the house into a web.

Each system group must first prove its own internal links.

A system group is clean when its own parts work together:

- entry point
- purpose
- member files
- authority file
- support files
- proof/history files
- flow order
- blocked moves
- open gaps
- next transition

Cross-group linking comes later.

First make each group internally coherent.

## Why This Matters

Recursive growth needs small systems that work inside larger systems.

If each group is internally clear, then larger flow can be built by linking clean groups.

If groups are internally foggy, cross-linking only spreads fog.

## Link Proof Rule

A group is not considered linked just because its files sit in the same folder.

A group is linked only when the map shows:

- what starts the group
- what each file does
- how files point to each other
- which file has authority
- which files are support only
- what proof applies
- what order the work flows
- what is missing
- what should not be touched

## Blocked Behavior

Do not repair groups during a link audit.

Do not restructure folders during a link audit.

Do not create cross-group highways until internal group links are checked.

Do not treat this room as authority over ACTIVE_ANCHOR.txt.

Do not treat link gaps as command authority.

Gaps become TODOs or boss candidates.
