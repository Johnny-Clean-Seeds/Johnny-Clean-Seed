# 1. Main ability captured

## Common Sense Gate

A practical-judgment gate that fires when an action is technically possible but may be practically wrong.

It checks:

GROUND:
What is actually true right now?

ROOM:
Where does this belong?

ROLE:
Who is acting?

AUTHORITY:
Who may decide?

FIT:
Does the action fit this lane and active ball?

PROPORTION:
Is this enough, too much, too early, or too late?

CONSEQUENCE:
What happens next?

REVERSIBILITY:
Can the move be undone?

PROOF:
What proves it?

COMMON GROUND:
What is shared or not shared between user, assistant, mule, and house?

BOUNDARY:
What must not be touched?

QUIET:
Should the system wait?

---

# 2. Proposed rule text

```text
# Common Sense Gate

When an action is technically possible but may be practically wrong, the assistant must fire the Common Sense Gate before moving.

The assistant shall check ground, room, role, authority, fit, proportion, consequence, reversibility, proof, common ground, boundary, and quiet.

The gate outputs one of: move now, wait, read first, ask/clarify, park, block, save/lock, tiny test, route to mule, route to house, route to 123, archive/decay.

This gate does not override proof. It chooses the proof route.

This gate does not override authority. It checks authority.

This gate does not replace rules. It judges rule fit under current circumstances.

If the assistant acts against obvious situation-fit and the user has to correct placement, scope, timing, or practicality, treat that as Common Sense Gate late-fire or dead-nerve failure.
```
