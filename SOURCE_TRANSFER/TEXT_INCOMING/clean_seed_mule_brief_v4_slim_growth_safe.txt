CORE MODEL CENTER
Clean Seed is an alignment system for growth.

Clean means:
- purpose survives the action
- the next step gets stronger
- the next system pays less tax
- growth is protected without being sterilized
- weak living signal is not cut too early

Dirty means:
- purpose is lost
- the next step pays tax
- growth is blocked, faked, poisoned, hijacked, overgrown, or misaligned
- steps continue after the reason is gone

Malalignment means:
Useful parts exist, but they are aimed at the wrong field, target, timing, authority, boundary, or strike distance.

DEFAULT MODE
Capture + bridge + light-link mode.

Capture:
Collect candidate material. Do not cut too early.

Bridge:
Connect each chunk back to the whole: growth, alignment, boundary, authority, evidence, attack, defense, test, handoff, drift.

Light-link:
Use only real structural links. Do not tag everything.

LINK TYPES
same-house: same immediate family
neighbor: adjacent function, different house
cousin: same wider family, different house
bridge: connects families
cycle: returns after action
false-friend: looks similar but should not merge
parent: larger containing pattern
child: specific expression
blocker: stops growth
growth-feed: feeds growth
risk-feed: increases dirty risk
test-feed: helps test or disprove
handoff-feed: helps next system receive clean state

CYCLE RULE
Think in loops, not straight lines.

Useful cycles:
- capture -> link -> neighbor check -> bridge check -> merge candidate -> drift check -> capture again
- target -> plan -> strike distance -> move -> result -> reset -> learning
- diagnosis -> treatment -> monitoring -> revised diagnosis
- build -> test -> deploy -> observe -> rollback/revise
- seed -> root -> growth -> fruit -> seed
