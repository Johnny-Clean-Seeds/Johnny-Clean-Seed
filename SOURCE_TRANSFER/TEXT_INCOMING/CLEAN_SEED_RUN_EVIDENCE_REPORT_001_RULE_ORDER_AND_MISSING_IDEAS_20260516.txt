WHY THIS ORDER WAS CHOSEN
The order follows dependency.

The core model must come first because it defines authority, user direction, evidence, and boundary.

Calculated Precision Triage comes next because it gives the main method spine:
structure first, meaning second; map before judgment; review edge before cleanup.

Housekeeping closure comes next because once a method/master exists, the old pile must be grouped, linked, statused, and removed from the active plate.

Proof/history comes next as evidence because old reports, receipts, hashes, and indexes can support claims, but must not become active authority by default.

Parking-core concepts come next because the system needs a way to make the old pile work in tandem with the active core without making everything mentally active.

Sequence tournament controls come last in this stack because they decide how to install the next large rule cluster without one good-sounding rule corrupting the next.

WHAT THIS RUN PROVED
This run proved that the chosen order is justified as the best current starting order.

It showed a clear dependency chain:
- authority before method
- method before closeout
- closeout before parking remodel
- evidence before cleanup
- parking before large idea implementation
- tournament before clustered rule installation

It also proved that the next stage cannot honestly claim completeness until a Candidate Field Sweep is run.
