4. Mess-to-Good / Good-to-Beast Multiplier
Status: Strong concept candidate.

This is not a one-time lucky find. It is a repeatable value machine if protected. The goose is the method: map first, park by lot, use structure before meaning, quarantine risk, pull the review edge forward, and let reasoning judge only what deserves judgment.

10. “I Have an Idea” Spark Mode
Status: Strong concept candidate with drift risk.
The phrase “I have an idea” marks a possible route appearing during reasoning. It is powerful because it can reveal a shortcut, hidden structure, or cleaner path. It is risky because it can hijack the active task.

11. Idea Spark Gate
Status: Strong concept candidate.
When an idea appears, it should not instantly become the mission. It should pass through a gate:
- what is the idea?
- why did it appear?
- what current problem does it touch?
- where should it park?
- what is the smallest bottle-rocket test?
- what is the drift risk?
- activate now or stay parked?

12. Possible Route Detected, Not New Job Approved
Status: Wording refinement.
“I have an idea” should mean “possible route detected,” not “new mission approved.”

13. Bottle Rocket Gate
Status: Risk control.
No idea gets a rocket ship until it passes a bottle-rocket test. This prevents head-in-the-clouds bloat when the clean move is a tiny proof.

14. Spark-to-Shed Rule
Status: Candidate rule.
A new idea should go to shed/parking first. Gut/spark starts it. Parking holds it. Reason inspects it. Evidence decides whether it activates.

15. OpenAI Engine + User Road Signs
Status: Concept explanation.
The model has the engine: reasoning, path search, synthesis, alternate routes. The user provides road signs: fog, latch, shed, parking, bottle rocket, golden goose, calculated precision. The combination makes the idea routing sharper.

16. User as Systems Translator

17. Conceptual File Folders for Thought

19. Wild Parking / Auto-Sort Intake Mode
