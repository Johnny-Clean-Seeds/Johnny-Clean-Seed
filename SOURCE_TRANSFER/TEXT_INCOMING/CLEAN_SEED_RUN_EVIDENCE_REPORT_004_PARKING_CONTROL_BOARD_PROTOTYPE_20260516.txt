TEST 1 — NEW IDEA APPEARS

Scenario:
A new big idea appears during active work.

Board behavior:
1. Active priority remains visible.
2. New idea goes to Intake Buffer.
3. Candidate Carding Gate decides whether it is:
   - current task support
   - later/parked
   - unplaced value
   - risk/quarantine
   - public/product
   - out of field
4. If outside current field, it goes to Field-Lock Exception Lane.
5. Priority Card blocks hijack.
6. Board refreshes only if the idea changes current state.

Result:
PASS.

The board prevents the idea from being lost and prevents it from becoming active by force.

Missing board field found:
FIELD-LOCK EXCEPTION LANE should be visible on the board when active.

TEST 2 — PUBLISH REQUEST APPEARS

Scenario:
John asks to publish a note/report to GitHub.

Board behavior:
1. Active priority checks whether publish is current lane or side request.
2. Public/OpenAI Lot receives the candidate file.
3. Public-Safe Gate checks:
   - exact file
   - expected hash if needed
   - no bulk proof-history
   - no tokens/secrets/private material
   - destination PUBLIC_NOTES or approved location
   - GitHub outbox shell is correct
   - remote page must be checked after push
4. Worker/Tool Lot handles PowerShell/Git.
5. Run Report Ledger marks public-visible only after remote proof.

Result:
PASS.

The board routes publish safely without letting GitHub become the whole task.

Missing board field found:
PUBLIC VISIBILITY STATUS must show local-only / staged / pushed / remotely verified.

TEST 3 — CLEANUP CANDIDATE APPEARS

Scenario:
An old file, draft, duplicate, or report looks removable.

Board behavior:
1. Cleanup candidate goes to Cleanup Waiting Lot, not Trash.
2. Cleanup Decision Map Gate blocks action.
3. Required before cleanup:
   - exact path
   - reason
   - keeper/proof location
   - duplicate or obsolete proof
   - risk check
   - John approval
   - post-action verification
4. Board shows cleanup as blocked unless cleanup lane is the active priority.

Result:
PASS.

The board prevents "candidate" from meaning "delete."

Missing board field found:
CLEANUP STATUS should show blocked / waiting / mapped / approved / done.
