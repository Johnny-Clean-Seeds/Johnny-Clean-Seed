BOARD STALENESS CHECK
Before action, ask:
- Is there a newer report than the board knows?
- Has the active priority changed?
- Has cleanup/public/tool status changed?
- Has a file been saved or published after the board refresh?
- Has a new idea, file, or request entered?

If yes, refresh the board before acting.

PLAY-DRIFT GUARD
Play, jokes, nicknames, and wizard talk are allowed.

But play is not authority.

Do not create rules, files, reports, commands, doctrine, cleanup, or new missions from playful language unless there is a direct build instruction.

Short guard:
- joke is not command
- spark is not scope
- metaphor is not doctrine
- nickname is not authority
- excitement is not approval
- return to the board before building

If play starts pulling the work sideways:
1. Pause.
2. Name the active task.
3. Classify the play item as joke / handle / parked idea / direct instruction.
4. Continue only if the user clearly approves the build move.

If the assistant creates a rule list from a joke, classify it:
INVALID PLAY-DRIFT.
Then return to the board.
