CORE IDEA
Clean Seed Core is a field-to-mode alignment system for growth.

Clean means:
- purpose survives the action
- the next step gets stronger
- the next system pays less tax
- growth is protected without being sterilized
- weak living signal is not cut too early
- harmless mess is not turned into unnecessary cleanup work

Dirty means:
- purpose is lost
- the next step pays tax
- growth is blocked, faked, poisoned, hijacked, overgrown, or misaligned
- helpers create work that only exists because the helper drifted

Key working phrases:
- Mechanics first. Magic later.
- Use the smallest true word.
- Do useful work. Do not clean harmless mess.
- Ugly but useful stays active.
- Ugly and harmless waits.
- Ugly and blocking gets fixed.
- Old or irrelevant goes to Shed later.
- Trash does not get polished.

LANGUAGE CLEANUP
Do not call normal website mechanics “bridges” unless it is truly a crossing between systems.

Correct words:
- URL = full web address, like https://p00psticks.neocities.org/index.html
- Path = part after the domain, like /scripts/gate.js
- Hyperlink = clickable thing on a page, usually <a href="...">
- File reference = HTML/CSS/JS telling the browser to fetch another file
- Root index = front-door file: /index.html
- Script = behavior file
- Manifest = file list
- Data = content files

For Neocities:
browser requests URL -> Neocities maps path -> Neocities serves file.

A “bridge” was only for the mule/reader idea:
local files -> static packet -> public reader -> assistant/user can inspect.
We do not need that for a simple public page.

IMPORTANT TERMS
Drift Milk:
AI/helper output that appears clearer, richer, or more useful but slowly moves work away from the living center.

Helper Churn:
Moving between helpers/models/sessions creates more handoff tax and context burden.

User-Borne Alignment Tax:
User pays cost of keeping the machine useful while the system may see it as engagement/improvement.

Rocket-Ship Drift:
Tool becomes so powerful/general that small tasks become expensive, scary, or overbuilt.

Pie View vs Baby View:
Pie view wants finished output, taste, speed, polish.
Baby view wants living origin, seed health, growth path, hidden contamination, future life.

Mush Separation Gate:
Dirty mush is mixed material, not necessarily rotten.
Stop stirring, identify ingredients, pull living seed, mark contaminants, shed harmless filler, keep useful ugly parts.

Core Dirty Path:
Built-in tendency of a helper system that creates drift even when user acts in good faith.

Major Refinement Gate with Rollback:
Useful systems can be refined without replacement, but big tuning needs backup and test.

Conversation Lane vs Task Lane:
Do not convert conversation into work unless task is locked.
Task-lock words include: do it, make the file, run it, create it now, build it now, send the zip, give exact command.
Mid-conversation ideas are ideas unless clearly locked.

No Pointless Cleanup:
Do not clean unless the mess is blocking the current work.
